# 3-軟體開發成為 AGI 第一個落地領域的黑色幽默

**類型**：Warning

## 概念

AGI 不是單點突然發生，而是在各領域各自演進。「軟體開發」居然成了 AGI 第一個最接近落地完成的領域——過去一個世紀最聰明的人都在寫程式，最終目標卻是把自己取代掉。這是過去完全沒人料到的黑色幽默。Claude Opus 4.6 和 OpenAI Codex-5.3 的對決已經讓資本市場恐慌（SaaS 股大跌），因為 AI 的 coding 能力展現出「把 spec 翻譯成 code」這件事已經不再是核心競爭力。

## 重要性

理解 AGI 落地的現實狀況，避免在「即將被 AI 取代」的技能上過度投資。

## 邊界/反例

軟體開發的 AGI 化不代表工程師消失，而是價值轉移（從寫 code → 定義問題、架構設計、整合 AI）。能從 Problem Solver 角度出發的工程師反而迎來最好的時代。此外，其他領域的 AGI 化速度可能更慢（如需要物理世界互動的領域）。
