# 7-從 GenAI 功能到可收費 Agent 的三層演進

**類型**：Model

## 概念

GenAI 要從「好看的 BI」變成「可收費的 Agent」需三步：(1) **加任務目標維度**：不是問「畫什麼圖」而是「我要做什麼」（找問題/做週報/實驗結論/異常監控），Agent 依任務類型選流程；(2) **拆成可重複的 Tool Call pipeline**：Query tool、比較 tool、異常偵測 tool、報告組裝 tool——這樣就有可持續優化的 pipeline；(3) **為每種 Agent 指定可度量 KPI**：週報 Agent（生成數量、人工校正時間）、成效診斷 Agent（被採用建議數、uplift）、異常 Agent（提前發現數、損失減少）。做到這三點就能清晰收費、談成果共創。

## 重要性

這是 AI 產品從「功能」升級為「可收費服務」的方法論——關鍵是可驗證的成果 KPI。

## 邊界/反例

需要有穩定的金質數據源（Gold Table）作為基礎；Agent Pipeline 需要持續優化維護成本；客戶需要認同「成果付費」模式。
