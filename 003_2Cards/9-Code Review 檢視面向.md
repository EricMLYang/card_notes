# 9-Code Review 檢視面向

**類型**：Heuristic

## 概念
AI 時代的 Code Review 重點從「語法正確」轉向「系統風險控制」。檢視順序應固定為：需求對齊、失敗模式、資料與權限邊界、可測試性、可觀測性、回滾策略。只有把 review 從程式碼層拉到系統層，才能攔下 AI 快速產碼帶來的隱性風險。

## 重要性
它提供可重複的審查框架，避免 reviewer 被 diff 細節淹沒。

## 邊界/反例
低風險一次性腳本不需完整檢查表；但任何會進正式環境或影響資料安全的變更，都不應跳過系統層 review。

## 知識鉤子
可作為 `#AI Coding Agent 的五種協作模式` 的治理補丁，並與 `#Checkpoints` 類卡片組成交付防線。
