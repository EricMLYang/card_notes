# Code Review 檢視面向

## 類型
開發實踐

## 概念
Code Review 時應檢視的面向：(1) 功能正確性——是否符合需求、邊界案例處理；(2) 程式碼品質——可讀性、命名、結構；(3) 效能考量——時間複雜度、記憶體使用；(4) 安全性——輸入驗證、權限檢查；(5) 測試覆蓋——是否有適當測試；(6) 架構一致性——是否符合現有模式；(7) 文件更新——API 變更是否有對應文件。重點：Code Review 不只是找錯，更是知識傳遞和團隊標準建立的過程。

## 重要性
系統化的 Review 面向可避免遺漏、提升審查品質。

## 邊界
根據專案特性調整優先順序；不要讓 Review 變成阻礙進度的瓶頸。
