# 模型會吸收周邊複雜度（系統結構重要性下降）

**類型**：Principle

## 概念

隨著模型能力提升，原本需要複雜系統架構處理的任務會被模型直接吸收。例如以前做 NLP 需要手動處理語法解析、語意理解、實體辨識等多個模組，現在 Transformer 把整個 stack 都吃掉了。系統的複雜性和精細結構變得不那麼重要，所有智慧都聚合到模型中。這意味著「精巧的工程架構」作為護城河的價值在下降。

## 重要性

幫助判斷哪些技術投資會被模型進步淘汰，避免在「會被吸收的複雜度」上過度投資。

## 邊界/反例

並非所有複雜度都會被吸收。領域知識（domain expertise）、合規要求、系統整合、成本優化等「非智慧型複雜度」仍然需要工程架構。例如：Databricks 的價值在平台整合和治理，不會被模型吃掉。
