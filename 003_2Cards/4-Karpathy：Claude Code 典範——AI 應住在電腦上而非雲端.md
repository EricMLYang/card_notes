# 4-Karpathy：Claude Code 典範——AI 應住在電腦上而非雲端

**類型**：Principle

## 概念

Karpathy 2025 回顧重點：Claude Code 是第一個令人信服的 LLM Agent 展示。「我認為 OpenAI 搞錯了方向，他們把早期 Agent 努力放在雲端容器部署，而不是簡單的 localhost。」關鍵區別不在於 AI 運算在哪裡，而在於**已存在的所有東西**：電腦環境、安裝、上下文、資料、密鑰、配置，以及低延遲互動。Anthropic 正確理解了這個優先順序，把 Claude Code 包裝成極簡 CLI。「這改變了 AI 的樣貌：它不只是你去訪問的網站，它是『住』在你電腦上的小精靈/幽靈。這是與 AI 互動的全新典範。」

## 重要性

這是判斷「AI Agent 產品方向正確性」的核心洞見——localhost > cloud 在現階段。

## 邊界/反例

雲端 Agent 群集確實是「AGI 終局」方向；但在當前能力參差、需要人類緊密協作的階段，localhost 模式更合適。
