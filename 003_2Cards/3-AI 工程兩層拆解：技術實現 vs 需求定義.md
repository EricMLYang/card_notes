# AI 工程兩層拆解：技術實現 vs 需求定義

## 概念

AI 工程包含兩層：

1. **讓模型做你想做的事**（prompt engineering、RAG、框架）→ 這層會被商品化/簡化

2. **搞清楚你到底想要什麼**（品味、判斷力、UX 設計）→ 這層無法被模型取代

就算未來第一層變簡單，第二層永遠逃不掉。

如 Cursor vs GitHub Copilot 長期用同樣模型，產品體驗卻天差地遠，差異來自「非模型部分」的設計與整合。

## 重要性

這是投資/招聘/團隊分工的判斷框架——要區分哪些能力會被工具取代、哪些是持久價值。

## 邊界/反例

若場景極度標準化（如「給我總結」），第二層重要性下降；但任何需要「判斷取捨」的場景（如寫程式碼要考慮可讀性/可維護性），第二層就是關鍵。
