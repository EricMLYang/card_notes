# 3-Agent 架構的做空檢驗：模型翻倍時你的系統會不會自動變簡單

**類型**：Heuristic

## 概念

大多數 AI Agent 框架靠加更多節點、子代理、工作流程提升可靠性，本質是在 scale 人力而非計算能力。用投資語言講：花三個月精心設計的 workflow 是一張做空波動率的單子——模型沒進步時賺穩定小錢（系統照常運行），模型能力跳升時你之前固化的架構不會自動變簡單，反而成了技術負債，你爆倉了。檢驗標準極簡：**如果明年模型能力翻倍，你的系統會不會在不做大改的情況下變得更便宜、更簡單、更可靠？** 答案是否定的，你就把太多「智慧」塞進了腳手架而不是留給模型。

## 重要性

這是設計 Agent 架構時最核心的一個自我檢驗問題，能避免在即將過時的人工設計上過度投資。

## 邊界/反例

Thin Harness ≠ No Harness——產品需要確定性、可稽核性和安全控制，不是今天就拆掉所有 workflow。短期內模型進步可能停滯，此時 workflow 投資有正回報。
