# 8-好 API 設計可以維持很久——Stripe V2 的穩定抽象哲學

**類型**：Principle

## 概念

Stripe CEO Patrick Collison 的 API 設計哲學：**好的抽象設計可以維持很久**，讓開發者在不改變程式碼的情況下應對底層系統的變動。Stripe V2 API 設計的核心原則：提供穩定的、不會隨意改變的介面。這與 iOS 的設計理念相符——好的抽象層可以讓開發者安心使用，不用擔心底層變動。同時，技術選擇會對組織產生深遠影響：早期選擇 MongoDB 和 Ruby 的決策，在多年後仍持續影響公司發展。AI 在軟體開發的角色：不僅能提高程式碼的產量，更能提升程式碼的品質。

## 重要性

這是判斷「API 設計是否成功」的核心標準——穩定介面勝過華麗功能。

## 邊界/反例

過度追求穩定可能導致無法適應技術演進；某些快速迭代的領域需要更彈性的 API 設計。
