## 標題
AI Agent 時代的可觀測性挑戰：衡量成為協作瓶頸

## 類型
Principle

## 概念
當 AI Agent 數量增加且持續自主運作時，**可觀測性（Observability）成為協作效能的核心瓶頸**。人類無法像監督人類員工那樣「感受」Agent 的工作狀態，必須依賴明確的衡量指標。

**兩個關鍵設計點**：

1. **指標選擇的任務適配性**：
   - **KPI 思維**：適合量化、重複性任務（如資料處理、報表生成），追求效率與準確率
   - **OKR 思維**：適合創意、探索性任務（如內容生成、策略建議），追求結果價值而非過程控制

2. **快速反饋修正迴圈**：
   - AI 執行速度快，但若無法快速獲得反饋並修正，效率優勢會被「錯誤方向的高速執行」抵消
   - 需設計自動化評測與人工抽檢的混合機制

這揭示一個矛盾：AI 讓自動化更容易，但也讓「設計正確的衡量方式」變得更關鍵且困難。

## 重要性
幫助判斷 AI Agent 系統的設計重點：不只是 Agent 能力，更是可觀測性與反饋機制的設計。適用於評估 Agent 架構的成熟度。

## 邊界/反例
- 過早優化衡量系統可能拖慢原型驗證速度
- 某些創意任務難以量化，過度依賴指標會扼殺探索空間
- 小規模 Agent 部署（< 5 個）可能靠人工監控即可，不需要複雜系統
