# 1-一切都是 Context：對 AI 和對人的底層能力是同一個

**類型**：Principle

## 概念

不了解對方的 Context，你的提案就是噪音——不管對方是 AI 還是人。理解 Context 的四個維度：痛點、KPI、目標、真正的問題。三種基於 Context 的策略：(1) 順勢——對方 Context 跟你一致，直接推；(2) 借勢——從對方的優先順序出發重新包裝你的提案；(3) 造勢——所有利害關係人 Context 互相衝突時，找出重疊的焦點，發展多贏提案。這個能力對 AI 和人一樣通用：AI 不了解你的 Context，回答馬馬虎虎；老闆不了解你的 Context，提案不買單。

## 重要性

這是跨領域最可遷移的底層能力——寫 prompt、做提案、推動決策、設計產品，底層都是 Context 管理。

## 邊界/反例

Context 理解需要資訊取得管道；在資訊不對稱嚴重的場景（如跨國、跨文化），Context 判斷容易出錯。過度揣測 Context 也可能導致過度包裝、失去真實性。
