# 1-Harness 是品味的具體實現

**類型**：Principle

## 概念

短期來看，AI 讓平庸的執行變得廉價；長期來看，AI 讓平庸的決策變得致命。當任何人都能生成任何東西，優勢轉移到知道「什麼值得存在」的人手上——那就是品味。但品味本身是抽象的，**Harness 是品味被具體實現的方式**：給它什麼工具？強制什麼順序？用什麼衡量成功？怎麼從失敗恢復？記住什麼？拒絕什麼？所以你的 Harness 就是一面鏡子，映照出你的價值觀。在複製貼上 Agent 的世界裡，價值觀就是差異化。

## 重要性

這連結了「品味」這個抽象概念與「系統設計」的具體實踐——把你會什麼變成你做了什麼。

## 邊界/反例

品味需要先被培養才能被實現。如果品味本身有問題（如對目標領域理解不深），再好的 Harness 也只是放大錯誤。此外，某些領域的「好」有客觀標準（如 coding 有測試），品味的空間相對有限。
