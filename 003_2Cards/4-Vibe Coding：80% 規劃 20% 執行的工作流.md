# 4-Vibe Coding：80% 規劃 20% 執行的工作流

**類型**：Heuristic

## 概念

Lovable 的官方 Vibe Coding 工程師 Lazar（從沒寫過一行程式）的工作流：花 80% 時間在規劃和對話，只有 20% 在執行。大多數人用 AI 的方式是：打開工具 → prompt → 遇問題 → 繼續 prompt → 越改越亂 → 放棄。正確流程：(1) 用「腦袋傾倒」模式讓 AI 釐清思路；(2) 同時開 5-6 個專案測試不同方向；(3) 找到最有潛力的方向後花一整天做規劃文件；(4) 執行變得極簡單，只需說「繼續下一個任務」。這方法看起來慢，實際省下指數級時間——避免在錯誤方向上越走越遠。

## 重要性

這是 Vibe Coding 的核心方法論——大多數人優化的是錯誤的速度。

## 邊界/反例

某些緊急修復或小型任務不需要這麼完整的規劃流程。此外，80/20 的比例可能因任務複雜度調整——簡單任務可以 50/50，複雜任務可能 90/10。
