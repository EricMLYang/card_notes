# 3-垂直 AI 護城河：Domain Know-how + Change Management，不是 API Wrapper

**類型**：Principle

## 概念

任何人都能寫 API Wrapper，那沒有護城河。真正的護城河是深入醫療帳單、法律搜證、營造合規這些又髒又重的 Domain Know-how，再加上能解決舊系統摩擦力並完成變革管理（Change Management）的能力。Garry Tan 預測：過去 20 年 SaaS 創造 300+ 獨角獸，接下來每個 SaaS 獨角獸都會對應誕生一個 Vertical AI 獨角獸，而且規模大 10 倍——因為它們取代的不只是軟體，還有操作軟體的人。策略：選自己懂的 Domain，鎖定單一痛點節點（人類覺得很煩、流程固定、極度耗時），做到極致。

## 重要性

提供了 AI 產品選題的決策框架——不是追技術前沿，而是深入特定領域的痛點。

## 邊界/反例

Domain Know-how 護城河的前提是領域足夠複雜和受監管；在簡單、標準化的領域（如基本文書處理），護城河很淺。大企業可能用資源優勢快速複製垂直方案。
