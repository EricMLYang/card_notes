# 4-Git 處理不了 AI 時代洪流——Checkpoints 綁定提示詞與代碼

**類型**：Warning

## 概念

前 GitHub 執行長 Thomas Dohmke 的新創 Entire 獲得史上最大種子輪（$60M，估值 $300M），核心洞見：**沿用幾十年的 Git 版本控制系統，根本處理不了 AI 時代的洪流**。問題描述：AI Agents 生成程式碼的速度遠超人類理解能力，導致開源專案被大量低品質的「AI 廢料」淹沒。解決方案：Entire 的核心產品 **Checkpoints** 能把 AI 寫程式的「提示詞」和「思考脈絡」跟代碼綁定——這不只是單純的 debug，而是讓人類工程師在 AI 海嘯中真的能看懂「機器在想什麼」。這是 AI coding 當前最大的痛點：代碼生成速度超越人類審核能力。

## 重要性

這是判斷「AI coding 下一個基礎設施需求」的信號——版本控制需要升級。

## 邊界/反例

小型專案可能不需要這種複雜度；某些 AI 生成的代碼確實品質很高不需要追蹤。
