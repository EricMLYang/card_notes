# 台灣 AEC 智慧出圖平台開發提案

## MVP：地籍現況展繪圖自動生成系統

## 一、提案摘要

本提案旨在開發一套面向台灣 AEC（建築、景觀、水利、營造）產業的智慧出圖平台，將自然語言或表單輸入，轉換為可交付的工程圖面。系統核心採用 **LLM + 規則引擎 + Python 幾何核心 + DXF/DWG 輸出** 的方式，目標不是讓 AI 直接「畫 CAD」，而是讓 AI 負責理解需求，再由受控的幾何與出圖模組穩定產出圖面。

本案第一階段聚焦在 **地籍現況展繪圖**。這個場景的原因很實際：台灣官方已有可申請的地籍圖 DXF / SHP 資料來源，資料型態穩定、幾何問題明確、成果容易驗證，也最適合作為規則化出圖平台的第一個 MVP。國土測繪中心目前提供 GIS 地籍圖檔，格式包含 SHP 與 DXF；同時，也提供其他台灣通用電子地圖與地形圖資料，適合作為底圖或套疊資料來源。([國家圖書館][1])

從市場角度看，AEC 軟體已經明顯朝「AI 輔助 + 參數化 + 自動生成」前進。Autodesk Forma 已將 AI-powered tools 用於 pre-design 與 schematic design；VIKTOR 也直接定位為讓工程師建立 AI-powered apps、workflows 與 agents；TestFit 則已把停車配置與 site feasibility 做成高度自動化產品；Hypar 也在做 building design logic 的雲端化與自動生成。這表示本案方向不是孤例，而是順著產業主流演進，只是目前仍缺少真正深度貼合 **台灣法規、中文出圖、地方作業習慣** 的在地化產品。([Autodesk][2])

---

## 二、專案背景與問題定義

台灣 AEC 產業中，許多工作仍高度依賴人工整理資料、套用既有 CAD 樣板、輸入標註、檢查尺寸、匯出 DWG，再交由工程師或技師進一步整併到正式圖框中。這些作業的共同特性是：

1. 規則明確
2. 幾何邏輯可公式化
3. 圖面重複性高
4. 仍需人工覆核，但前段可高度自動化

現況最大的浪費，不在於缺少繪圖軟體，而在於大量工程師時間被消耗在「重複而可規則化」的出圖前處理工作上。這類工作若能標準化，就能明顯縮短交圖時間、降低初稿錯誤率，並提高團隊對標準圖、構造圖、基地圖、現況展繪圖的處理吞吐量。

本提案不嘗試讓 LLM 直接輸出底層 DXF 座標或 CAD 指令。原因是幾何與法規判斷需要可追溯、可測試、可驗證的 deterministic engine。LLM 在本系統中的定位是：**做需求理解、參數轉換、缺參提醒、操作引導**；幾何推導、圖層控制、尺寸與表格生成則全部交由 Python 模組處理。

## 三、專案目標

### 1. 主要目標

建立一套可擴充的智慧出圖平台，讓使用者以簡單輸入條件，即可自動生成符合台灣業界作業習慣的 2D 工程圖面，並輸出 DXF / DWG 供 AutoCAD、ZWCAD、Civil 3D 等工具接續使用。

### 2. 第一階段目標

以 **地籍現況展繪圖** 作為 MVP，完成以下能力：

* 匯入地籍資料或座標資料
* 自動生成宗地封閉多邊形
* 自動標示地號、面積、邊長
* 疊加底圖或參考圖層
* 在網頁中直接預覽圖面
* 匯出 DXF / DWG 供後續正式圖面整併

### 3. 第二階段擴充方向

擴充到 **法定停車空間劃設**。這個方向的可行性也很高，因為台灣《建築技術規則建築設計施工編》已對車位尺寸、車道寬度、轉彎與停車位前方淨空有明確規定，例如一般汽車停車位為寬 2.5 公尺、長 5.5 公尺；單車道寬度 3.5 公尺以上，雙車道 5.5 公尺以上；停車位角度超過 60 度時，前方需留設深 6 公尺、寬 5 公尺以上空間。這類規則很適合做成演算法。([law.moj.gov.tw][3])

## 四、MVP 方案定義：地籍現況展繪圖

### 1. 使用情境

使用者匯入地籍圖檔、宗地座標 CSV、或指定地號與地段資料後，系統自動完成：

* 宗地外框生成
* 面積計算
* 地號標註
* 基本圖層整理
* 展繪圖輸出
* 預覽與修正

### 2. 使用者角色

* 建築師事務所助理 / 繪圖人員
* 土木、水利、景觀顧問公司工程師
* 營造廠工程規劃人員
* 政府標案前期資料整理人員
* 技師簽證前的圖面整理支援人員

### 3. 為什麼先做這個

因為它具備 MVP 最重要的條件：

* **資料來源官方可得**：國土測繪中心可申請地籍圖 SHP / DXF。([國家圖書館][1])
* **幾何規則清楚**：封閉多邊形、面積、標註、套疊屬於成熟問題。
* **成果容易驗證**：很容易比對人工圖與系統圖。
* **法規責任較低**：比起結構配筋或施工架安全佈置，風險較小。
* **可自然擴展**：之後可往停車、標準斷面、水利構造圖擴充。

## 五、系統解決方案概述

### 1. 核心設計原則

本系統採「AI 理解需求，Python 負責真值」的架構：

* LLM：負責需求理解、參數整理、欄位補全、錯誤提示
* 規則引擎：負責法規與繪圖規範判斷
* 幾何引擎：負責座標、面積、偏移、交疊、裁切等計算
* Drafting Engine：負責圖層、文字、標註、表格、樣板套用
* Renderer：負責網頁端預覽
* Exporter：負責 DXF / DWG 輸出

### 2. 技術路線

技術選型上，Shapely 適合做平面幾何分析與操作；ezdxf 適合做 DXF 讀寫與繪圖資料生成；ezdxf 的 drawing/export add-on 可將 DXF 轉為向量或點陣圖供預覽；ODA File Converter 提供 GUI 與 command-line interface，可用於後端靜默轉換 DWG / DXF 檔案。([Shapely][4])

### 3. 標準流程

1. 使用者輸入地號、上傳座標 CSV、或匯入 DXF/SHP
2. 系統解析資料並轉成標準 JSON / geometry objects
3. 幾何引擎生成宗地圖形、面積、邊界資訊
4. Drafting Engine 套用台灣圖層與標註樣板
5. Renderer 產生 SVG / PNG 預覽
6. 使用者檢查並修正輸入條件
7. 確認後輸出 DXF / DWG
8. 匯入 AutoCAD / ZWCAD / Civil 3D 進行正式交圖整併

* 驗證圖層名稱與文字樣式是否符合樣板

輸出：

* `validation_report.json`
* `warnings.json`

功能：

* 多邊形建立
* 偏移
* 相交/聯集/差集
* 標註位置推算
* 面積與邊長計算

功能：

* 載入標準 DXF 樣板
* 套用圖層、線型、文字樣式
* 寫入宗地輪廓、界址點、文字標示
* 自動生成表格或面積欄位
* 輸出 `.dxf`

## M5. 預覽模組

功能：

* DXF 轉 SVG / PNG
* 前端圖面預覽
* 放大縮小
* 基本圖層開關
* 標註位置檢查

ezdxf 官方文件說明其 drawing/export add-on 可將 DXF 渲染為 rasterized 或 vector-graphic image；也支援與 matplotlib 等 backend 搭配輸出 SVG / PNG / PDF。([ezdxf][5])

## M6. 匯出模組

功能：

* DXF 版本管理

## 十、風險與因應

### 1. 中文字型與 SHX / TTF 差異

這是最大技術風險之一。ezdxf 官方文件明確提到，SHX 字型格式未公開，也不被 Matplotlib、Qt 等許多套件原生支援，因此預覽時通常需要對應的 TTF 或額外字型支援；另外，某些 backend 的文字呈現不會與 AutoCAD 完全 pixel perfect。([ezdxf][7])

**因應：**

* 第一天就定義字型策略
* 固定伺服器字型環境
* 建立 CAD 實測清單
* 區分「預覽一致性」與「最終 CAD 一致性」

### 2. 預覽不等於最終交付

ezdxf 的 rendering backend 很適合做 2D 預覽，但官方也列出限制，例如不是完整 3D rendering engine、MTEXT rich text 不是 pixel perfect、部分 entity 支援有限。([ezdxf][8])

**因應：**

* 預覽僅做邏輯檢查
* 正式交付前以 CAD 軟體抽驗
* 建立匯出版本驗證流程

### 3. 法規與作業慣例差異

各機關、事務所、技師或標案可能有不同圖層、字高、圖框與標註習慣。

**因應：**

* 將樣板做成可配置模板
* 區分核心 engine 與樣板層
* 先支援 1–2 家目標單位樣式

### 4. LLM 生成腳本的不可控性

若讓 LLM 動態生成大量繪圖邏輯，會造成版本控管與錯誤追溯困難。

**因應：**

* 正式系統只允許 LLM 輸出結構化參數
* 幾何與出圖邏輯全部走固定函式庫與受控模板

## 十三、結論

本案的關鍵不是「讓 AI 替代 CAD」，而是建立一個 **受控、可測試、可累積規則資產的智慧出圖平台**。以地籍現況展繪圖作為第一個 MVP，最能降低風險、快速做出可驗證成果，也最適合作為後續停車配置、標準斷面圖、水利構造圖等場景的基礎平台。

這個方向在技術上可行，在產業上也有相近案例可對照；真正的差異化會出現在：
**台灣資料源整合、中文樣板、法規規則化、以及可直接接上本地 CAD 工作流。** ([國家圖書館][1])
