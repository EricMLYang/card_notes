未來的 LLM 運算瓶頸將從 GPU 轉移到儲存 I/O ？！

Agent 場景中，模型與環境互動數十至數百輪，每輪只新增少量 token，但累積上下文極長，超過 95% 可直接從 KV-Cache 讀取。這讓瓶頸從傳統的 GPU 運算轉移到儲存 I/O，GPU 大部分時間不是在算，而是在等 KV-Cache 從遠端儲存讀回來。

現有 PD 分離架構中，只有 Prefill 引擎負責從儲存讀取 KV-Cache，其網卡頻寬被打滿，Decode 端的網卡卻閒著，進一步加劇了儲存 I/O 瓶頸。

DeepSeek 提出 DualPath：讓 Decode 端也能幫忙讀取 KV-Cache，再透過高速 RDMA 網路轉給 Prefill 端，把原本閒置的頻寬利用起來。搭配 QoS 流量隔離確保不影響模型推論、以及自適應排程平衡各節點負載。

在真實 Agentic 工作負載下，離線推論最高加速 1.87 倍，線上服務平均加速 1.96 倍，並展示至 1,152 GPU 的近線性擴展能力。
