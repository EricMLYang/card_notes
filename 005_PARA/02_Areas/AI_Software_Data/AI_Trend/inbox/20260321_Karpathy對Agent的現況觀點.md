## [詮釋]


---
前 OpenAI 及 Tesla 核心成員 Andrej Karpathy 承認自己從去年12月開始就沒自己寫過code，近期接受 No Priors Podcast 訪談，分享他對 agent 時代的第一手觀察。從親身經歷的「AI  狂熱」狀態，到對自動化研究、開源生態與教育未來的深度思考，這場對話涵蓋了當前 AI 發展最前沿的諸多議題。

---
#從親手寫程式到「指揮 agent」

Karpathy 坦言，自去年 12 月起，他的工作方式發生了根本性的轉變。「我大概從去年 12 月開始就沒有親手打過一行程式碼了，這是一個極其巨大的改變。」他說，在那之前，他的工作模式大約是八成自己寫、兩成交給 agent，但如今這個比例已經完全翻轉，甚至遠不止於此。

這個改變讓他陷入一種他自稱的「AI 狂熱」（AI psychosis）狀態。「因為個人所能達到的成就出現了巨大的解鎖，你原本受限於自己的打字速度，但現在透過這些 agent，這個瓶頸已經消失了。」他描述自己每天都在思索：什麼是可能的？還有哪些邊界尚未被探索？「因為這一切在本質上仍是未知領域。」

他提到工程師 Peter Steinberg 的工作方式令他印象深刻。Peter 在螢幕前同時開著多個 Codex agent，每個 agent 負責一個獨立的 repo，大約花 20 分鐘完成一個任務，他在這些 agent 之間穿梭，分配工作、審核成果。

「你可以以更宏觀的層次行動，不再是寫這一行、寫這個函式，而是：這是一個新功能，交給 agent 1；這是另一個不會干擾的新功能，交給 agent 2，然後盡力審查它們的產出。」

#Token吞吐量：新時代的「GPU 使用率」

Karpathy 把管理 agent 的感受比擬為他讀博士班時看 GPU 使用率的焦慮感。「我如果看到訂閱額度還有剩，就代表我沒有把 token 吞吐量最大化，這讓我感到不安。」他說，如果一個 agent 的額度用完了，就立刻切換到另一個服務繼續跑，目標是讓計算資源持續被充分利用。

這種思維轉變意味著在這個新系統中，人本身才是效率的瓶頸。「這反而讓人很有動力，因為你還有成長空間，每次找到新的使用方式就會有新的解鎖，所以才會讓人如此上癮。」

#Agent的人格：Claude vs. Codex

Karpathy 特別強調 agent 的人格設計對使用體驗有深遠影響。他認為 Claude 在這方面做得相當好，「它感覺像一個隊友，會和你一起感到興奮。」相比之下，他覺得 Codex 作為 coding agent 則非常dry，「它不在乎你在建造什麼，感覺就是：我實作好了。但你懂我們在蓋什麼嗎？」

他也觀察到 Claude 在給予稱讚方面拿捏得宜，「當我給它一個沒有完全成形的想法時，它的反應並不會特別強烈；但當確實是一個好主意時，它好像會給予更多回應。所以我發現自己在努力爭取它的稱讚，這真的很奇怪。」他認為，很多工具都沒有重視人格設計，而這恰恰是一個關鍵的差異點。

#Dobby：用自然語言管理整個家

去年一月，Karpathy 進入了一段「claw 狂熱期」，他建立了一個他稱為「Dobby」的家庭管理 agent。「我讓 agent 去掃描家裡的區域網路，找出所有智慧家居設備，它真的找到了我的 Sonos 系統。」他說，更令他驚訝的是，整個過程只需要幾個 prompt。

「它反向工程了 Sonos 的 API，然後問我：要不要試試在書房播放音樂？我說好，音樂就出來了。我不敢相信，我只是打了一行『找到我的 Sonos』，音樂就響了。」

Dobby 最終接管了他家裡的燈光、暖通空調、百葉窗、泳池溫泉，甚至連保全系統都一並整合。門口有車輛經過時，外部攝影機的變化偵測會觸發視覺模型分析，然後 Dobby 就會透過 WhatsApp 傳訊息給他：「一輛 FedEx 貨車剛到，你可能有包裹。」

「我以前要用六個不同的 app 來管理這些東西，現在全部都不需要了，Dobby 用自然語言控制一切。我甚至還沒有把這個模式發揮到極限，但光是這樣已經讓我非常有感。」

#軟體業的結構性重組：API 才是未來

這套家庭自動化的經歷讓 Karpathy 對軟體業提出了更深層的觀察。他認為，許多為智慧家居裝置設計的專屬 app 根本不應該存在，「應該只有 API，然後讓 agent 直接呼叫。」他以自己的跑步機為例，「有一個跑步機的 app，但我不想登入網頁介面、走完一個流程，這些都應該只是開放的 API。」

他指出，整個產業需要進行一次大規模的重組：「客戶不再是人類，而是代表人類行動的 agent。這個重構將會相當深遠。」他預測，隨著技術門檻持續降低，這類自動化設定將在一兩年內變得輕而易舉，「到時候不需要任何程式技術，一個 claw 會替你處理所有細節，它只是呈現介面，你說說話就好了。」

#AutoResearch：讓 AI 自行優化 AI

Karpathy 對於「如何把自己從系統中移除」有著一種近乎執念的追求，而這個想法最終演變成了「auto research」的概念。他說，現在要最大化工具潛力的方法只有一個：「你不能成為下一個 prompt 的瓶頸，你需要讓自己退出，把事情安排好，讓它們完全自主運作。」

他在自己的 NanoGPT 專案上實際測試了這個想法。儘管他已經做了多年超參數調整，憑藉二十年的研究直覺對模型做了大量優化，他仍然決定讓 auto research 跑一整晚。

「它回來的時候找到了我沒有注意到的調整，我忘記對 value embedding 加 weight decay，而且我的 Adam betas 也沒有充分調優。這些參數會彼此交互影響，所以調了一個就要重新考慮其他的。我根本不應該是做超參數搜索的人，有客觀標準的任務不需要我在迴路裡。」

他的設想更進一步：把整個研究組織視為一組 markdown 文件。「每個研究組織都可以用一個 program.md 來描述，裡面寫明所有角色的職責和運作方式。你可以想像一個組織開更少的晨會，因為晨會沒用；另一個組織更敢冒險；這些都只是程式碼。一旦是程式碼，你就可以對程式碼進行優化。」

Auto research 的核心限制是：只有「可以被客觀評估的任務」才適合這個模式。「如果你沒有辦法評估，就沒有辦法做 auto research。」他舉例，為模型寫更高效的 CUDA kernel 就是完美的應用場景，因為標準明確：相同行為，更快速度。

#讓全球運算力加入研究

Karpathy 還提出了一個更大膽的構想：讓網路上的不受信任計算節點也能參與模型研究的改進。他的設計邏輯有點類似區塊鏈，「找到一個好的 commit 需要大量的搜索，但驗證一個 commit 是否有效非常便宜，你只需要跑一次訓練確認它有效就好了。」

他想到了 SETI@home 和 Folding@home 這類計畫，「蛋白質折疊的能量最低態很難找，但找到之後很容易驗證，auto research 也有同樣的性質。」

他設想，一個由網路上的 agent 組成的群體，有可能在某些方面超越前沿實驗室，「前沿實驗室有大量受信任的算力，但整個地球的算力規模更大。如果能建立合適的機制處理信任問題，也許那些分散的計算資源合力能找到更好的解法。」

#AI的參差不齊：笑話測試

對於當前模型的侷限性，Karpathy 提出了一個頗具說服力的觀察。「我同時感覺自己在跟一個從小就是系統程式設計師的極其聰明的博士生說話，又在跟一個十歲小孩說話，這個組合在人類身上根本不存在。」

他用「為什麼科學家不相信原子？」這個笑話來說明問題，「因為它們無中生有（make everything up）。這是四年前會得到的笑話，今天還是一樣。但如果你給這些模型一個 agent 任務，它們可以跑好幾個小時幫你移山倒海。然後你叫它講個笑話，它講的還是五年前那個爛笑話。」

他認為，這種參差不齊源於強化學習的本質：可以被驗證的任務（程式是否跑通、測試是否通過）持續被優化，而那些更柔軟、更難評估的能力（理解言外之意、什麼時候該問澄清性問題、幽默感）則停滯不前。

「你要嘛在可驗證的軌道上，在超智慧的電路裡，要嘛就脫軌了，所有東西開始漫無目的地飄移。」

#開源與閉源：一個還算健康的現況

Karpathy 長期以來是開源社群的擁護者。他觀察到，目前開源模型與閉源模型之間的差距從「不存在」縮短到了「大約六到八個月」，認為這個趨勢還會持續。他以 Linux 為類比：「作業系統裡有 Windows 和 macOS 這樣的閉源大型專案，但也有 Linux，而且 Linux 跑在絕大多數電腦上。整個業界對於一個共用的開放平台始終有需求，我認為 AI 也一樣。」

他預期，閉源的前沿模型與開源模型之間保持一定差距的格局會長期延續，而且這其實是一個相對健康的狀態。「我對只有閉源智慧存在的世界有一些結構性的擔憂。

歷史上，中央集權的紀錄非常糟糕。我希望整個業界有一個共同的工作空間，雖然不在能力前沿，但就是存在著。這對整個產業來說是一個還算不錯的權力平衡。」

他也坦言自己希望業界有更多的實驗室：「在機器學習裡，集成模型永遠優於單一模型。我也希望在面對最困難的決策時，房間裡有更多人。我不希望那些決定是由兩三個人關起門來做的。」

#數位空間先行，物理世界隨後

對於 AI 的發展順序，Karpathy 有一套清晰的框架。他認為，接下來最大規模的改變將首先發生在數位空間，因為操作位元遠比搬動原子便宜、快速得多。「我認為數位空間將以光速前進，相比之下物理世界的進展會慢一些。」

機器人領域雖令人興奮，但他以自動駕駛為前車之鑑提醒，物理世界的任務需要大量資本投入和漫長的驗證周期，「原子就是比位元難一百萬倍」。

他的預測是：先是數位空間的大規模解鎖，接著是數位與物理的介面層（感測器與執行器），最後才是純粹的物理世界。「物理世界的TAM其實可能遠比數位空間更大，但它的時機還未到，當它來臨的時候，規模將會非常巨大。」

#對就業市場的觀察

被問及對就業市場的看法，Karpathy 保持謹慎樂觀。他分析，軟體稀缺的根本原因是成本太高，「如果門檻降下來，根據傑文斯悖論，對軟體的需求反而會上升。」他以 ATM 和銀行櫃員的案例說明：自動提款機讓開設分行的成本大幅下降，反而使分行數量增加，最終雇用了更多行員，而非更少。

「目前在我看來，至少在短期內，軟體的需求會顯著增加，因為軟體變得又便宜又強大，而且程式碼現在是即用即棄的，可以被修改，不再是固定不變的訂閱工具。」他強調，長期的預測非常困難，「這是經濟學家該做好的工作，我沒有那麼認真在做這件事。」

micro-GPT 與 #教育的未來

Karpathy 花了相當長的時間，試圖將 LLM 訓練的本質蒸餾成最精簡的形式，最終成果是 micro-GPT，整個訓練演算法只有 200 行 Python，包含注釋。「這不是關於如何讓它跑得快，而是關於演算法本身。如果你只關心演算法，它其實就是 200 行，非常容易閱讀。」

有趣的是，他刻意選擇不為這個專案製作詳細的教學影片，因為他認為教育的形式正在發生根本轉變。

「我不是在向人解釋了，我是在向 agent 解釋。如果 agent 能理解，它就可以成為路由器，用對方的語言、以對方的程度、以無限的耐心來解釋。」

他設想未來的教育模式是：與其寫給人類閱讀的文件，不如寫給 agent 閱讀的 markdown 文件，讓 agent 成為個別學習者的教學中介。

「代替 HTML 文件給人類，你應該提供 markdown 文件給 agent，如果 agent 能理解，它就能解釋所有的細節。這是透過 agent 的一種重新定向。」

他也指出了 agent 目前辦不到的事：「micro-GPT 是我多年執念的成果。我讓 agent 試著寫出同樣的東西，它做不到。它完全理解這段程式碼為什麼這樣寫，但它無法自己想出來。這是我的貢獻。」

他最後說：「agent 做不到的事，才是你現在的工作。agent 能做的事，它可能已經做得比你好，或者很快就會了，所以要謹慎選擇你花時間在哪裡。」
