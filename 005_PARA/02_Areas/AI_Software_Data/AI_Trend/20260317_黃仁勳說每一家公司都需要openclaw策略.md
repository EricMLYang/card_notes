## [詮釋]


---

黃仁勳說：「每一家公司」都需要一套「OpenClaw 策略」，什麼是OpenClaw 策略？Nvidia 的 NemoClaw 又是什麼？

▋AI Agent 到底跟以前的 AI 助理差在哪裡？

算力往本地端走的背後有一個推力，就是 Agentic AI 的崛起。
過去我們用 AI，每次都是獨立的對話：你問一個問題，它回答，然後結束，等待下一輪對話重新開始。前一輪的狀態不會延續下去， AI 只是一個工具，你拿起來用，用完就放在那邊了。

但是 Agentic AI 的邏輯完全不同，它不等你問出問題，就能自己把一個複雜目標分解成多個步驟、自己決定用哪些工具、自己生出「子智能體」去執行不同部分的任務，然後把結果整合起來。它的工作是持續的，可能跑幾個小時、幾天，中間不需要人介入。

一月的時候 OpenClaw 的出世和爆紅，又創造了另一個 AI 新時刻，OpenClaw 讓 AI 自己寫程式、自己產生更多 AI 去處理子任務、自己學會用各種專業工具，至今的發展速度反應市場等這個酷東西已經很久了！

但這裡有一個根本矛盾，Agentic AI 越有用，就越需要廣泛存取你的資料和系統，因為它需要知道很多才能自主行動，但企業偏偏不可能把機密資料、客戶資料、還有各種專有系統的存取權限，全部開放給一個跑在雲端伺服器上的 AI。 

而且雲端本身還有另一個問題，Agentic AI 的工作流不是間歇性的，它是連續的。每一次 API 呼叫都要繞一趟雲端，就有延遲，也有費用。一個長時間持續運行的 Agent，每天產生的推論請求量可能是一般 AI 助理的幾十倍，雲端費用很快會失控。

所以，本地算力解決的就是這兩個問題，一個是推論在你自己的環境裡跑，資料不出門，延遲幾乎是零，另外是成本也從按用量計費變成一次性的硬體投資。

▋NemoClaw × OpenClaw

黃仁勳在 GTC 表示 OpenClaw「向所有人開啟了 AI 的下一個疆界，成為史上成長最快的開源專案」，過去「Mac 和 Windows 是個人電腦的作業系統，未來 OpenClaw 是個人 AI 的作業系統。」

他也認為「每一家公司」都需要一套「OpenClaw 策略」，並預言 SaaS 公司將從賣軟體轉型為賣 agent，轉變為「AaaS」-Agentic AI as a Service 的商業模式。

同時，輝達也宣布了推出 NemoClaw，為了讓企業能安全使用這項技術，輝達與 Open Claw 的開發者合作，整合了名為 Open Shell 的安全技術。

把幾個名詞放在一起看，簡單來說，OpenClaw 是 Agent 本體，會自己拆任務、呼叫工具、長時間持續執行的數位助理，另外，OpenShell 是讓 claw 安全運行的底層執行沙箱，定義 agent 能做什麼、不能做什麼，NemoClaw 則是把這兩者整合起來的部署層，它把 OpenClaw 放進 OpenShell，加上安全與隱私控管，串接本地推論能力。

▋NemoClaw 解決的是什麼問題？

OpenClaw 的安全漏洞早已是公開的秘密。它預設會取得磁碟存取、終端機控制、瀏覽器操作與各種 OAuth 授權，對所有網路介面開放，包括公開網際網路。

安全研究人員掃描後發現，全球超過 13.5 萬個 OpenClaw 實例直接暴露在外，其中逾 5 萬個存在遠端程式執行漏洞，它可以控制你整台電腦，卻沒有防禦機制，這是讓許多企業不敢嘗試的主因之一。

NemoClaw 要補的就是這一塊。它透過 OpenShell runtime 將每個 agent 隔離在沙盒環境中，附加隱私路由與以政策為基礎的安全管控，讓 agent 在有權限做事的同時，不會失控存取敏感資料。

▋NemoClaw 的戰略邏輯

另一個重點是，NemoClaw 並不綁定輝達自家的 GPU，硬體上具備相容性，並整合進輝達原有的 NeMo AI agent 軟體套件。目前屬於 alpha 早期版本，官方也坦承「預期會有粗糙之處」。

但輝達產品副總裁 Kari Briski 說：「Claw 是 AI 的新應用層，正在帶動運算需求以數量級成長。」這才是核心。Agent 要 24 小時不間斷運作，就需要專屬的算力，算力的需求，最終會落回 GPU 伺服器上。透過 NemoClaw，輝達把 OpenClaw 的生態系直接接進自己的硬體護城河，從 GeForce RTX 工作站到 DGX Spark 全線受惠。

輝達的最強戰略：基礎設施先行，應用層的勝負留給別人去打。
