AI 編程三個時代的演進：

第一時代：Tab 自動補全
幾年前大部分程式碼還是一個鍵一個鍵敲出來的，Tab 補全開啟了 AI 輔助編碼的第一階段，擅長處理低熵、重複性的工作，持續了將近兩年。

第二時代：同步 Agent
模型能力提升後，Agent 能持有更多上下文、使用更多工具、執行更長的操作序列。開發者從「按 Tab」轉變為「跟 Agent 對話」。2025 年 3 月 Tab 用戶是 Agent 用戶的 2.5 倍，現在反過來了——Agent 用戶是 Tab 的 2 倍，Agent 使用量一年暴增 15 倍。但這個時代可能撐不到一年就會被取代。

第三時代：雲端長時間運行的 Agent（正在到來）
Agent 在雲端獨立的虛擬機上運行，開發者交出任務後就去做別的事。Agent 自己花數小時迭代測試，完成後回傳可快速審查的成果（log、錄影、即時預覽），而不是 diff。這讓「同時跑多個 Agent」變得可行，因為你可以透過產出物快速評估結果，不需要重建每次對話的上下文。人的角色從「引導每一行程式碼」轉變為「定義問題 + 設定審查標準」。

Cursor 內部實踐：
目前 Cursor 內部合併的 PR 中，35% 是由雲端 Agent 自主完成的。採用這種新工作方式的開發者有三個特徵：

1. Agent 寫了幾乎 100% 的程式碼
2. 人花時間在拆解問題、審查產出、給回饋
3. 同時啟動多個 Agent，而不是手把手帶一個跑完

挑戰：在工業規模下，一個 flaky test 或壞掉的環境會中斷每一個 Agent 的運行，還需要確保 Agent 能完整存取所需的工具 and 上下文。

一句話總結：開發者正在從「寫程式碼的人」變成「建造生產軟體的工廠的人」，而工廠裡跑的是一群 Agent 隊友。
