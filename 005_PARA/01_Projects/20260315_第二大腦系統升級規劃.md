# 在 AI 寫作時代打造個人獨特高產出系統 — 第二大腦升級規劃

> 建立日期：2026-03-15
> 狀態：規劃中

---

## 現況診斷

你已經建立了一套架構完整的第二大腦系統（PARA + Zettelkasten + Write Pipeline），擁有 181 張卡片、100+ 篇已發佈文章、15 個 Skills、7 個 Agents、46 個 Prompts。系統的「骨架」已經很強。

**但核心問題是：系統正以 60% 的效能在運作。**

### 三個結構性瓶頸

| 瓶頸 | 現狀 | 影響 |
|------|------|------|
| **索引全空** | 15 個 Index 檔案的 `## Cards` 區塊皆為空 | gather-cards 無法有效運作，每次寫作都要重新人工找卡片 |
| **草稿卡關** | 13 篇草稿中 8 篇無 pipeline_stage、4 篇 DONE 卻未搬到 Publish | 注意力被殭屍草稿佔據，無法聚焦新產出 |
| **缺少回饋迴圈** | 寫完就結束，沒有「哪些觀點被讀者認可」的訊號回流 | 無法系統性放大你的獨特優勢 |

### 你的護城河（AI 時代的不可取代性）

在 AI 時代，**你的護城河不是寫作速度，而是：**

1. **跨領域的獨特連結** — Data Pipeline → 知識管理、系統動力學 → Coding Agent
2. **第一人稱實踐者的判斷力敘事** — 不是轉述，是做過才說
3. **有明確取捨的框架思維** — 不當好好先生，每篇都有立場

## 規劃核心思路

**讓系統幫你更快找到獨特連結、更順暢地從卡片走到發佈、並從結果中學習。**

---

## 方向一：修復基礎設施（讓現有系統跑滿）

> 不加新功能，先讓已建好的東西真正運作起來。

### 行動 1：批量填充 Index 卡片連結

- **現狀**：15 個 Index 檔案的 `## Cards` 全空
- **做法**：寫一個 skill `populate-index`，自動掃描 `010_CardNotes/02_Cards/` 各分類資料夾，將所有卡片以 `[[檔名]]` 格式填入對應 Index
- **效果**：gather-cards 立即能透過 Index 做主題搜尋，寫作前期素材搜集速度 ×3

### 行動 2：自動化 Index 更新（嵌入 break-cards 流程）

- **現狀**：break-cards 建立卡片後，Index 不會自動更新
- **做法**：在 `break-cards` 和 `refine-cards` 的 SKILL.md 末尾加入步驟 — 「卡片確認建立後，自動將 `[[卡片檔名]]` 加入對應 Index 的 `## Cards` 區塊」
- **效果**：Index 永遠保持最新，知識網絡自動成長

### 行動 3：清理殭屍草稿

- **現狀**：13 篇草稿，8 篇無追蹤狀態
- **做法**：逐一檢視 `015_Write/Draft/` 中無 pipeline_stage 的草稿，標記為 `STALLED` 或歸檔。4 篇 DONE 的搬到 Publish
- **效果**：Draft 資料夾只留真正在進行的工作，降低認知負擔

## 方向二：加速「卡片 → 文章」的合成能力

> 你最耗時的不是寫，而是「怎麼把 10 張卡片排列成一個獨特論點」。

### 行動 4：新增 Skill — `synthesis-outline`（合成大綱產生器）

- **定位**：介於 gather-cards 與 write-pipeline Step 4（FRAMEWORK）之間
- **輸入**：gather-cards 產出的 5-15 張卡片 + 用戶的寫作意圖
- **輸出**：3 種不同論證結構的大綱，每種標註：
  - 核心論點（一句話）
  - 卡片排列順序與角色（開場故事 / 支撐論據 / 反例 / 收尾昇華）
  - 這個結構的「獨特角度」是什麼
- **為什麼需要**：目前 write-pipeline 的 Step 2（IDEATION）做的是發散，但缺少「收斂成結構」的自動化步驟。你的文章品質來自結構，不是文字

### 行動 5：新增 Skill — `remix-cards`（跨域重組搜尋）

- **定位**：gather-cards 的進階版，專注「意外連結」
- **做法**：給定一個主題，刻意從**距離最遠的 2-3 個卡片分類**搜尋可能的類比或反例
- **範例**：寫「AI Agent 的瓶頸」→ 從 `01_mental_models`（系統動力學）+ `07_product_management`（PMF）+ `14_domains`（營建業流程）找跨域素材
- **為什麼需要**：你最受歡迎的文章都是跨域洞察（Data Pipeline → 知識管理），但目前系統沒有主動促進這件事

### 行動 6：新增 Skill — `quick-assess`（快速評估器）

- **定位**：Capture 階段的第一道過濾
- **輸入**：一篇文章 URL 或貼上的文字
- **輸出**（30 秒內）：
  - 與你現有卡片的重疊度（高/中/低）
  - 潛在新卡片數量預估
  - 推薦處理優先級（立即拆卡 / 放入 Area / 略讀存檔）
- **為什麼需要**：避免花時間讀完才發現「這篇跟我已有的卡片 80% 重複」

## 方向三：建立回饋迴圈（讓系統越用越聰明）

> 目前是開迴路：寫完就結束。要變成閉迴路：寫完 → 學到什麼有效 → 下次更精準。

### 行動 7：新增 Skill — `review-published`（發佈後回顧）

- **時機**：文章發佈 3-7 天後
- **流程**：
  1. 用戶貼上讀者反饋（留言、轉發數、私訊摘要）
  2. 分析：哪個觀點最被引用？哪段被截圖分享？讀者追問什麼？
  3. 輸出：「讀者共鳴點報告」→ 標記原始卡片為「高共鳴」
  4. 可選：更新 `000_MyContext/capture_profile.md` 的偏好權重
- **為什麼需要**：讓你知道自己的「獨特角度」具體是哪些，下次可以刻意放大

### 行動 8：卡片熱度追蹤

- **做法**：每張卡片加入 `usage_count` 和 `last_used` 欄位
- write-pipeline 每次使用卡片時自動更新
- **效果**：
  - 高頻使用卡片 → 你的核心觀點，值得深化
  - 從未使用卡片 → 可能需要重寫或歸檔
  - 搭配 workspace-stats 呈現「卡片健康度儀表板」

### 行動 9：升級 scout-news — Gap-Filling 模式

- **現狀**：根據 capture_profile.md 的興趣搜尋
- **新增模式**：分析卡片分佈的「空白區」，主動搜尋填補
  - 範例：`06_databricks` 只有 1 張卡片 → 主動搜尋最新 Databricks 文章
  - 範例：`09_software_engineering` 只有 3 張卡片但你列為想學技能 → 提高搜尋權重
- **資料來源**：卡片數量統計 + `000_MyContext/004_0Question/問題清單.md`

## 方向四：放大個人獨特性（AI 時代的護城河）

> AI 人人都能用，你的不可取代性在於：觀點 × 經驗 × 品味。

### 行動 10：新增 Agent — `VoiceDNA`（聲紋分析師）

- **功能**：分析你 100+ 篇文章，萃取出「寫作聲紋」
  - 常用句型結構（例：你偏好「不是 A，而是 B」的轉折句式）
  - 觀點建構模式（例：先講反直覺現象 → 引入框架 → 推導出行動建議）
  - 獨特用詞與比喻習慣
  - 你「不會寫」的東西（負面特徵同樣重要）
- **輸出**：`000_MyContext/voice_dna.md` → 供 write-pipeline 的 POLISH 階段參考
- **為什麼需要**：AI 輔助寫作最大的風險是「寫出來都像 AI」。Voice DNA 讓 AI 輔助的每一步都往你的風格靠攏

### 行動 11：新增 Skill — `contrarian-check`（反共識檢查器）

- **時機**：write-pipeline 的 Checkpoint 2（CP2）之前
- **做法**：
  1. 提取文章的 3 個核心主張
  2. 搜尋網路上是否已有大量類似觀點
  3. 標記：「這個觀點已經是共識」/「這個有獨特角度」/「這個可能有爭議性（好事）」
- **為什麼需要**：AI 時代最大的風險是寫出「正確但不獨特」的文章。這個檢查器幫你確保每篇都有至少一個「只有你會這樣說」的觀點

### 行動 12：活化問題清單機制

- **現狀**：`004_0Question/問題清單.md` 是靜態的 5 個問題
- **改進**：
  - 每次 break-cards 產出新卡片時，自動檢查是否回答了某個問題 → 標記進度
  - 每次 write-pipeline 完成時，提示「這篇文章是否觸發了新問題？」
  - 每月由 workspace-stats 提醒回顧問題清單
- **效果**：問題清單變成活的「研究議程」，驅動閱讀和寫作方向

## 方向五：系列文章規劃能力

### 行動 13：新增 Skill — `series-planner`（系列文章規劃器）

- **輸入**：一個大主題（例：「AI 時代的判斷力」）
- **流程**：
  1. 搜集相關卡片（用 gather-cards + remix-cards）
  2. 規劃 3-5 篇系列文章，每篇有獨立論點但整體有遞進邏輯
  3. 標註每篇需要的卡片缺口（哪些還需要先閱讀/拆卡）
- **為什麼需要**：系列文章是建立讀者認知的最強武器。單篇是點，系列是線

### 行動 14：新增 Prompt — `W9_series-connector`（系列文章串接器）

- **功能**：在系列的第 2 篇以後，自動產生與前篇的「回顧橋段」和「預告鉤子」
- **效果**：讓讀者有動力追完整個系列
