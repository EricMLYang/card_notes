最簡化、最容易擴散的一版流程

---

# 1️⃣ 以「你（提供角色 Repo 的人）」角度

你只需要做 **3 件事**

### ① 做一個 Template Repo

例如：

```
pm-agent-template
```

裡面包含：

```
context/
agent/
scripts/init.sh
README.md
```

然後把 repo 設成 **GitHub Template Repository**

---

### ② 在 context 放「可替換的範本」

例如：

```
context/project.template.md
```

讓別人之後可以改成自己的專案內容。

---

### ③ 提供一個初始化 script

例如：

```
scripts/init.sh
```

功能只有一件事：

> 把 template context 變成 project context

---

# 2️⃣ 以「其他開發者」角度

他們只需要 **3 步**

### ① 用 Template 建立自己的 repo

GitHub：

```
Use this template
```

建立：

```
my-project-pm
```

---

### ② clone repo

```
git clone my-project-pm
cd my-project-pm
```

---

### ③ 執行初始化

```
bash scripts/init.sh
```

然後改自己的

```
context/project.md
```

就可以開始用 Agent。

---

# 3️⃣ 整個擴散流程（超簡版）

```
你
↓
建立 template repo
↓
團隊
↓
Use template 建 repo
↓
run init.sh
↓
改 context
↓
開始用 agent
```

---

# 4️⃣ 為什麼這版最容易擴散

因為使用者只需要記住：

```
Use template
run init.sh
edit context
```

三件事而已。
