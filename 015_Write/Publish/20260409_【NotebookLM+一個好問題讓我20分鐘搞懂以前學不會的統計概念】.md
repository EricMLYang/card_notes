【 NotebookLM + 一個好問題，讓我 20 分鐘搞懂以前學不會的統計概念 】

卷積、MGF、動差生成函數。

這些名詞在課本裡看起來像另一個星球的語言。

但如果你手上剛好有一個業務場景：
廣告播放延遲 = 偵測延遲 + 換圖延遲，
你需要知道這個總延遲會散到什麼程度

突然間，卷積變成了你每天在處理的問題。

我用 NotebookLM 做了一次實驗，
從提問到搞懂只花了 20 分鐘。
以下是過程和心得。


▋ 起因：Genie Code 讓我發現自己少了什麼

最近在用 Databricks Genie Code 做資料分析。
AI 跑查詢、跑模型，速度飛快。

但結果一出來，我卡住了。

分佈長這樣代表什麼？
這個變異數合理嗎？
兩個隨機變數加在一起，結果會怎樣？

這些問題以前唸書時應該學過。
老實說，考完就還給老師了。

我意識到一件事：
AI 能幫你跑完所有計算，但「這個結果代表什麼」的直覺判斷力，它給不了你。
那是你自己的事。


▋ 方法：把教科書丟進 NotebookLM，問一個好問題

以前想重學統計，做法是翻書、Google、對著公式發呆。
一個概念搞半天，還不確定自己有沒有真的懂。

這次用之前 Manny 分享的方法。

把好幾本機率統計教科書丟進 NotebookLM，問了一個問題：

「這個領域的專家共享哪些核心心智模型？」

NotebookLM 整理出一份清單。
其中一個以前比較少去理解的東西跳出來：
生成函數與記帳工具（Generating Functions / Moment Generating Functions）。

然後我用 Gemini 和 ChatGPT 交叉提問、互相驗證。

整個過程不到 20 分鐘。
睡前做的。


▋ 打通的瞬間：公式遇上業務場景

讓我真正通的，是因為手上剛好有一個真實場景可以對接。

在 SmartSignage DataHub 專案裡，廣告播放的「準度」是關鍵商業指標。
準度是指觀眾站在螢幕前那 3 秒，廣告能不能精準切換。

總延遲 Z = 偵測延遲 X + 換圖延遲 Y。

Z 的分佈太散，觀眾走過去了廣告才跳出來。
這就是一次失敗的商業行為。

卷積做的事情，
就是把兩個誤差的規律疊加在一起，算出「剛好在 z 秒播放」的機率。

直覺上，這就是在做加權滑動平均。
AI 幫我圖像化之後，一個關鍵的東西浮出來：

誤差不會互相抵銷，只會越捲越寬。

就算平均值很快，只要分佈的尾巴太長，
就一定有一部分觀眾看到空秒。

MGF 則是另一個實用的工具。

如果要頻繁測試不同 AI 模型和不同網路環境的組合，每次都從頭算卷積太累了。
MGF 把延遲規律打包成代數函數，
兩個系統的 MGF 相乘，就能預測組合後的結果。

不算積分，只做函數乘法。
這讓我可以快速回答一個商業問題：
在特定硬體配置下，廣告播放的準確率會掉多少？


▋ 睡前 20 分鐘的三個體會

回頭看這段學習過程，有三個感受比較深。

第一，AI 幫我最多的地方，是建立連結。
它把抽象的數學概念對接到我手上的業務場景，
讓我秒懂這個公式到底在解什麼問題。
以前要花好幾個小時推導的東西，現在幾分鐘就能模擬出趨勢。

第二，問對問題比背公式管用。
我沒有從第一章開始看。
直接問「專家的核心心智模型是什麼」，
這個問題幫我跳過大部分枝節，命中最值得花時間的概念。

第三，業務經驗是最好的學習加速器。
手上有真實場景的人，學理論會快得多。
因為每個公式都有地方可以接地，不再是懸在空中的符號。
