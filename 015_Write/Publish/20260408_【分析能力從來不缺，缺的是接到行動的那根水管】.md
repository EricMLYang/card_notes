【 分析能力從來不缺，缺的是接到行動的那根水管 】
——從 Dentsu、Salesforce、Shell 等五家企業，看 Agent 在數據分析的三層價值定位
ㅤ
幾年前跟教授做產學合作的計量分析，模型跑了好幾輪，變數調了又調，回歸結果終於漂亮了。
ㅤ
拿去給業務主管看，對方盯著螢幕看了三秒，問了一句：
「所以我下週一的會議該講什麼？」
ㅤ
那一刻我才意識到，分析能力從來不是瓶頸。
瓶頸在後面那段——分析結果跟業務行動之間那條看不見的鴻溝。
ㅤ
最近用 Databricks Genie 做了一輪數據分析，體驗很流暢。
AI 讓分析變快了、門檻變低了、產出變好看了。
但「所以我該怎麼做」這個問題，依然沒有消失。
ㅤ
我花了一段時間拆解 Dentsu、causaLens、Salesforce、S&P Global、Shell 這五家企業怎麼把 Agent 用在數據分析上，整理出一個判斷：
ㅤ
Agent 真正在解決的，是三種完全不同層次的問題。
搞清楚這三層，才知道自己該從哪裡切入。
ㅤ
ㅤ
▋ Agent 真正被買單的，是交付速度
ㅤ
先說一個觀察。
ㅤ
目前市場上 Agent 在數據分析領域最被買單的成果，幾乎都跟「分析本身的品質」無關。
ㅤ
企業公開強調的指標，大多是 analysis time、time-to-insight、比較分析速度、resolution rate、ROAS、工時節省。
這些指標的共同點——它們都在衡量交付與執行的效率。
ㅤ
我把這件事分成三層來看。
ㅤ
ㅤ
▋ 第一層：讓分析更快被拿到
ㅤ
這是目前做得最多、也最容易產生可見價值的一層。
ㅤ
Dentsu 是全球行銷與媒體服務集團。
他們用 Azure AI Foundry 做了一個 predictive analytics copilot，讓 media insights 更容易被員工和客戶取得。
ㅤ
做的事情看起來不複雜，但成果很具體——analysis time 減少 80%，time-to-insight 減少 90%。
ㅤ
S&P Global 也是類似的邏輯。
他們把能源與商品資料接上 Microsoft 365 Copilot，讓客戶用自然語言查詢高價值市場數據。
data extraction 快了 95%，comparative analysis 快了 98%。
ㅤ
這兩家的共通點：底層的分析能力本來就在。
資料有了、模型跑了、報表也出了。
Agent 改變的是分析結果的「交付介面」。
ㅤ
我自己的理解是這樣——用一個比喻來說，這一層做的事情就是「接水管」。
水源和淨水廠都在了，但水龍頭離用戶太遠。
Agent 做的是把水龍頭搬到用戶的桌上。
ㅤ
如果你帶一個 5-10 人的團隊，正在考慮 Agent 怎麼用在數據分析，這一層幾乎一定最適合先切入。
ㅤ
原因很直接：它解決一個非常常見的痛點——分析結果明明存在，但取得太慢、太靠專家、太難被業務端消費。
這是最快能讓老闆看到成果的切入點。
ㅤ
ㅤ
▋ 第二層：讓分析本身更深
ㅤ
第一層解決的是「拿得到」。
第二層要處理的，是分析引擎本身能不能再往前推一步。
ㅤ
causaLens 是做 Causal AI 和 decision intelligence 的公司。
它的主軸是以 causal reasoning 為基礎的 digital workers，強調在金融服務等高決策價值場景的可用性。
ㅤ
這家公司做的，更接近「進階分析引擎 + Agent 調用」的方向。
ㅤ
傳統 BI 回答的是「發生了什麼」。
進階一點可以回答「會發生什麼」。
但 causaLens 想走到的是「為什麼發生」和「改什麼最有用」——也就是因果推論加上指導性分析。
ㅤ
說實話，這是我最想達到的分析理想。
ㅤ
但回頭來看，它跟第一層走的路完全不同。
因果推論需要結構化資料、明確的因果設計、完整的評估機制、高品質的業務建模。
ㅤ
一個簡單的區分方式：
MLOps 管的是預測模型的部署——回答「會發生什麼」。
DecisionOps 管的是決策模型的部署——回答「該做什麼」。
這兩條路的工程差異很大。
ㅤ
對大多數帶小團隊的技術主管來說，這一層值得知道，但不急著做。
除非你的業務場景天生需要因果推論——像是定價策略、行銷投放歸因、醫療決策——否則投資報酬率不見得最高。
ㅤ
ㅤ
▋ 第三層：讓分析驅動行動
ㅤ
第一層讓你拿得到分析結果。
第二層讓分析更深。
第三層要做到的是：分析結果真的去推動一件事情發生。
ㅤ
Salesforce 是最典型的代表。
他們用 Agentforce + Data 360，把數據、分群、預測、規則，直接接進 sales、marketing、service 流程。
ㅤ
成果已經跳脫效率指標，直接是商業產出：
ㅤ
lead revenue 和 contract value per lead 增加 60%
resolution rate 達到 85%
projected 減少 98,000 個 support cases
透過 AI-powered Slack notifications 發現 2,300 萬美元潛在續約收入機會
ㅤ
另一邊，Shell 展示的是工業端的落地。
AI 用在管線腐蝕模擬，比傳統 CFD 方法快 106 倍。
CCS storage site assessment 流程加速超過 60%。
ㅤ
這告訴你一件事：第三層的應用場域很寬。
前台的 CRM 和後台的工程評估、監控、模擬，都是 Agent 可以接上行動的地方。
ㅤ
我目前的判斷是，如果系統最後要有真正的商業影響，終局通常會走到這裡。
光是有更好的分析介面還不夠，分析結果要能推動 outreach、resolution、campaign，或者工程評估流程。
ㅤ
但老實說，大多數 AI 應用目前還停在建議層——能說、會分析、給建議，但沒有真的承擔後果。
要從「建議」走到「執行」甚至「代理」，需要補齊的是工具調用、狀態記憶、驗證機制和回饋迴圈。
ㅤ
ㅤ
▋ 隱藏的死穴：資料治理
ㅤ
講完三層，有一件事必須直說。
ㅤ
Agent 毫無資料清洗能力。
ㅤ
這聽起來像廢話，但很多團隊在規劃 Agent 導入的時候，潛意識裡會有一個假設：AI 夠聰明，應該能自動處理髒數據、統一異質系統。
ㅤ
實際做下去，完全不是那麼回事。
ㅤ
若缺乏 Data Catalog、中繼資料管理和統一的業務邏輯定義，Agent 強大的自然語言生成能力只會做一件事——加速產出具高度說服力的錯誤結論。
ㅤ
這就是「數據幻覺」。
比模型幻覺更麻煩，因為它格式正確、邏輯通順、用了真實的表名和欄位名，但算出來的數字根本不對。
ㅤ
有一個觀察讓我印象很深：多數 Agent 在正式環境失敗，問題出在 context 管理、memory 設計、錯誤處理、評估回饋機制。
「大多數創辦人以為自己在做 AI 產品，其實是在做 context selection 系統。」
ㅤ
Agent 跨越了人機互動的最後一哩路。
但底層資料工程的第一哩路沒搞好，那條水管接上去只會流出汙水。
ㅤ
嚴格的資料治理，是 Agent 上線的前置條件。
ㅤ
ㅤ
▋ 我目前的判斷
ㅤ
回到帶小團隊的視角。
三層能力不是升級路線，是三種不同的投資方向。
ㅤ
先做第一層。
把現有分析能力的交付介面做好。
用 Semantic Layer、NL-to-query、RAG 讓業務端可以直接取用分析結果。
這最快能讓老闆看到成果。
ㅤ
同步投資資料治理。
不需要一步到位，但至少要有 Data Catalog、統一的業務邏輯定義、基本的資料品質監控。
否則 Agent 越強，錯誤越有說服力。
ㅤ
心裡記住終局是第三層。
分析結果最後要能接到行動。
推動 outreach、resolution、campaign、opportunity follow-up，或工程評估流程。
這才是真正產生商業影響的地方。
ㅤ
至於第二層？
如果你的場景天生需要因果推論，值得投資。
但對大多數小團隊來說，這不會是第一步。
ㅤ
ㅤ
▋ 分析成本降低之後
ㅤ
最後多說一件事。
ㅤ
Agent 正在讓「分析交付」的成本大幅降低。
過去需要分析師花一週整理的報告，現在可能 10 分鐘就有了。
ㅤ
但價值沒有消失，它往兩個方向轉移了。
ㅤ
往上游——資料治理、語意統一、Metadata 管理變得更值錢。
因為 Agent 放大了好資料的價值，也放大了壞資料的傷害。
ㅤ
往下游——行動執行、流程接口、系統整合變得更值錢。
因為分析做完只是開始，商業影響在行動端。
ㅤ
如果你問我 Agent 時代什麼能力最值得投資？
ㅤ
我目前的看法是：讓資料更乾淨，讓行動更順暢。
ㅤ
分析這件事本身，正在快速變便宜。
真正稀缺的，是分析結果兩端的基礎設施——上游的資料品質，和下游接到行動的那根水管。
