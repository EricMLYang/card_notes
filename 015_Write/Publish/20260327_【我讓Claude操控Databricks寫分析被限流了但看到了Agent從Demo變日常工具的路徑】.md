【 我讓 Claude 操控 Databricks 寫分析 Genie Code，被限流了 】
——但看到了 Agent 從 Demo 變日常工具的路徑
ㅤ
ㅤ
這幾天我做了一件有點中二的事：讓 Claude 的 Computer Use 直接操控我的電腦，然後連進 Databricks Genie Code，讓 Agent 去操作另一個 Agent。
ㅤ
它成功寫出了一個分析範例。
ㅤ
Agent 操控 Agent，聽起來像科幻，但畫面就在我的螢幕上。
ㅤ
然後我的 $20 月費訂閱就被限流了。
ㅤ
這個「成功但馬上撞牆」的體驗，反而讓我想認真看一件事：地端 Agent 現在的問題到底卡在哪？
ㅤ
我翻了最近蒐集的十幾張筆記卡片，發現答案比想像中清晰。
ㅤ
模型夠聰明了，操控技術也到了。
卡住的是三條趨勢線還沒交匯到臨界點而已。
ㅤ
ㅤ
▋ Token 成本下降——整個互動架構在省
ㅤ
大部分人聽到 Token 成本下降，直覺想到的是模型定價在降。
ㅤ
實際上變化比這深。
ㅤ
Google 的 WebMCP 讓網頁主動告訴 Agent 可以做什麼，Agent 不用再「看」畫面猜操作。
Token 消耗直接少了 89%，準確率拉到 98%。
ㅤ
agent-browser 用 CLI snapshot 只吃 200-400 tokens，對比 Playwright MCP 的 13,700+。
Vercel 團隊的 2-tool 極簡架構達到 100% 成功率，贏過 17-tool 框架的 80%。
ㅤ
工具少、Token 省、成功率還更高。
ㅤ
底層也在變，

Agentic workloads 超過 95% 在重複使用 KV-Cache，DeepSeek 的 DualPath 方案加速近 2 倍。
ㅤ
整個成本結構正在被重新定義。
ㅤ
ㅤ
▋ Agent 操控能力——從瀏覽器到整個桌面
ㅤ
我操控 Databricks 那次體驗印證了一件事：Agent 正在突破瀏覽器的邊界。
ㅤ
agent-browser 已經能透過 CDP 直接連 Electron 桌面應用——Slack、VS Code 都能操作了。
ㅤ
以前 Agent 活在瀏覽器頁籤裡，現在活動範圍正在變成整個電腦桌面。
ㅤ
更大的結構性變化是 AEO 正在取代 SEO：應用程式設計從「呈現給人看」轉向「讓 Agent 理解和操作」。
ㅤ
Cursor Agent 用量年增 15 倍，35% 的 PR 已經由 Agent 生成。
開發者從「寫 code」正在轉變成「管理 Agent 團隊」。
ㅤ
ㅤ
▋ 讓 Agent 可靠的是系統設計
ㅤ
這是我覺得最反直覺的一條。
ㅤ
三種控制 Agent 行為的方式比較：
ㅤ
長 prompt 可靠度 82.5%。
Workflow graph 80.8%。
Steering Hooks（在執行節點放運行時驗證鉤子），100%。
ㅤ
在對的時間點做對的檢查，比寫更長的 prompt 更有效。
ㅤ
核心觀念是把 LLM 決策（機率性的部分）和狀態轉換（確定性的部分）分開處理。
讓 AI 做它擅長的判斷，讓系統處理流程控制。
ㅤ
我測 Claude 操控 Databricks 時深刻感受到這件事——Agent 夠聰明，操控也成功了，但那個「不太穩」的感覺，來自缺乏系統級的檢查和恢復機制。
ㅤ
要解決的是讓圍繞模型的系統更可靠。
ㅤ
ㅤ
▋ 三條線的交匯
ㅤ
Anthropic ARR 15 個月翻 20 倍到 $20B，需求是真的。
ㅤ
但 40% 企業目標整合 Agent，只有 2% 規模化部署。
Gartner 預測 40%+ 計畫會取消。
ㅤ
被砍的幾乎都是只看模型能力、沒做好系統設計的專案。
這恰恰印證了第三條趨勢線。
ㅤ
三條線各自看，可能只是某個領域的進步。
但它們同時在發生，而且都在加速。
ㅤ
交匯，就是臨界點。
ㅤ
ㅤ
我讓 Claude 操控 Databricks，成功了又被限流。
ㅤ
如果只看今天，確實是個有趣但不實用的 Demo。
ㅤ
但把視角拉長，看三條線的斜率：Token 架構在省、桌面操控在擴展、系統設計在成熟——會看到一個正在加速靠近的交匯點。
ㅤ
今天被限流的體驗，和明天 Agent 變成日常工具的未來之間，不是技術鴻溝。
是時間問題。
ㅤ
而且這個時間，可能比大部分人想的還短。
