【 Genie Code 讓我想重新補回數據理論 】
—— 因為我預感判斷 Agent 的分析結果又要變成了主要工作
ㅤ
Genie Code 推出後，我這幾天做了一個很認真的決定：要把之前的數據分析理論補回來。
ㅤ
不是因為懷舊，而是因為我意識到，接下來我的主要工作要回到評斷 Agent 產出的分析結果。
ㅤ
對，體驗 Genie Code 帶給我的改變就是這麼大。
ㅤ
ㅤ
▋ 我的角色轉了三次
ㅤ
回頭看，我在「寫 Code」和「判斷結果」之間來回了好幾趟。
ㅤ
第一階段：自己做分析。
早期我是純數據分析的工程師，機器學習、深度學習、數據分析都自己來，讀了不少理論，也做了大量實作。
ㅤ
第二階段：變成出一張嘴的人。
當 Project Leader 後，逐漸不太自己寫 Code 了。主要工作變成 judge 同事的分析結果。
ㅤ
說實話，我一直覺得數據分析的流程比較像設計思維，

你要靠靈感或直覺來導航，再用理論和工具去驗證。

但我猜那時候同事大概覺得我們只是出一張嘴，沒有實際參與分析過程。
ㅤ
第三階段：又開始寫 Code，但一個人扛太多。
ChatGPT 問世後我又碰回 Code，到現在 Coding Agent 逐漸成熟，我身兼 PM 和數據工程師。

小團隊要兼顧好幾個專案，說實話有點滿，我自己對產品成果沒有滿意過。
ㅤ
經典的實力跟不上品味。
ㅤ
ㅤ
▋ Genie Code 帶回了「協作感」
ㅤ
但 Genie Code 出現後，我突然找回了以前跟同事「協作」的感覺。
ㅤ
差別在於，這次不管我提出的方向多天馬行空，Agent 都會想辦法嘗試去達成。

不會先跟你爭論方向對不對，而是直接試了再說。
ㅤ
這反而讓我意識到一件事：當執行變便宜了，判斷品質就變成唯一的瓶頸。
ㅤ
如果我沒辦法判斷 Agent 跑出來的分析結果對不對、好不好、有沒有漏掉什麼，那 Agent 再快也沒用。
ㅤ
所以我需要把之前的數據理論補回來。不是為了自己寫分析，而是為了能精準地評估 Agent 做出來的東西。
ㅤ
ㅤ
▋ 我怎麼補：把書丟進 NotebookLM，模仿那個 MIT 研究生
ㅤ
為了快速複習，我把之前收集的大量數據相關書籍，按主題分別丟進 NotebookLM，一口氣建了超過 10 個。
ㅤ
靈感來自 Manny 分享的一篇文章：一個 MIT 研究生用 NotebookLM，在 48 小時內通過了一個他從沒碰過的學科的資格考。
ㅤ
方法是這樣的——他把六本教科書、十五篇論文、所有課堂逐字稿全部丟進去，然後問了三個問題：
ㅤ
→ 這個領域的專家共享哪些核心心智模型？
→ 專家之間根本性的分歧在哪裡，各方最強的論點是什麼？
→ 給我十個問題，它們要能區分真正理解和死背的人。
ㅤ
接著用那十道題自測，每答錯一題就追問「為什麼錯了、我漏掉了什麼」。
ㅤ
作者的結論是：學習的差距不在內容量，而在你問的問題品質。
ㅤ
ㅤ
▋ 我的體會
ㅤ
我沒有要 48 小時通過什麼考試。但這個方法確實幫我快速重建判斷框架。
ㅤ
以前學理論是為了自己做分析。
現在學理論是為了判斷 Agent 的分析結果夠不夠好。
ㅤ
目的不同，學法也不同——我不需要記住每個公式，但我需要知道什麼時候該懷疑、什麼時候該追問。
ㅤ
回頭來看，這整個過程其實就是一句話：
ㅤ
當 AI 把執行成本壓到趨近於零，你的判斷力就是你僅剩的槓桿。
ㅤ
而判斷力不會從天上掉下來。它來自你願不願意把那些基礎的東西，老老實實地補回來。
