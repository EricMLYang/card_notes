## 原始素材

因為我們 Team 同仁有養一隻龍蝦在公司的 AWS EC2 上測試對外抓資訊，

這幾天也手癢嘗試 Claude 的 Cowork + Computer Use + Claude in Chrome 來讓 Agent 操控電腦和瀏覽器，

配合 Dispatch 用手機來發指令，

因為最近很愛 Databricks 的 Genie Code 愛到沒事就掛著 Databricks 畫面想到就跟他互動一下，

因此還測試讓 Claude 在電腦 UI 直接操作 Databricks Genie Code，

他有成功寫出一個分析範例，

頗有 Agent 操控 Agent 的味道，


雖然我的 20美 訂閱每次用 Cowork 沒幾下就被限流了 (因為公司有發無限用的 Github Copilot，Claude 沒訂 Max 純粹想保持跟最強的 Coding Agent 接觸)，

但這幾天是下來還是讓我腦洞大開，



在之前用經濟系的趨勢分析觀念來看，

雖然現在地端 Agent 因為太耗 Token 顯得不太靈活，

但未來感覺潛力無限 ( 讓我又想買蘋果股票了)，


用幾個重要變數的長期趨勢來看，

- Token 成本持續下降

- Agent 操作瀏覽器、電腦、程式 ... 等便利會提高，因為大家都會為 Agent 調整目前讓人用的機制

- 模型聰明能力和執行任務穩定性穩定增強


這樣的交匯點實在讓人無法不期待 Agent 合乎成本的靈活操控電腦，


---

以下是從最近筆記中搜集到的補強素材，可以支撐上面三個趨勢論點：


## 補強一：Token 成本持續下降 — 不只是模型便宜，是整個互動架構在省

Token 成本下降不只是模型定價在降，而是整個 Agent-Web 互動架構正在重新設計：

- **WebMCP 的衝擊**：Google 推出的 WebMCP 讓網頁主動暴露結構化工具定義給 Agent，直接減少 token 消耗 89%、運算量 67%，準確率提高到 98%。這代表未來 Agent 不需要「看」網頁再猜怎麼操作，而是網站直接告訴 Agent 可以做什麼 → 參考 [[20260228_Google真要統治世界_當網頁主動獻身給Agent 時_WebMCP來了]]

- **agent-browser 的 Token 效率革命**：agent-browser 用 CLI snapshot 只消耗 200-400 tokens，對比 Playwright MCP 的 13,700+ tokens，而且 Vercel 的 2-tool 極簡架構達到 100% 成功率，打敗 17-tool 框架的 80%。工具少、Token 省、成功率還更高 → 參考 [[20260313_兩個月21000星_agent-browser]]

- **KV-Cache 讓多輪對話成本結構改變**：Agentic workloads 有 >95% 是 KV-Cache 重複使用，每次新增的 token 很少，瓶頸從 GPU 運算轉移到儲存 I/O。DeepSeek 的 DualPath 方案實測加速 1.87-1.96 倍 → 參考 [[20260301_LLM運算瓶頸將從GPU轉移到儲存IO]]


## 補強二：Agent 操控能力正在急速提升 — 從瀏覽器到整個桌面

- **agent-browser 控制 Electron 桌面應用**：透過 Chrome DevTools Protocol (CDP) 直接連接 Electron app，可以自動化操作 Slack、VS Code 等桌面應用。這跟我測試 Claude Computer Use 控制 Databricks 的體驗一致 — Agent 正在突破「只能操控瀏覽器」的限制 → 參考 [[20260307_agent-browser控制桌面Electron]]

- **AEO 取代 SEO 的時代**：未來網頁開發會從「怎麼呈現給人看」轉向「怎麼讓 Agent 理解和操作」。沒有針對 Agent 優化的網站會被排除在自動化工作流之外 → 參考 [[20260302_Agent產品化地圖]]

- **Cursor Agent 使用量爆炸 15 倍**：Agent 用量年增 15 倍，Agent 操作數已經是 Tab 補全的 2 倍多。35% 的 Cursor PR 是 Agent 生成的，開發者從「寫 code」轉變成「管理 Agent 團隊」的角色 → 參考 [[20260302_CursorCEO捕捉AI改變軟體開發格局的趨勢]]


## 補強三：模型穩定性和控制機制 — 不只靠模型聰明，靠系統設計

- **Steering Hooks 比長 Prompt 可靠**：在執行節點放運行時驗證鉤子，可靠度 100%，遠勝長 prompt 的 82.5% 和 workflow graph 的 80.8%。Agent 的穩定性不只靠模型更聰明，更靠「在對的時間點做對的檢查」 → 參考 [[20260321_agent控制流程不能只靠長prompt]]

- **確定性與機率性的分離設計**：把 LLM 決策（機率性）和狀態轉換（確定性）分開，可以大幅提升可靠度。競爭優勢在於可觀察、可恢復、安全的系統設計，而不是更聰明的模型 → 參考 [[20260316_AI_Agent生產架構_工具記憶評估護欄]]

- **用 Coding Agent 打造 Agent**：生產型 Agent 面對的挑戰跟互動式使用完全不同 — 結構化 I/O、事件驅動觸發、不同的緊急程度。這正是我在測試 Agent 操作 Agent（Claude 控制 Databricks Genie Code）時深刻感受到的複雜度 → 參考 [[20260320_用codingAgent打造Agent]]


## 補強四：市場面現實 — 差距就是機會

- **Anthropic 營收爆發**：ARR 從 $1B → $9B → $20B，15 個月翻 20 倍；其中 Claude Code 貢獻超過 $2.5B → 參考 [[20260309_Claude與Anthropic趨勢整理]]

- **企業部署差距巨大**：40% 企業 app 目標整合 Agent，但只有 2% 達到規模化部署。Gartner 預測 2027 年前有 40%+ 計畫會被取消 — 真正卡住的不是技術能力，而是治理、權限、合規、整合 → 參考 [[20260228_Kore.ai_AI Agents 2026 從炒作到企業現實]]

- **結構化流程勝過原始能力**：有紀律的結構化工作流（brainstorm → plan → TDD execute）效果比模型原始能力更好。chardet 7.0.0 一人用 Agent 改寫，達到 41 倍加速 → 參考 [[20260316_Superpowers_AI_Coding_Agent結構化工作流]]

回到我最初的觀察：地端 Agent 現在「太耗 Token 顯得不靈活」，但 Token 省法正在系統架構層面被解決（WebMCP、agent-browser CLI、KV-Cache），控制能力正在從瀏覽器擴展到整個桌面，而穩定性靠的是系統設計而非純粹模型能力的提升。

這三條線的交匯，就是 Agent 從「有趣的 Demo」變成「可靠的日常工具」的臨界點。

## 發想筆記（觀點深化）

### 二階思考：Agent 成本過臨界點後會發生什麼？

- **一階**：Agent 成本下降 → 更多人用
- **二階**：更多人用 → 網頁/應用必須為 Agent 優化（AEO 取代 SEO）→ Agent 更好用 → 正向飛輪啟動
- **三階**：Agent 好用到一定程度 → 技術團隊的「人力單位」定義改變，從「幾個工程師」變成「幾個工程師 + 幾個 Agent」→ 團隊規模指標要重新定義
- **對技術主管的啟示**：現在不準備 Agent 治理框架和系統設計能力，等成本過臨界點時會措手不及

### 逆向思考：如果 Agent 導入最終失敗，原因是什麼？

1. 治理框架沒準備好（Gartner 40%+ 取消印證）
2. 把 Agent 當成「更快的人」而非「不同類型的隊員」
3. 只看模型能力，忽略系統設計（確定性 vs 機率性分離）

→ 這些失敗原因反過來就是「現在該準備什麼」的路線圖

**Hook 1（搭配標題 1：三條趨勢線交匯）**

因為團隊同仁在 EC2 上養了一隻爬蟲對外抓資訊，我這幾天也手癢，嘗試讓 Claude 的 Computer Use 直接操控電腦跟瀏覽器。

模型夠聰明了，操控技術也到了。
卡住的是三條趨勢線還沒交匯到臨界點而已。


▋ Token 成本下降——整個互動架構在省

大部分人聽到 Token 成本下降，直覺想到的是模型定價在降。

實際上變化比這深。

Google 的 WebMCP 讓網頁主動告訴 Agent 可以做什麼，Agent 不用再「看」畫面猜操作。
Token 消耗直接少了 89%，準確率拉到 98%。

agent-browser 用 CLI snapshot 只吃 200-400 tokens，對比 Playwright MCP 的 13,700+。
Vercel 團隊的 2-tool 極簡架構達到 100% 成功率，贏過 17-tool 框架的 80%。
工具少、Token 省、成功率還更高。

底層也在變——Agentic workloads 超過 95% 在重複使用 KV-Cache，DeepSeek 的 DualPath 方案加速近 2 倍。

整個成本結構正在被重新定義。


▋ Agent 操控能力——從瀏覽器到整個桌面

我操控 Databricks 那次體驗印證了一件事：Agent 正在突破瀏覽器的邊界。

agent-browser 已經能透過 CDP 直接連 Electron 桌面應用——Slack、VS Code 都能操作了。
以前 Agent 活在瀏覽器頁籤裡，現在活動範圍正在變成整個電腦桌面。

更大的結構性變化是 AEO 正在取代 SEO：應用程式設計從「呈現給人看」轉向「讓 Agent 理解和操作」。

Cursor Agent 用量年增 15 倍，35% 的 PR 已經由 Agent 生成。
開發者從「寫 code」正在轉變成「管理 Agent 團隊」。


▋ 讓 Agent 可靠的是系統設計

這是我覺得最反直覺的一條。

三種控制 Agent 行為的方式比較：
長 prompt 可靠度 82.5%。
Workflow graph 80.8%。
Steering Hooks（在執行節點放運行時驗證鉤子），100%。

在對的時間點做對的檢查，比寫更長的 prompt 更有效。

核心觀念是把 LLM 決策（機率性的部分）和狀態轉換（確定性的部分）分開處理。
讓 AI 做它擅長的判斷，讓系統處理流程控制。

我測 Claude 操控 Databricks 時深刻感受到這件事——Agent 夠聰明，操控也成功了，但那個「不太穩」的感覺，來自缺乏系統級的檢查和恢復機制。

要解決的是讓圍繞模型的系統更可靠。


▋ 三條線的交匯

Anthropic ARR 15 個月翻 20 倍到 $20B，需求是真的。
但 40% 企業目標整合 Agent，只有 2% 規模化部署。
Gartner 預測 40%+ 計畫會取消。

被砍的幾乎都是只看模型能力、沒做好系統設計的專案。
這恰恰印證了第三條趨勢線。

三條線各自看，可能只是某個領域的進步。
但它們同時在發生，而且都在加速。

交匯，就是臨界點。


我讓 Claude 操控 Databricks，成功了又被限流。

如果只看今天，確實是個有趣但不實用的 Demo。

但把視角拉長，看三條線的斜率——Token 架構在省、桌面操控在擴展、系統設計在成熟——會看到一個正在加速靠近的交匯點。

今天被限流的體驗，和明天 Agent 變成日常工具的未來之間，不是技術鴻溝。
是時間問題。
而且這個時間，可能比大部分人想的還短。
