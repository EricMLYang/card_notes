# 【 第二大腦的過濾美學：別讓卡片盒變成資訊垃圾場 】

(隨筆心得)

這幾天密集整理了這個 Repo 的目錄結構，
從原本混亂的 Capture，
一路重構到現在清晰的 PARA Areas。
看著那些被我刪除的「過時新聞」與「空標題檔案」，
我突然意識到一件事：
個人知識管理最危險的動作，
就是「太輕易地讓資訊進入核心」。

以前我的流程很直接：
看到好文章 -> Capture -> 拆解成卡片 -> 進入卡片盒。
聽起來很勤勞，
但久而久之發現卡片盒變得非常擁擠，
裡面充斥著各種「暫時覺得重要」但其實缺乏生命力的資訊。
卡片數量上去了，
但知識的核心卻被稀釋了。

經過這幾次的 Refactor，
我開始把流程落實得更「順」但也更「慎重」：

1. **強化緩衝層 (The Staging Area)**：
現在資訊會先經過 `enrich-capture`，
自動分類到 005_PARA 的各個 Area Inbox。
這一步是「找重點」的階段（Process Level 1），
它像是一個過濾網，
讓內容在裡面「飛一會兒」。

2. **在 Inbox 進行初步萃取**：
檔案進入 Area Inbox 後，
我會先填寫標頭的 [重點]、[摘要] 與 [詮釋]。
這個過程其實是在測試：
「這篇文章真的值得我花時間轉化成永久筆記嗎？」
如果連摘要都寫得索然無味，
那它就沒有資格進入我的卡片盒。

3. **卡片盒是「萃取後的表達」**：
只有經過深度詮釋、
能跟其他知識產生化學反應的內容，
才會最終被 `break-cards` 拆解成原子化卡片。
這讓卡片盒從「資訊儲存庫」變成了「思考發源地」。

現在回頭看，
一個順手的第二大腦系統，
目的不是為了讓我們「存得更多」，
而是為了讓我們「留得更精」。
別讓勤勞的搜集，
掩蓋了思考的懶惰。
好的卡片盒，
應該是越用越輕盈，而不是越用越沉重。
