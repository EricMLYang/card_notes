# SQL 要學到什麼程度才夠？大致上可以分幾種角色來看：

reference: <https://www.threads.net/@adamchang40>

1. 如果工作中是負責營運，通常只需要撈業務報表，熟悉基本的 SQL 語法如 Select, From, Join, Where, Group by, Order by等，就非常夠用了。可以的話，再多了解 SQL 語法的執行順序更好。

2. 如果是數據分析師或數據科學家，會需要熟悉 CTE, Union, Window Function, Subquery 等組合操作，用於複雜的數據處理、特徵工程、實驗或用戶分析。場景上要能實現較複雜的分析。

3. 如果是資料工程師，SQL 就會需要點好點滿了。除了上述的語法，還得熟悉各種 CRUD 、實現多種 Data Modeling、效能調教、建立索引、建立自定函數、 View、各種語法的『組合』運用等，SQL 的進階語法都需要熟悉，甚至得知道不同資料庫的差異和適合的場景。
   SQL 的學習坦白說也有些學無止境，永遠有新的寫法可以學。工作上，大多數只要學到能解決手邊業務的需求即可。但若是面試，最近幾年好像有越考越難的傾向，會需要面試者，針對複雜的分析考題去額外刷題做準備。
