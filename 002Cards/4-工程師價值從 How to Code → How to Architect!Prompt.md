# 4-工程師價值從 How to Code → How to Architect/Prompt

**類型**：Model

## 概念

AI 時代的工程師需要立刻累積三種新價值：(1) **定義問題的精準度**：精準拆解商業邏輯，讓 AI 聽懂需求；(2) **系統架構的廣度**：當 AI 搞定模組實作後，設計高可用、高擴展的雲端架構來承載服務；(3) **整合 AI 的深度**：將強大的模型能力，轉化為客戶真正需要的商業價值。單純的「程式碼產出」不再是核心競爭力（AI 寫得比你快且不需休息）。如果你的價值僅是「把 spec 翻譯成 code」，就該焦慮；但若能從 Problem Solver 角度出發，這是最好的時代。

## 重要性

這是工程師職涯轉型的行動指南——明確指出該累積哪方面的能力。

## 邊界/反例

並非所有工程師都需要轉型為架構師。在某些場景（如嵌入式系統、底層優化、安全審計），深度的技術細節仍然重要（AI 還做不好）。但對於主流的 Web/App 開發，價值確實在往「問題定義 + 架構設計 + AI 整合」轉移。此外，過度依賴 AI 而不理解底層原理也很危險（回到「認知範圍決定協作上限」）。
