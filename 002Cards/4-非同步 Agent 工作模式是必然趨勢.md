# 4-非同步 Agent 工作模式是必然趨勢

**類型**：Principle

## 概念

A16Z 觀點：非同步不是趨勢，而是**必然的互動方式**。原因：AI 幻覺無法完全避免，需要人類互動確認，但人的專注時間無法 scale。隨著模型能力增強、回應時間變長、可獨立處理的任務變大，Agent 互動模式必然往非同步發展。工作流程變化：(1) 在 Azure DevOps 建立 task；(2) Webhook 觸發 Agent 接手，在 Sandbox 改 code、跑 test、發 PR，花費 15-30 分鐘；(3) 多個 issue 同時讓多個 Agent 並行執行；(4) 隔天上班逐條確認 merge。**關鍵不是 IDE + Agent，而是協作平台（如 GitHub、Azure DevOps）+ Agent 的組合**——唯有非同步協作能力，才「養」得起多個 Agent 不眠不休工作。

## 重要性

這是判斷「你的工作流程能不能 scale」的核心——同步協作無法 scale，非同步可以。

## 邊界/反例

互動式工具仍有價值，但會轉移到前段（修飾需求文件）和後段（修正小 bug 收尾）；核心工作往非同步移動。
