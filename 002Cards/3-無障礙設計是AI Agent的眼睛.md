# 無障礙設計是 AI Agent 的眼睛

- **類型**：Principle
- **概念**：無障礙（Accessibility）設計是讓 AI Agent 也能操作應用程式的關鍵介面。AI 是純軟體，沒有真正的視覺和觸覺，因此為視障、聽障、行動障礙者設計的輔助功能，正好成為 AI 理解應用程式狀態的最佳管道。具體案例：Playwright MCP 在將 HTML 精簡為 YAML 時，會參考網頁的 Accessibility 標記——只要網頁遵循 Accessibility 設計，AI Agent 就能精準操作。如同 API 有設計要點讓 AI 理解，UI 的「AI 可讀性」就是無障礙設計。當每個用戶同時運用 1-5 個 AI Agent 時，無障礙設計將成為主流需求。
- **重要性**：這是 UI 設計的典範轉移——無障礙不再只是「善事」，而是「讓 AI 能用你的產品」的商業必需。
- **邊界/反例**：不是所有 UI 都需要 AI 操作——純娛樂、高互動性應用可能例外。且目前 Computer Use 類技術仍有效能瓶頸，尚未達 100% 可用。
