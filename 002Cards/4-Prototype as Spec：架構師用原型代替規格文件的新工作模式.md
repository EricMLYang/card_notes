# 4-Prototype as Spec：架構師用原型代替規格文件的新工作模式

**類型**：Model

## 概念

安德魯從 Kent Beck 的話出發：「AI 讓我 90% 技能價值變 $0，但剩下 10% 放大 1000x」。架構師如何成為那 10%？答案是**將 Prototype 當作 Spec**。架構任務簡化為兩階段：(1) 處理不確定性問題；(2) 執行確定性任務。中間產物是明確的 SPEC。善用 vibe coding 做 Prototype 來驗證收斂 SPEC——Prototype 敘述了行為（可直接 RUN）、設計（有實際 API/schema）、整合（多專案需求可用 Prototype 呼叫 API 驗證）。優勢：SPEC 異動可靠重構工具改對、可靠測試工具驗證、有具體畫面操作可跟 PO/RD 對焦。最終瓶頸變成：你能多快構思系統運作樣貌、多有效率跟多少團隊溝通？

## 重要性

這是架構師在 AI 時代保有價值的方法論——用低成本原型取代高成本文件維護。

## 邊界/反例

對於穩定的、變化少的系統，傳統 SPEC 可能仍足夠；Prototype 方法更適合早期、需求不確定、跨團隊複雜度高的場景。
