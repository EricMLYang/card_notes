# 3-Opinionated Agent 產品設計原則

**類型**：Principle

## 概念

最好的 Agent 產品不是最有彈性的，而是最有主見（Opinionated）的。**彈性陷阱**：沒有用戶會興奮地自己調整模型溫度和分塊策略，他們想要結果而非選擇（iPhone 一個按鈕但功能完整就是例子）。**替用戶做大量前置工作**：(1) 測試每個模型讓用戶不用測；(2) 寫詳細 prompt 定義成功樣貌；(3) 每個必填選項都代表你沒替用戶做好決定。**模型在框架裡不可替換**：模型智力是「尖刺狀」的，框架設計隱含繞過模型強弱點，升級模型常會打壞框架。

## 重要性

這是 Agent 產品設計的核心方向——做選擇、承擔後果、而非把選擇丟給用戶。

## 邊界/反例

對於專家用戶（如開發者工具、研究平台），適度的彈性可能是必要的。但即使如此，好的預設值仍然重要。此外，當領域知識高度專業化時，用戶可能比你更懂什麼設定最適合。
