# 3-Harness 是把模型變成 Agent 的關鍵（不是模型本身）

**類型**：Principle

## 概念

把「會聊天的 LLM」和「能交付成果的 Agent」區分開來的，不是模型的權重，是 **Harness（驅動框架）**。模型是原始認知能力，Harness 是落地行動力。Harness 像外骨骼而非一段 prompt，包含：(1) 迴圈/協調器；(2) 工具（檔案、shell、API）；(3) 上下文管理（工作記憶）；(4) 持久化（跨時間身份）；(5) 驗證（測試、linter）；(6) 限制條件（邊界與安全）。兩個產品用「同一個模型」體驗卻像不同物種——差別在 Harness。

## 重要性

這是理解 Agent 競爭力來源的核心認知——工程能力從「寫程式」轉移到「設計迴圈、限制、工具、驗證」。

## 邊界/反例

在模型能力差距極大時（如 GPT-3 vs GPT-4），模型本身仍是關鍵因素。但在模型同質化的今天（各家頂級模型差異縮小），Harness 的重要性越來越高。
