# Context Graph 捕獲組織隱性知識

- **類型**：Model
- **概念**：AI Agent 失敗的核心原因不是智慧不足，而是缺乏「制度記憶」。組織運作仰賴兩套平行系統：(1) 檯面上的正式流程、SOP、指引；(2) 沒人寫下來的潛規則、先例、覆寫和跨部門脈絡。Context Graph 是一種新架構，它將「行為事件」而非「靜態資料」視為主要物件，編碼「一個動作觸發另一動作的機率」和「標準流程在何時分歧」。傳統系統告訴你「現在狀態是什麼」，Context Graph 則告訴你「組織實際上如何做決策」。這是 AI Agent 從「新人」變成「老手」的關鍵基礎設施。
- **重要性**：解釋了為何多數 AI Agent 專案停留在原型階段（僅 11% 進入生產），並指出知識工程的新方向。
- **邊界/反例**：Context Graph 需要數年基礎建設投資，且組織脈絡「非常滑溜」難以精確定義。錯誤建模可能比無模型更危險，尤其在醫療、金融等高風險領域。
- **可連結關鍵詞**：#AIAgent #組織知識 #知識圖譜 #隱性知識 #決策脈絡
