

1.API 轉 MCP 是重要開發典範轉移: 
過去我們寫 API，主要是給人（例如前端工程師或應用程式）用的，
但在 MCP 架構裡，
你會有三個用戶：終端用戶、client app 開發者，還有 AI 本身，
所以要重新設計，讓 AI 真的能理解、使用、甚至組合這些工具，
這就像在設計一個「AI 能看懂的 SDK」。
    


2.在思考 AI 怎麼用你的工具前，
你要先想清楚用 AI 的人，會以自然語言問什麼問題？
MCP 工具應該提供**剛好能回答這些問題**的介面，而不是照搬 API 結構。

3.MCP First:
傳統做法是先開發一堆 API，然後再「包裝」成給 AI 用的工具；
但未來的趨勢是一開始就為 AI 設計工具，
想像 AI 是你的主力使用者，而不是附帶的客戶端。
