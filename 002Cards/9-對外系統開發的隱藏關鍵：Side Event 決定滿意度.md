# 9-對外系統開發的隱藏關鍵：Side Event 決定滿意度

**類型**：Heuristic

## 概念

對外系統開發與內部系統的基本差別：**如何讓系統達到客戶滿意的程度**。客戶有自己的時程、預算和優先序考量，你認為重要的功能客戶不一定要先做。關鍵洞見：客戶滿意度的關鍵常常不在主功能，而在 **Side Event**——使用者活動、系統監控與日誌、異常處理、數據同步。必須在設計、開發、驗測、試營運各階段都納入考量的面向：(1) **系統完成度**：UI/UX 設計、程式 error/event handling；(2) **維運 incident management**；(3) **資訊安全**：除了黑白箱測試，還有操作型資安（密碼規則、資料輸入驗證、回上頁漏洞等）；(4) **效能**：系統穩定與回應速度，更重要的是使用者操作效率（CRUD 最便捷路線、避免 page reload 等），與 UX 設計高度相關。

## 重要性

這是判斷「對外系統開發該關注什麼」的 checklist——功能是入場券，side event 決定滿意度。

## 邊界/反例

某些客戶確實功能優先、其他次之；內部系統可能可以妥協 side event。
