# 4-TDD 在 AI 時代的角色轉變：從開發者自律工具到 Agent 的控制介面

**類型**：Model

## 概念

傳統 TDD 是開發者自律工具——先想清楚行為規格再動手。當執行實作的是 Coding Agent，TDD 變成「控制介面」——測試不再只驗證對不對，它是你跟 AI 溝通「我到底要什麼」的唯一精確語言。時間分配發生結構性翻轉：傳統模式（需求 20% + 寫測試 15% + 寫實作 50% + debug 15%）→ AI+TDD 模式（需求 30% + 寫測試 40% + AI 實作 5% + 驗證 25%）。實作從佔一半壓縮到近乎消失，「定義什麼叫做對」從 35% 膨脹到 70%。開發者工作沒消失，但重心徹底搬家了。

## 重要性

重新定義了「寫程式」的價值結構——寫程式碼的邊際價值趨近於零，寫測試（定義驗收條件）才是核心產出。

## 邊界/反例

TDD 在探索性開發場景失效——你不知道要什麼時寫不出測試。過度依賴 TDD + AI 可能讓團隊傾斜向「有明確規格的任務」，壓縮創新空間。
