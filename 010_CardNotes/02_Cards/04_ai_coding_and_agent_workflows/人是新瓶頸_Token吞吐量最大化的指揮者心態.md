# 人是新瓶頸——Token 吞吐量最大化的指揮者心態

- 類型：Heuristic
- 來源：AI_Trend 多篇統整（Andrej Karpathy、Andrew Wilkinson）

## 核心命題

在多 agent 並行的工作模式中，人本身的工作分配和審核能力成為系統效率的唯一瓶頸。

## 卡片內容

Karpathy 把管理 agent 比擬為 GPU 使用率管理：「訂閱額度還有剩就代表沒有最大化 token 吞吐量。」他和 Peter Steinberg 的工作方式是同時開多個 agent 各跑獨立任務，在 agent 間穿梭分配、審核。Andrew Wilkinson 吃早餐同時啟動十幾個 Claude Code 任務。這意味著「從執行者到指揮者」的角色轉變不只是隱喻——你的分配工作能力和審查速度，字面上決定了整個系統的輸出上限。Karpathy 認為這「反而讓人很有動力，因為你還有成長空間」。

## 使用情境

1. 設計個人 AI 工作流時，反思自己是否成為瓶頸
2. 團隊管理中，重新定義管理者的價值——從監督執行到編排 agent

## 邊界 / 失效條件

1. 多 agent 並行時審核品質可能下降——speed vs quality 的取捨
2. 長時間高強度指揮可能導致決策疲勞
