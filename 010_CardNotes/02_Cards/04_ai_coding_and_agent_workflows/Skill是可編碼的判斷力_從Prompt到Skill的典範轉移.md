# Skill 是可編碼的判斷力——從 Prompt 到 Skill 的典範轉移

- 類型：Pattern
- 來源：AI_Trend 多篇統整（律師 Zack Shapiro、稅務師畠山謙人、Manny 曼報）

## 核心命題

Skill 與 prompt 的關鍵差異在於可系統化迭代、可轉移、可交易。它是把隱性知識變成可複利資產的核心機制。

## 卡片內容

2026 年 AI 工作流的焦點從 prompt 轉向 skill。Prompt 是一次性指令，Skill 是「可被系統化迭代更新」的結構化指令文件。律師 Shapiro 用 Claude 分析幾個月的工作模式，提煉出六個生產級 Skill（合約審查、修訂追蹤、法律研究等），每個都編碼了他十年執業經驗的分析框架與偏好。稅務師畠山的 CLAUDE.md 則將分錄規則、稅務分類、安全政策、判斷邊界全部寫成 AI 可讀格式。Skill 有三個關鍵特性：(1) 可轉移——裝到 50 名初級律師的機器上，每人都用資深合夥人的框架工作；(2) 可迭代——隨經驗累積持續更新；(3) 可交易——從「模型即服務」走向「技能即服務」（SaaS → Skill as a Service）。

## 使用情境

1. 設計 AI 工作流時，判斷該投資在 prompt 工程還是 skill 建設
2. 評估知識工作者的 AI 成熟度：是否已從「每次手寫 prompt」進化到「維護可迭代 skill」
3. 思考個人或組織知識資產化的路徑

## 邊界 / 失效條件

1. 模型大版本升級時，skill 可能需要重新校準（指令格式或行為漂移）
2. 高度動態、無法標準化的判斷場景，skill 的覆蓋率有上限
3. Skill 的品質取決於撰寫者的判斷力——垃圾進垃圾出
