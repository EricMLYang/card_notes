# 3-Agent 導入先做失敗管理：三問起手式

**類型**：Heuristic

## 概念

評估 Agent 工具時，先做失敗管理，再做功能探索。可用三問作為導入起手式：(1) 系統會拿到哪些權限；(2) 判斷錯誤時影響能否被隔離；(3) 失控時誰能立即剎車。這套順序把「能不能跑」的展示問題，轉成「出錯後能不能收」的生產問題，能提前暴露高成本風險。

## 重要性

可直接作為 PoC 到 Production 的守門機制，避免把 demo 成功誤判為可上線能力。

## 邊界/反例

在純離線、低權限、可隨時丟棄的個人實驗環境，可降低門檻；一旦涉及資料寫入或外部系統調用，三問不可省略。
