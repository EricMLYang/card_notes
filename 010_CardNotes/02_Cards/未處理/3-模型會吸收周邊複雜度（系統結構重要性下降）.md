# 3-模型會吸收周邊複雜度（系統結構重要性下降）

**類型**：Trend

## 概念
隨模型能力上升，部分原本由工程邏輯承擔的複雜度會回流到模型層（例如語意解析、策略選擇）。這使應用層流程可簡化，但也提高了評估、監控與責任歸屬的重要性。

## 重要性
它提醒架構師：未來競爭不只在系統結構設計，也在模型治理與驗證能力。

## 邊界/反例
高一致性與高可稽核場景仍需明確流程與規則引擎，不能把全部責任交給模型。

## 知識鉤子
可與 `#Harness 是把模型變成 Agent 的關鍵` 形成對照：複雜度不是消失，而是轉移。
