# 3-AI Error Analysis：LLM 應用評估方法

**類型**：Model

## 概念
LLM 產品評估不應先追單一分數，而應先建立錯誤分類。Error Analysis 的機制是：先抽樣真實案例、人工標註失敗模式、再把每類失敗對應到可量測指標與修復策略。這能把「模型好不好」改寫成「哪種錯誤最值得先修」。

## 重要性
它讓評估資源集中在最影響用戶體驗與商業風險的錯誤。

## 邊界/反例
若任務是標準答案型（如分類題），直接用傳統指標即可；但開放生成型任務若只看平均分，會遮蔽致命錯誤。

## 知識鉤子
可與 `#100樣本法`、`#可觀測性` 串成完整 eval loop，作為 agent 上線前的品質閘門。
