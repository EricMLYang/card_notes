# 領域專家是 AI 時代最大受益者——知道「該自動化什麼」才是真正瓶頸

- 類型：Principle
- 來源：AI_Trend 多篇統整（稅務師畠山謙人、Andrew Wilkinson、Dan Shipper）

## 核心命題

能判斷「應該自動化什麼」的，只有每天在現場操作的人。不懂程式但懂業務的領域專家，在 AI 時代獲得的槓桿最大。

## 卡片內容

稅務師畠山不是工程師，但他用 Claude Code 建了完整的自動化系統，自己寫的程式碼幾乎為零。他的核心優勢是：知道分錄規則、排除條件、安全邊界——這些「業務的型」是花了十幾年累積的，AI 無法自己生出來。Claude Code 缺的正是這種「現場知識」。Andrew Wilkinson 案例顯示設計師在 AI 時代被賦能最多，因為他們一直知道怎麼做好體驗，缺的只是把設計變成產品的能力。Dan Shipper 觀察：「程式碼便宜了，人人都能做 App，但知道怎麼做好體驗的人有了超能力。」三個獨立案例（稅務、投資、設計）交叉驗證：技術只是放大器，放大的是你對業務和人的理解。

## 使用情境

1. 勸說非技術背景同事/主管嘗試 AI 時的論證框架
2. 評估團隊 AI 導入優先序：哪個角色的「業務型知識」最適合編碼
3. 個人職涯規劃：投資在領域判斷力 vs 技術深度的取捨

## 邊界 / 失效條件

1. 基本的技術素養仍是門檻——完全不理解 AI 能做什麼的人無法有效指揮
2. 高度複雜的系統整合仍需工程師介入（如 API 串接、安全架構）
3. 「知道該自動化什麼」需要持續更新——AI 能力邊界在快速移動
