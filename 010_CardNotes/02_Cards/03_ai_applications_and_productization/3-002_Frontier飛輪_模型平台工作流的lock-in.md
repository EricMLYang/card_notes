# Frontier 飛輪 — 模型/平台/工作流的 lock-in

## 核心命題
模型愈強，平台愈有價值；平台嵌入愈深，轉換成本愈高；通過系統運行的工作流愈多，平台愈難被替換——遞迴鎖死的飛輪。

## 卡片內容
Frontier 飛輪揭示 AI 平台層的 lock-in 機制：模型強帶動平台採用，平台採用累積 workflow context，workflow context 提高換廠商成本，反過來給平台議價權再投資模型。對企業 IT 採購方的警訊：選 agent 平台比選模型嚴重得多，因為前者鎖死週期長且有累積債（類比 Salesforce/AWS 過去的 lock-in 歷史）。對 vertical AI 創辦人，escape hatch 在「multi-model abstraction」「agent runtime 標準化」與「保留 workflow context 所有權」三條路。

## 使用情境
- 協助客戶評估 agent 平台採購
- 診斷 lock-in 風險
- 設計多模型抽象層架構

## 邊界 / 失效條件
- 飛輪需要規模才能轉動
- 開源 agent runtime 標準化會弱化此飛輪
- 多模型抽象層可降低鎖死強度
