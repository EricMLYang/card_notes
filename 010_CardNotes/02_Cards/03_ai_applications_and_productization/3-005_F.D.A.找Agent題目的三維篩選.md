## 核心命題
Friction（高頻重複需人工審核）× Data Nexus（非結構化數據交匯點）× Actionable Outcome（從給圖表進化到交付工作成果）三維打勾才值得做 Agent。

## 卡片內容
F.D.A. 是把「該不該做這個 agent 題目」拆成可檢核的三軸。Actionable Outcome 通常是一票否決——若 agent 只能給 insight 不能交付成果，難以收 outcome-based fee。Friction 確認痛點頻率（值不值得花成本去自動化），Data Nexus 確認競爭壁壘來源（你比別人有什麼資料的特殊處理權）。三軸都中才值得進場，少一軸就退回到 GenAI 功能而非 Agent 產品。

## 使用情境
- 為 vertical AI 機會排序
- 篩選新 Agent 題目時的快速三問
- 對客戶解釋為什麼某個需求只值得做 GenAI 而不該做 Agent

## 邊界 / 失效條件
- 與 Workflow Embedding 4 特徵略有覆蓋，但角度不同（Agent 題目選擇 vs vertical 機會評估）
- Actionable Outcome 的可衡量門檻在某些品類（如創意產出）難定
