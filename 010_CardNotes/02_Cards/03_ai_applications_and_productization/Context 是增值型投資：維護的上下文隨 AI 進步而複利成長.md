# Context 是增值型投資：維護的上下文隨 AI 進步而複利成長

**類型**：Principle

## 概念

你花時間維護的結構化 Context（角色定義、決策原則、產品脈絡），會隨著 AI 能力提升而自動增值。今天的模型能讀你的 Repo 回答問題；明年的模型能自己判斷「哪些地方對不上、需要問」然後主動生成訪談問卷；再之後能設計多方訪談協議，一次性拿到所有關鍵視角。同一份 Context 資產，被越來越強的模型消費時，產出的價值呈非線性增長。這跟一般的文件維護根本不同——傳統文件的價值隨時間折舊（過時、無人讀），但 Context Repo 的價值隨消費者（AI）能力提升而增值。前提條件：Context 必須被結構化地持續維護，雜亂或過時的 Context 不會自動變好。

## 重要性

為「現在就該投入時間維護角色 Repo 和知識結構」提供了明確的經濟理由——這不是成本，是會複利增長的資產。

## 邊界/反例

增值前提是 Context 被持續維護；停止維護的 Repo 跟過期文件一樣會折舊。且 AI 能力提升的速度和方向不完全可預測——如果模型演進方向偏離「深度理解結構化文本」，這筆投資的回報可能低於預期。
