# 4-Code 是表達系統設計意圖最精準的語言，放棄它就放棄了必要性

**類型**：Stance

## 概念

AI Coding 時代不需要為實現功能寫那麼多 code，但 code 仍是表達系統設計和意圖最精準的語言。原因：(1) code 有編譯器和測試保護，改了這邊不會忘那邊；(2) code 可重構，比改文件穩定；(3) Unit Test 同時是 spec 和 sample code，純文件做不到。放棄設計和檢核，你在開發流程中的必要性也不存在了——不是不寫 code，而是寫的 code 從「實作」轉向「規格和驗收」。

## 重要性

這是對「不用再寫 code」論調的有力反駁——重點不是寫不寫，而是寫什麼。

## 邊界/反例

若 AI 未來能可靠地從自然語言生成 Interface + Test，這個主張的強度會減弱。目前 AI 生成的 Test 品質仍不夠可靠，需要人工校準。
