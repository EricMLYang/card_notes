# 8-Agent 時代 CPU 從配角變指令層：工具處理佔總延遲 50%-90%

**類型**：Model

## 概念

純推論時代，CPU 僅作為主節點轉發請求給 GPU，計算密集工作全在 GPU 上，CPU 很少成瓶頸。代理型 AI 翻轉了這個局面：Agent 串連工具呼叫、API 請求、記憶體查詢、編排邏輯——全在 CPU 上處理，GPU 反而閒置。喬治亞理工與英特爾 2025.11 論文量化了這個變化：CPU 工具處理佔代理工作負載總延遲的 50%-90%，CPU 動態能量消耗高達 44% 總動態能量。更關鍵的變化：從人機循環（人是瓶頸）到代理循環（每位使用者有數百個次代理同時運行），CPU 需要同時協調獨立/依賴關係、判斷完成時機、整合結果。**CPU 效能最終決定了 GPU 的使用量。**

## 重要性

改變了 AI 基礎設施選型的優先級——Agent 場景下，CPU（核心數、時脈、記憶體延遲）可能比 GPU 更值得優化。

## 邊界/反例

對於純推論（非 Agent）場景，GPU 仍是主要瓶頸。如果工具呼叫被更多地在 GPU 端用 Programmatic Tool Calling 處理，CPU 壓力會部分轉移。
