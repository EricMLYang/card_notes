# Clone Test — 判斷 vertical AI 護城河的單句判準

- 狀態：KEEP

- 索引：010_CardNotes/01_Index/07_product_management.md

## 核心命題
「如果今天用 frontier model 把創辦團隊和程式碼重現一次，他們為什麼不會贏過原版？」答得出來才有護城河。

## 卡片內容
Clone Test 是把「護城河」這個過度使用的詞，原子化成可逐題回答的判準。能撐過測試的不是程式碼也不是模型，而是無法快速複製的累積資料、operational learning、跨客戶 network effect、客戶端 workflow contextual knowledge。這個 test 也可套在自己的服務、產品、內容、個人品牌——如果一個 frontier model + 一個新人就能複製你做的事，你就沒有累積護城河。對 Eric「個人品牌的 Clone Test」是定期自檢工具。

## 使用情境
- 評估 vertical AI 投資
- 診斷自家產品 moat
- 做個人品牌定期自檢
- 給客戶做 due diligence

## 邊界 / 失效條件
- 通過 Clone Test 不代表商業成功（還要看 GTM）
- 早期公司本來就還沒累積，需給時間窗口
- 某些品類本身就不可能形成累積（純 thin wrapper）
