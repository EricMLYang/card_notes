\[參考\]

- 硬體參考附檔 ( 註: 是去年第一版資訊 )

- 對外報價 5000 – 6000 台幣

- 有過簡易的 FCC (美國)  跟  CE(歐洲) 相關自主認證 ( 非法規安規 )

- AI Camera 會裝設在電子螢幕正下方 ( 全家目前都是裝在結帳櫃台上方 )

- 模型: 原 MTCNN，後續改 YuNet， 配合 SORT (Simple Online and Realtime Tracking) 演算法

- 主要輸出 ( 沒細查是 YuNet、SORT 原始輸出，還是程式後處理 )
   	初始會設定一個方形區域 ( 進入此範圍的才算人流 )
   	進入時間、離開時間
   	觀看起始、觀看結束 ( 這個應該是根據模型輸出的臉部角度，判斷有無正對螢幕 )
   	Person_id ( 這個不會辨識是不是同一個人臉，只是 SORT 演算法的機制 )
   	年齡、性別 (目前還不準，沒有納入報表 )

- SORT 補充:
   	SORT 演算法用於AI攝影機，可以在連續的影像幀中追蹤多個人臉，因為在Edge裝置上有成本考量，所以演算法主要依靠「臉部框的移動距離」和「時間」來判斷是否為同一個人臉。
   	max_age：對應您描述中的「最大允許消失時間」，物體在這麼多幀未被檢測到後會被視為消失
   	min_hits：一個物體需要被連續偵測到多少次才會被分配一個穩定ID
   	iou_threshold：決定匹配的閾值，類似「最大允許消失距離」

- 效能相關參數: 目前每秒偵測 5 – 10幀 ( 0.1 – 0.2秒 一幀)

- 因為以上模型特性，所以會有以下高估人流的回推問題，當成參考資訊
