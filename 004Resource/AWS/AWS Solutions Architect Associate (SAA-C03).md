▋為什麼選擇 AWS Certified Solutions Architect – Associate (SAA-C03)?

被廣泛視為多個 AWS 專業認證路徑的重要基礎或起點。

它之所以重要,有以下幾個原因:

### 1\. 涵蓋廣泛且深入的 AWS 核心服務和概念

Solutions Architect – Associate (SAA) 專注於設計可擴展、高可用性、容錯和經濟高效的系統。這涉及對大量 AWS 核心服務(如 EC2、S3、VPC、RDS 等)以及架構設計原則的深入理解。

**這些知識不僅是擔任解決方案架構師的基石,也是進行大多數其他專業工作(例如開發、運營或專項領域)的通用和核心知識。**

---

### 2\. 專業級認證的推薦先修條件

雖然 AWS 認證沒有嚴格的強制性先修條件,但 SAA-C03 奠定的基礎知識對考取更進階的認證非常有幫助:

#### **AWS Certified Solutions Architect – Professional (SAP)**

- SAA 幾乎被視為考取此專業級認證的**自然下一步**

- SAP 建立在 SAA 概念的基礎上,要求更深入的知識和在複雜、多帳號環境中設計解決方案的能力

- **重要**: 通過 Solutions Architect Professional 會自動重新認證您的 Associate 級別證照

#### **AWS Certified DevOps Engineer – Professional (DOP)**

- **推薦的先修路徑**: AWS Certified Developer Associate 或 AWS Certified SysOps Administrator Associate

- 雖然 SAA 提供的廣泛架構視角對於理解如何設計和部署可運營的系統是極有價值的,但 **Developer 或 SysOps Associate 是更直接和推薦的路徑**

- SAA 可以作為輔助基礎,但不是最直接的前置認證

---

### 3\. 銜接專業化認證 (Specialty Certifications)

SAA 的內容能幫助您選擇並過渡到專業化認證,例如:

- **AWS Certified Security – Specialty**\
   SAA 涵蓋了 IAM、網路安全組、加密等基礎安全概念

- **AWS Certified Advanced Networking – Specialty**\
   SAA 涵蓋了 VPC、子網、路由表、安全組 vs NACL 等網路基礎知識

- **其他 Specialty 認證**\
   SAA 提供的廣泛 AWS 服務知識為深入特定領域打下良好基礎

---



▋AWS 認證路徑的常見起點

根據您的背景和經驗,AWS 認證路徑通常從以下**兩個起點**之一開始:

### **起點 1: AWS Certified Cloud Practitioner (CCP)**

**適合對象:**

- 對雲端計算完全沒有經驗的新手

- 非技術背景人員(如銷售、行銷、專案經理)

- 想要建立 AWS 基礎知識的初學者

**提供內容:**

- 高層次的雲端概念

- AWS 的商業價值

- 基本的計費模式和支援方案

- 核心 AWS 服務的概覽

**考試規格:**

- 65 題(50 題計分 + 15 題測試題)

- 90 分鐘

- 費用: USD 100

- 及格分數: 700/1000

---

### **起點 2: AWS Certified Solutions Architect – Associate (SAA-C03)**

**適合對象:**

- 具有 1-3 年 IT 經驗的專業人員

- 想直接深入學習 AWS 核心架構設計的人

- 已有雲端計算或本地基礎設施經驗者

- **可以直接跳過 Cloud Practitioner**

**提供內容:**

- 深入的架構設計原則

- AWS Well-Architected Framework

- 實際情境式的解決方案設計

- 安全、可靠、高效能和成本優化的系統設計

**考試規格:**

- 65 題(50 題計分 + 15 題不計分)

- 130 分鐘

- 費用: USD 150

- 及格分數: 720/1000

- Domain 1: Design Secure Architectures (30%)

▋推薦的認證進階路徑

### **標準學習路徑**

基礎級 → Associate 級 → Professional/Specialty 級



#### **路徑 A: 從零開始(無 IT 背景)**

1. **AWS Certified Cloud Practitioner** (基礎)

2. **AWS Certified Solutions Architect – Associate**

3. 選擇以下之一:

   - AWS Certified Developer – Associate

   - AWS Certified SysOps Administrator – Associate

4. 進階至 Professional 或 Specialty:

   - AWS Certified Solutions Architect – Professional

   - AWS Certified DevOps Engineer – Professional

   - 或其他 Specialty 認證

#### **路徑 B: 有 IT/雲端經驗**

1. **直接從 AWS Certified Solutions Architect – Associate 開始**

2. 根據職涯目標選擇:

   - **架構師路徑**: Solutions Architect – Professional

   - **開發者路徑**: Developer – Associate → DevOps – Professional

   - **運維路徑**: SysOps Administrator – Associate → DevOps – Professional

   - **專業領域**: Security, Networking, Data Analytics 等 Specialty 認證

---





▋認證之間的協同效應

研究顯示,不同認證之間有大量的服務和概念重疊:

- **Cloud Practitioner 與 Solutions Architect – Associate**: 約 46 個服務重疊

- **Solutions Architect – Associate 與 Professional**: 高度協同,是最自然的進階路徑

- **SAA 與其他 Associate 認證**: 共享大量核心 AWS 服務知識

**這意味著:** 每通過一個認證,您為下一個認證的準備時間會大幅縮短!
