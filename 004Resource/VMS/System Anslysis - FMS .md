# System Anslysis - FMS 



# 簡易版 Fleet Management System 需求文件

## 1\. 系統概要

本 Fleet Management System (FMS) 旨在提供電動貨車車隊管理功能，包含多租戶架構，支援不同權限呈現，並與 DMS（經銷商管理系統）整合。

## 2\. 功能需求

### 2\.1 用戶權限管理

- **FR-001**: 系統應基於多租戶架構設計

- **FR-002**: 不同用戶登入後，功能畫面及內容應依權限/組織有不同呈現

### 2\.2 車輛管理

- **FR-003**: 系統應以車籍ID (VIN) 為主要識別碼

- **FR-004**: 系統應支援無車牌號碼的車輛管理

- **FR-005**: 系統應接收並處理 DMS 提供的車籍資料

- **FR-006**: 系統應處理車籍資料更新（如車輛轉賣）

### 2\.3 車輛監控功能

- **FR-007**: 系統應記錄並統計行駛總里程

- **FR-008**: 系統應記錄單次里程

- **FR-009**: 系統應追蹤 GPS 定位里程

- **FR-010**: 系統應管理車輛定保計畫

- **FR-011**: 系統應記錄保全信息及保養記錄

- **FR-012**: 系統應記錄維修資料與維修部件訊息

### 2\.4 特殊功能

- **FR-013**: 系統應收集並分析駕駛行為數據（針對異常及事故確認）

- **FR-014**: 系統應提供異常通報功能（電壓、溫度）

- **FR-015**: 系統應計算及管理電耗/碳排數據

- **FR-016**: 系統應提供燈號儀表板功能

## 3\. 整合需求

- **IR-001**: 系統應與 DMS 系統整合，接收車籍及組織資料

- **IR-002**: 系統應開發 API 以接收相關資料

## 4\. 未來規劃功能

- 將來系統應能夠呈現充電地圖

- 將來系統應能夠整合商用地圖（北辰/勤威）以提供充電規劃

## 5\. 實施注意事項

- 系統使用權限前三年免費

- FMS 維運費用需納入成本考量

- 第一階段暫不開發 iOS/Android APP 系統
