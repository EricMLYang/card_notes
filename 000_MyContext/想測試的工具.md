# 想測試的工具（精煉版 v2）

篩選原則：每個面向最多 5 個，全部有公開 GitHub、架構輕量、可被 Claude Code 直接閱讀理解。舊清單項目保留並標註 ✅。

## Coding Agent SDK

1. **[Claude Agent SDK (Python / TS)](https://github.com/anthropics/claude-agent-sdk-python)** — 把 Claude Code 的能力（40+ 內建 tools、MCP、subagents、hooks）包成 library。你做 PydanticAI 時已上手 agent loop，這個是官方版、直接扒原始碼最划算。
2. **[Codex SDK (OpenAI)](https://developers.openai.com/codex/sdk)** — `codex.startThread().run()` 的極簡 API，可跑 MCP server 模式，也能接 Agents SDK。想跟 Claude Agent SDK 對比雙方設計哲學，這是第一手。
3. **[gstack](https://github.com/garrytan/gstack)**（YC 執行長 Garry Tan 推的 Skills）— 23 個 opinionated skills 把 Claude Code 變成 CEO/Designer/Eng Manager/Release/QA 團隊，含 `/plan-ceo-review`、`/codex`（跨模型互審）、Conductor 平行 10–15 sprint。跟你團隊 L1/L2/L3 Code Review 精神相通，可以當成你 Agent Role Repo Framework v2 的對照組。
4. **[Addy Osmani / agent-skills](https://github.com/addyosmani/agent-skills)** ✅ — SDLC 六階段 20 skills，跨 Claude Code / Gemini CLI / Codex / Cursor。
5. **[Auto-Skills](https://github.com/Toolsai/auto-skill/)** ✅ — Skills 自動生成。

## Agent 開發框架與 Harness

1. **[PydanticAI](https://ai.pydantic.dev/install/)** ✅ — 你已在用。
2. **NanoClaw / OpenClaw / Hermes Agent** ✅ — 個人/進化型 agent。
3. **Rivet Sandbox Agent SDK** ✅ — 15MB Rust binary，跨 harness 部署。
4. **agent-browser** ✅ — Accessibility Tree 取代 MCP 瀏覽。
5. **[Letta（原 MemGPT）](https://github.com/letta-ai/letta)** — OS-inspired memory-first agent runtime（core/recall/archival 三層記憶），對比 Mem0「只是 memory layer」的設計；Letta Code 是記憶優先的 coding harness，可看它怎麼把 memory 變 first-class。

## Edge / Local Model

1. **[Gemma 3 / gemma.cpp](https://github.com/google/gemma.cpp)** ✅（你寫的 Gemma 4 應該是指 Gemma 3 系列；Google DeepMind 官方 repo 已支援 Gemma 3 的 1B/4B/12B/27B，1B 僅 529MB 可跑在手機）— C++ 輕量推論引擎，讀原理很直接。
2. **[Karpathy nanochat](https://github.com/karpathy/nanochat)** — 8000 行 Python，涵蓋 tokenizer → pretrain → SFT → eval → inference → chat UI 全流程，$100 可訓出 GPT-2 等級模型。Claude Code 能完整 grep 整個 LLM pipeline。
3. **[Karpathy autoresearch](https://github.com/karpathy/autoresearch)** ✅ — agent 自動做 LLM 訓練實驗的最小化 harness。
4. **[llama.cpp](https://github.com/ggml-org/llama.cpp)** — GGUF 生態的事實標準，edge 模型落地繞不過。
5. **[MLX-VLM](https://github.com/Blaizzy/mlx-vlm)** — Apple Silicon 上跑多模態 LLM（含 Gemma 3 vision）的最快路徑。Mac 開發特別有感。

## Agent Memory

1. **[Mem0](https://github.com/mem0ai/mem0)** — 最大社群、48K+ star，3 行整合，framework-agnostic。你的 MI 2.0 semantic RAG / 決策支援 agent 需要 cross-session 記憶時的首選。
2. **[Letta](https://github.com/letta-ai/letta)** — OS 式三層記憶，適合長期運行 agent（投資人關係簡報 auto-draft 這類場景）。
3. **[OpenMemory](https://github.com/CaviraOSS/OpenMemory)** — 本地持久化、MCP native、把 temporal 當 first-class 維度，適合 Claude Desktop / Codex / Copilot。
4. **[Memori](https://github.com/MemoriLabs/Memori)** — LLM-agnostic memory infra，benchmark 上贏 Zep/LangMem/Mem0，prompt size 比 Zep 省 67%。
5. **[Awesome-Agent-Memory](https://github.com/TeleAI-UAGI/Awesome-Agent-Memory)** — 要系統性比較時直接看這個 curated list。

## RAG / 文件處理

1. **MarkItDown (Microsoft)** ✅
2. **LangExtract (Google)** ✅
3. **[LightRAG](https://github.com/HKUDS/LightRAG)** — EMNLP 2025 論文、知識圖 + hybrid search + reranker，低 RAM GPU（甚至 6GB）可跑，比 GraphRAG 輕量。你 DSBG news intelligence PoC 的 Databricks Vector Search 要比對架構時值得讀。
4. **[LlamaIndex Core](https://github.com/run-llama/llama_index)** — RAG 基礎設施事實標準，配合 LlamaParse 的 agentic OCR（130+ 格式）對你的結構化抽取場景特別有用。
5. **Mockdown.design** ✅

## 數據分析 / Text-to-SQL Agent

1. **[Vanna AI 2.0](https://github.com/vanna-ai/vanna)** — Text-to-SQL 開源代表，2.0 改成 user-aware agent 架構、row-level security、支援 Claude 4.5 / GPT-5。跟 Databricks Genie 的比較特別有參考價值（你的 Genie Code Notebook PoC 可直接對照）。
2. **[PandasAI (sinaptik-ai)](https://github.com/sinaptik-ai/pandas-ai)** — 對 DataFrame 對話式分析，含 Docker sandbox 安全執行，適合快速為 BU 做 ad-hoc 分析 demo。
3. **[DataMind (ICLR/AAAI 2026)](https://github.com/zjunlp/DataMind)** — 開源 LLM-based data analysis agent，含訓練資料與 7B/14B 權重，14B 在多個 data analysis benchmark 上超過 DeepSeek-V3.1 和 GPT-5。研究級別的 agent 訓練參考。
4. **[DBHub](https://github.com/bytebase/dbhub)** — 資料庫 MCP server，PostgreSQL/MySQL/MSSQL/SQLite 都支援，read-only 安全設計，是跟 agent 接真實資料庫最乾淨的介面。
5. **Databricks Genie + Unity Catalog**（官方，非開源但你已在用）— 列入對照點。

## 因果推論（你沒有但很值得加）

1. **[PyWhy / DoWhy](https://github.com/py-why/dowhy)** — Microsoft 主導、4 步驟框架（model → identify → estimate → refute），是你 D2D / DecisionOps 敘事最硬的技術底牌。A/B test 做不了時的 observational causal effect 估計。
2. **[EconML (Microsoft)](https://github.com/py-why/EconML)** — 跟 DoWhy 同生態系，專攻 CATE (Conditional Average Treatment Effect)，給「行銷活動 vs 轉換」、「會員計畫 vs LTV」這類分群因果題目用。
3. **[CausalML (Uber)](https://github.com/uber/causalml)** — Uplift modeling 導向，Ad Tech / RMN 場景裡「這次 push 對誰有增量效果」的核心工具。
4. **[DoWhy-GCM](https://www.pywhy.org/dowhy/)** — DoWhy 的圖模型擴充（AWS Amazon 貢獻），做根因分析、微服務延遲歸因。跟你 VMS 異常追因、供應鏈根因分析直接對得上。
5. **[An Open Source Ecosystem for Causal ML](https://www.pywhy.org/)** — PyWhy 總入口，含 Databricks 生態的 case study。

1. **Pascal Editor** ✅
2. **[FreeCAD](https://github.com/FreeCAD/FreeCAD)** — 開源 BIM + 參數化 3D 建模，Arch Workbench 給牆/門窗/結構件，IFC/DWG/STEP 支援齊全。你伴侶在工程規劃顧問業有土木/水利背景，這個是你們未來交集最直接的工具（參數化腳本 + AI 自動化）。

4. **[Cocoon-AI / architecture-diagram-generator](https://github.com/Cocoon-AI/architecture-diagram-generator)** — 2.5k star 的 Claude Code skill，自然語言 → 乾淨深色主題 SVG/HTML 架構圖。單檔 skill，最容易 fork 改成你的 SmartSignage/VMS template。
