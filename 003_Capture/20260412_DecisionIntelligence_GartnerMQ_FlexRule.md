# Gartner 首次發布「Decision Intelligence Platforms」魔力象限，代表什麼？

2026 年 1 月 26 日，Gartner 首次發布 **Magic Quadrant for Decision Intelligence Platforms（DIPs）**。這件事的真正意義，不只是多了一張新象限，而是 **Gartner 正式把「決策」從資料分析、規則引擎、AI 模型、流程自動化之間,拉成一個可被獨立評估與採購的企業軟體類別**。多家入選廠商都直接把這份報告稱為 **first-ever / inaugural** Magic Quadrant;公開資料也顯示這次評估涵蓋 **17 家供應商**。

## 一、這份新魔力象限代表的意涵

過去企業常把「決策」拆散在不同系統裡:

* BI 負責看報表
* 規則引擎負責條件判斷
* AI / ML 負責預測
* BPM / Workflow 負責流程執行
* 各業務系統各自保存自己的邏輯

Gartner 這次把 **Decision Intelligence Platforms** 拉成獨立市場,代表它認為企業已經需要一種更完整的產品:
**不只是看資料,而是把決策本身明確建模、執行、治理、監控,並從結果持續學習。** Gartner 在公開市場描述中提到,這類平台是用來建立 **decision-centric** 解決方案的軟體,會結合 **資料、分析、知識、AI**,去支援、增強或自動化人與機器的決策,並支援顯性 decision modeling、執行時協調 decision flow、監控 decision quality、從行動與 outcomes 中學習。

因此,這個新象限的核心訊號是:

**企業軟體的焦點,正在從「看懂資料」往「做出更好的決策並落地執行」移動。** 這也是為什麼多家廠商在對外說法中,都強調 decision-centric、decisioning、decision orchestration、decision governance,而不再只講 analytics 或 automation。

## 二、Gartner 在看什麼

Gartner Magic Quadrant 仍是用兩個軸來評估廠商:

* **Ability to Execute**
* **Completeness of Vision**

但對這個新類別來說,重點不只是產品功能多不多,而是它是否真的能支撐企業的決策閉環,包括:

* 明確建模決策邏輯
* 把資料、規則、模型與流程接起來
* 在執行時推動 decision flow
* 監控 decision quality 與業務結果
* 持續調整決策策略

從多家廠商公開提到的 **Critical Capabilities** 來看,Gartner 對這一市場已不只是概念命名,而是開始建立更細的評估框架,去看不同 decision use case 的適配性。

## 三、這些產品到底是什麼:用業務抽象來看

如果不看術語,這類產品可以抽象成四種用途。

### 1. 把公司的判斷規則系統化

很多企業的決策邏輯,其實散在 Excel、文件、舊系統、主管經驗裡。
這類平台幫客戶把「怎麼判斷」抽出來,變成可維護、可測試、可部署的 decision service。

典型問題像是:

* 保險理賠要不要過
* 銀行授信額度怎麼核
* 客戶該給哪一種優惠
* 哪些案件要人工複核

背後常用技術是:

* **business rules engine**
* **decision modeling / DMN**
* 視覺化建模
* 版本控管、測試、監控
* API / service 化部署
  這條路線的代表廠商包括 ACTICO、FICO、IBM、SAS、FlexRule、Sapiens、CRIF、InRule、Pegasystems。

### 2. 把分散資料拼成可做判斷的上下文

有些企業不是沒有規則,而是資料太碎,無法形成完整情境。
例如銀行要判斷一個客戶是不是高風險,不只看單一交易,而要把帳戶、公司、地址、電話、交易關係、異常模式一起看。

這類平台的用途,是幫客戶建立 **decision context**。
背後常用技術是:

* 資料整合
* entity resolution
* graph / network analysis
* contextual data layer
* 再接上 AI 或決策邏輯

代表廠商是 Quantexa、o9、RelationalAI 等。Quantexa 特別強調 contextual fabric、entity resolution、graph 與 trusted AI;o9 則以 enterprise knowledge graph 與 Digital Brain 為核心。

### 3. 把決策直接接上執行

有些客戶要的不只是「系統算出答案」,而是「系統能推動下一步」。
例如供應鏈系統發現某地缺貨,能不能自動建議調貨甚至觸發後續流程。

這類平台的本質,是把 decision layer 接上 workflow 與 action layer。
背後常用技術是:

* workflow / process orchestration
* event-driven decisioning
* decision engine
* agent / assistant interface
* 與 ERP、CRM、SCM 等整合
* feedback loop

代表廠商像 Aera、Decisions、Pegasystems,FlexRule 也有很強的 orchestration 色彩。

### 4. 在複雜情境下找更好的方案

有些決策不是簡單規則,而是多方案比較與最佳化。
例如:

* 供應鏈要怎麼分配庫存
* 哪種定價策略最好
* 哪個處理方案風險最低
* 如果條件改變,決策結果會怎樣

這類平台會結合:

* 預測模型
* 最佳化
* scenario analysis / what-if
* explainability
* 持續校正 decision quality

這條路線在 Aera、Rulex、Faculty、Oracle、部分 SAS / FICO / o9 身上特別明顯。

## 四、哪些客戶會用這類產品

最常見的使用者,不是一般中小型公司,而是 **決策頻率高、風險高、流程複雜、資料多、需要治理與審計的企業**。公開產品頁能看到的重點場域主要包括:

* **金融服務 / 銀行 / 保險 / 信用風控**
  例如授信、理賠、詐欺偵測、KYC、AML、風險評分、催收。FICO、CRIF、ACTICO、IBM、SAS、Quantexa 都在這一塊很明顯。

* **供應鏈 / 營運 / 企業規劃**
  例如庫存調度、需求預測、供應風險應對、跨部門營運協調。Aera、o9、Rulex 很典型。

* **受監管、需要明確治理與審計的決策場景**
  例如政府、金融、保險、大型企業的內控流程。這也是為什麼很多廠商反覆強調 explicability、governance、compliance。

換句話說,這類產品最適合那些:
**「不是只想看報表,而是真的想把高價值決策做成系統能力」** 的客戶。

## 六、為什麼這類市場現在冒出來

從趨勢上看,這個市場的形成有幾個背景。

### 1. BI 與 dashboard 已經不夠

企業不再只問「發生了什麼」,而更常問:

* 接下來怎麼做
* 哪個決策比較好
* 這個決策能不能自動執行
* 怎麼確保規則、模型與流程一致

這正是 Gartner 把市場拉成 decision-centric 的背景。

### 2. AI / ML 不再只是模型,而要進入業務決策流

光有模型預測不夠,企業更在意:

* 模型如何進到決策流程
* 誰來治理
* 出錯怎麼回溯
* 怎麼和規則、人員、流程整合

所以 Decision Intelligence Platforms 本質上是把 **AI、規則、資料、流程、治理** 放到同一層來看。

### 3. Agent 與 orchestration 讓這個類別更容易成立

近年的 agentic AI、workflow orchestration、decision automation,也讓市場更容易接受「決策平台」這種概念。Aera、Decisions、IBM 等公開敘事都明顯往這個方向靠。

## 七、市場有反應嗎?

有,而且是正向反應;Gartner 本身也把這個市場定位為 **late-stage emerging market**(晚期新興市場),代表它已從 niche adoption 進入可被任何規模、地區、產業採購評估的階段。

### 已經看得到的反應

**1. 供應商反應很強。**
多家廠商同步把這份 Gartner 報告當成重要里程碑,代表他們認為企業會開始用「Decision Intelligence Platforms」這個類別來評估產品。

**2. 分析師與評估框架開始完整化。**
除了 MQ,Critical Capabilities 也同步被拿來比較 decision use cases,顯示這不是只有概念炒作,而是開始形成採購與評估語言。

**3. 資本市場也有部分強訊號。**
例如 Quantexa 等 decision intelligence / contextual intelligence 平台在近年陸續取得大型融資,顯示資本市場願意把這類平台視為重要 enterprise category。

### 但也要看到限制

**1. 目前熱度仍以供應商與分析師生態為主。**
公開訊號多來自廠商新聞稿、分析師頁面、行銷活動,終端客戶大規模標準化採購的訊號還在積累中。

**2. 類別剛被正式命名。**
「首次 MQ」本身就代表這個類別剛進入可被標準化評估的階段,雖然 Gartner 已將其定位為晚期新興市場,但距離全面普及仍有差距。

## 八、可以怎麼理解這個新魔力象限

如果要把這件事講得最白話,可以這樣說:

**Gartner 這次不是在宣布一種新 AI 模型,而是在宣布:企業以後會越來越常用「決策系統」這個角度來買軟體。**

這類產品的核心,不是報表,也不是單一模型,而是把:

* 規則
* 資料
* AI / ML
* 流程
* 治理
* 執行

整合成一套能讓企業 **做決定、推動行動、監控結果、持續修正** 的平台。

因此,這個新象限代表的不是單一產品熱潮,而是一個更大的轉向:

**從 data-driven,走向 decision-centric。**
