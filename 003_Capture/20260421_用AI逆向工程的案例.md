之前有分享過幾個用 AI 來逆向的案例，週末整理了一下寫了篇文章。前四個案例是之前貼文講過的，五六七是新的。

第五個案例是混淆過的 app，AI 分析一下寫了個反混淆腳本，把一些字串跟 class 名稱還原，然後也分析了 so 檔案，把 key 還有加解密算法拿出來。

案例六是加殼過的銀行等級 app，但這個殼只加在 so 上，app 本身倒是只有混淆，不是把整個 dex 的邏輯全都塞到 native 層或是弄了個 VM，等級沒到這麼高。

總之後來東弄西弄，可以在 root 的模擬器上跑起來，能 hook 到發請求的函式。

案例七是加殼過的遊戲，殼沒解開，但因為遊戲是用 Lua，而 asset 那段只有稍微加密過，因此繞過加殼直接把 Lua 拿出來就結束了。我有試著讓 AI 脫殼但沒成功，一天沒成功就懶得繼續試了。

結論就是 AI 的能力比我想得還強，而且感覺還會一直進化。話雖如此，也不是說沒碰過解不開的東西，還是有遇過解不開的（如案例七），也沒碰過 VMProtect 那種直接搞了個虛擬機的大魔王，但反正現在 AI 能解的東西已經比我還多了，而且碰過的都解得開，這樣就夠了。

講到價錢，這次我有認真測了一下，一個 App 大概落在 30 塊美金左右。之前我貼文講的比這便宜許多，後來發現是因為 Cursor 方案還沒到期還在舊制的計價，照請求數量算錢的，所以 200 萬 token 才收我 0.1 塊美金，難怪這麼便宜...（現在沒了，一年期的方案結束了），照 token 收錢的話，用 Claude Code Max 應該是唯一正解，會比用 token 算錢省很多。
