## 0) 預售（12 月說明會）

### 情境問題

- 活動形式？

- 最希望客戶 第一眼看到什麼畫面？（假設：電池狀態監控、續航能力、能耗報表）

- 預計只展示靜態圖片，可能會用什麼形式呈現?（例如截圖、海報）

- 以前是怎麼準備這些素材的？在哪一步最花時間，或最容易出錯？



### 假設討論框架

- 誰用 → 銷售主管、售前顧問、行銷人員

- VMS 功能 → 直接提供報表或 KPI 的截圖，快速拿來展示

- 目的 → 讓客戶感受到買車後還有數據衍伸的價值

- 現在的做法 → EDM、傳單、網站頁面、人工截圖

- 痛點 → 準備過程花很多時間、資料容易不一致，或視覺效果不足以打動客戶









## 1) 初次接洽

第一次見面通常要快速建立認識與信任，

可先強調 EV 的核心價值主張：零排放、長續航、營運成本優勢。

### 訪談問題

- 第一次拜訪時，會用 VMS 數據準備介紹資料嗎？

- 續，如果不會，通常本階段在什麼情況下會用到 VMS ？

- 回答「跟柴油車比呢？」這類問題時，您覺得需要在 VMS 裡找到什麼資料，才有信心回答？

- 如果本階段只能展示 三張 VMS 相關的 PPT，哪些內容最重要，可爭取到第二次會議？

### 假設討論框架（供客戶補充或修正）

- 誰用 → 銷售人員

- 怎麼用 → VMS 相關截圖，歷史數據彙整成介紹文案

- 目的 → 建立信任，推進下一次深入會談

- 現在的做法 → 純靠規格等資訊來介紹

- 痛點 → 客戶追問數據證據時，不容易很快回覆











## 2) 需求挖掘／資格確認



主要是要了解客戶的車隊狀況（規模、行駛模式、載重需求、預算、充電設施現況），

以便判斷導入電動貨車的潛力。

### 訪談問題

- VMS 那些數據，是判斷「客戶適不適合電動車」的關鍵資訊？

 （例如：日行駛里程、平均載重、停靠或怠速情況）

- 您希望 VMS 數據分析，能自動幫您得出什麼樣的結論？

- 實際要用 VMS 數據來跟客戶說明時，可能會怎麼進行？



### 假設討論框架（供客戶補充或修正）

- 誰用 → 銷售、售前人員

- 怎麼用 → 匯入或引用客戶的任務特性，產生 「電動化適配度初評」

- 目的 → 快速判斷這個案子是否「值得投入更多資源深耕」

- 現在的做法 → 用 Excel 或人工估算

- 痛點 → 客戶提供的資料常常不完整，而且不同人算出來的結果口徑不一致，容易造成誤差











## 3) 技術簡報／價值證明



針對客戶需求，用技術數據來佐證 EV 的價值，

並且能和柴油車做出清楚比較，

讓客戶的技術團隊與採購部門都能放心。

### 訪談問題

- 客戶最常關心的指標有哪些？您通常怎麼利用 VMS 的數據來回應？

- 在談 TCO（總持有成本） 的時候，客戶最常追問您哪三個假設？

 （例如：電價、維修頻率、車輛殘值…）

- 在整理銷售資料時，您對於 匯出、視覺化、客製化報表 有哪些需求？

 （例如：匯出 PDF、設定篩選條件、數據權限控管…）

- 簡報結束後，客戶通常會要求哪些 附件或匯出，才願意進一步考慮 PoC？

- 展示數據前，通常怎麼確認資料的正確性與完整性？有沒有固定流程或特別的檢查方式？

### 假設討論框架（供客戶補充或修正）

- 誰用 → 售前、產品人員

- 怎麼用 → 每個客戶微調介紹報告

- 目的 → 讓客戶技術部門相信數據，讓 採購部門接受價值，兩方都能點頭

- 現在的做法 → 主要依靠 PPT、手動計算表格

- 痛點 → 不同假設會導致 TCO 結論差異大；簡報常常需要反覆改版









## 4) 試駕

### 訪談問題

- 試駕或 PoC 階段，需要 VMS 系統的協助嗎？

- 客戶在這個階段，最想看到的「數據證據」是什麼？

- 如果 PoC 的時間有限，您最希望 VMS 幫您快速準備什麼樣的展示？









## 5) 交付導入與追蹤

### 訪談問題

- 車輛交付後的 30 / 60 / 90 天，會不會持續追蹤客戶的用車狀況？

→ 如果會，您最想要哪些追蹤報表？

- 回訪客戶時，希望 VMS 提供什麼數據摘要，能快速了解車隊運行情況？

- 這些追蹤結果，通常會拿來給 誰看？（客戶？主管？自己？）

### 假設討論框架

- 誰用？ → 負責該客戶的業務

- 做什麼？ → 持續追蹤車輛狀況

- 目的 → 了解客戶實際使用情況、確保滿意度

- 現在的做法 → 打電話詢問、人工紀錄

- 痛點 → 缺乏自動化數據、難以持續追蹤、資訊不直觀





## 6) 異議處理／爭議釐清

### 訪談問題

- 在銷售過程中，通常哪些問題會需要您去釐清或回覆？

- 如果遇到像「電池健康度」這種質疑，您希望 VMS 能幫您做到什麼？

- 當發現異常狀況時，您會：

   - 主動去找保固/保修團隊釐清？

   - 還是希望能先透過 VMS 自行找到相關數據？

- 在處理爭議時，您認為 哪些資料只能內部查看？哪些能拿去對客戶展示？

### 假設討論框架

- 誰用？ → 業務人員

- 做什麼？ → 追溯維修紀錄、駕駛行為片段；生成「責任判讀摘要」

- 目的 → 降低不確定性

- 現在的做法 → 到處找紀錄、手動拼湊截圖報表

- 痛點 → 數據分散、時間線難對齊、對外版資料不可直接引用
