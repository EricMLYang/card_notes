# Azure DevOps

## Pull Request（PR）

在 Azure DevOps 中開啟 PR 並在核准後推送程式碼，您可以依照下列步驟操作：

**1\. 建立新的分支**

- 在您的本機儲存庫中，從主要分支（通常是 `main` 或 `master`）建立一個新的分支，用來存放您即將提交的變更。

- 將您的變更提交到這個新分支。

**2\. 推送分支到遠端儲存庫**

- 將您新建立的分支推送到 Azure DevOps 上的遠端儲存庫。

**3\. 在 Azure DevOps 中建立 Pull Request**

- 登入 Azure DevOps 並進入您的專案。

- 進入 "Repos" -> "Pull requests"。

- 點擊 "New pull request" 按鈕。

- 在 "Create a pull request" 頁面中：

   - 選擇您剛推送的新分支作為 "Source branch"。

   - 選擇主要分支作為 "Target branch"。

   - 填寫 PR 的標題和描述，清楚說明您所做的變更。

   - 指定審核者，這些人將會審核您的程式碼變更。

   - 點擊 "Create" 按鈕來建立 PR。

**4\. 審核和核准**

- 審核者將會收到通知，並對您的程式碼變更進行審核。

- 審核者可以提出評論、建議修改，或者直接核准您的 PR。

- 您可能需要根據審核者的意見進行修改，並將新的提交推送到您的分支。

**5\. 完成 PR**

- 當您的 PR 獲得足夠的核准後，您可以點擊 "Complete" 按鈕來完成 PR。

- 在 "Complete pull request" 頁面中，您可以選擇合併策略（例如，"Merge" 或 "Squash merge"）。

- 點擊 "Complete merge" 按鈕來合併您的變更到主要分支。

**6\. 程式碼推送**

- 一旦 PR 完成，您的變更就會被合併到主要分支。

- Azure DevOps 將會自動觸發 CI/CD 管線（如果已設定），將程式碼部署到您的環境中。

**重要注意事項**

- **分支保護：** 您可能需要設定分支保護規則，要求 PR 必須經過審核和核准才能合併到主要分支。

- **CI/CD：** 設定 CI/CD 管線，以便在 PR 完成後自動構建、測試和部署您的程式碼。

- **溝通：** 在整個 PR 過程中與審核者保持良好的溝通，確保您的變更符合團隊的標準和要求。
