# 店內電子螢幕廣告觀看分析：功能優先級清單

下面用「**非廣告 / 非 RMN** 也聽得懂」的方式，把你們可能做的功能分成 3 類，**各類都從高優先→低優先排序**，並標注**市場價值（高/中/低）+ 技術成熟度**。

每個功能都用同一種格式：**功能名稱｜市場價值｜成熟度｜常見於｜代表產品/公司**。

---

## 一、一般系統功能（先把「能穩定運作、可交付」做起來）

**1) 多螢幕內容排程與派送｜市場：高｜成熟：高**

- 說明：把內容「什麼時段、什麼店、哪面螢幕」播什麼，排程下去並準確派送。

- 常見於：Digital Signage CMS、戶外數位看板（DOOH）。

- 代表產品：Broadsign、Scala、ScreenCloud

**2) 網路監控與「有沒有真的播」的證明（Proof-of-Play）｜市場：高｜成熟：高**

- 說明：螢幕離線、黑屏、漏播、播錯，一眼看出來；並能輸出「確實播出」紀錄。

- 常見於：Signage / DOOH 網路營運（NOC）。

- 代表產品：Broadsign Control

**3) 對外整合能力（API / SSO / 企業系統串接）｜市場：高｜成熟：高**

- 說明：和既有 CMS、門店系統、BI、工單系統、身分系統串起來，才能真正落地。

- 常見於：企業 SaaS / 平台型產品。

- 代表產品：SpinetiX HUB（整合 Microsoft 365 / Power BI）

- **店內分析特別重要**：需串接 POS、會員系統、CRM 才能做「曝光 → 轉換」分析

**4) 權限、稽核、單一入口（Portal + RBAC + Audit）｜市場：高｜成熟：高**

- 說明：誰能看哪些店/哪些資料、做了什麼操作要留痕跡；多客戶/多部門都能管。

- 常見於：企業級平台（含 IoT / Signage / 分析平台）。

- 代表產品：SpinetiX HUB

- **店內分析特別重要**：AI Camera 涉及人臉/行為資料，合規與隱私控管是紅線

**5) 內容素材庫、模板、品牌風格控管｜市場：中｜成熟：高**

- 說明：素材版本、尺寸、版型、品牌規範；讓內容產出與上架變快。

- 常見於：CMS、內容管理平台。

- 代表產品：ScreenCloud、Scala

---

## 二、分析或非 Agent 的 AI 功能（AI 幫你「找線索、講清楚」，但不自動做決策/下指令）

**1) 自動洞察摘要（AI 自動解讀指標：趨勢/異常/原因）｜市場：高｜成熟：中高**

- 說明：不用會分析也能看懂「發生什麼事、可能原因」。

- 常見於：線上產品分析、BI 平台。

- 代表產品：Tableau Pulse、Power BI Copilot

- **店內分析特別重要**：門店主管不懂數據，需要 AI 直接講「這週哪幾家店效果掉了、為什麼」

**2) 指標異常偵測＋解釋（KPI Anomaly Detection）｜市場：高｜成熟：中高**

- 說明：例如「某店某時段觀看數掉了」，AI 主動指出 driver / contributor / outlier。

- 常見於：BI / 監控 / 營運儀表板。

- 代表產品：Tableau Pulse Insights platform

- **店內分析特別重要**：即時發現「螢幕位置不佳」或「內容不吸引人」的問題

**3) 多感測器/多資料源融合分析（Camera + POS + 會員資料）｜市場：高｜成熟：中**

- 說明：把 camera / 播放紀錄 / POS / 會員資料整合成一張「門店表現」全景圖。

- 常見於：零售分析平台、智慧建築/IoT 分析。

- 代表產品：RetailNext

- **店內分析核心價值**：這是你們的差異化優勢，能回答「看了廣告有沒有買」

**4) 「人群型態」自動分群（Audience Segments Discovery）｜市場：中高｜成熟：中**

- 說明：把「店/時段/區域」切出可理解的族群型態（午休上班族、晚間家庭客…）。

- 常見於：DOOH 受眾導向投放、零售數據分析。

- 代表產品：Vistar、Hivestack

- **店內分析調整**：需要大量高品質資料，且受隱私法規限制，屬於「進階功能」

**5) 內容影片解析與自動貼標（Video Understanding / Tagging）｜市場：中高｜成熟：中**

- 說明：把影片內容變成結構化標籤（情緒、物件、促銷訊號），後續才有「內容 → 效果」的統計。

- 常見於：DOOH 素材管理、內容審核、內容分析。

- 代表產品：Broadsign（AI 輔助素材分類，處於早期階段）

- **店內分析調整**：「內容效果分析」屬於進階需求，大多數客戶還在解決「有沒有人看」

## 三、Agent 功能（AI 不只講，還會「幫你做一段工作流程」）

**1) 素材 QA / 審核 Agent（分類、合規檢查、審核輔助）｜市場：高｜成熟：中**

- 說明：素材來了先自動分類、檢查規格/政策、把「需要人工看的」降到最少。

- 常見於：DOOH / 大型 Signage 網路（素材量大、人工審核很貴）。

- 代表產品：Broadsign AI Assistant（開發中）

- **店內分析特別重要**：連鎖門店每天收到大量素材，自動檢查「尺寸/品牌規範/敏感內容」能大幅降本

**2) Chat with Data：用自然語言問數據、生成圖表/頁面｜市場：高｜成熟：中高**

- 說明：直接問「這週哪幾家店掉最多？原因可能是什麼？」並產出視覺化。

- 常見於：BI / 企業資料平台。

- 代表產品：Power BI Copilot

- **店內分析特別重要**：讓區域主管、行銷人員自助查詢，減少「臨時取數」需求

**3) 成效/實驗設計 Agent（Privacy-safe 的 uplift / 對照組設計）｜市場：中高｜成熟：低**

- 說明：在不交換明細的前提下，協助設計「曝光組 vs 對照組」、輸出增量（sales lift）。

- 常見於：零售/金融等高隱私產業的跨方分析、廣告量測。

- 代表產品：Snowflake Clean Room

- **店內分析調整**：適用於「大型零售/金融客戶」，屬於高端需求

**4) 投放/排程規劃 Agent（Planner：給目標→產出計畫）｜市場：中｜成熟：中**

- 說明：你輸入目標（受眾、區域、期間、預算），Agent 產出螢幕組合與排程建議。

- 常見於：Programmatic DOOH、媒體採買平台。

- 代表產品：Vistar、Hivestack

**5) 維運 Agent（離線/漏播/異常 → 建議處置/工單）｜市場：中｜成熟：中**

- 說明：螢幕故障或內容播出異常，Agent 先判斷可能原因與建議動作，減少維運成本。

- 常見於：Signage / IoT 網路營運。

- 代表產品：Broadsign Control

**6) 動態內容觸發設定 Agent（用天氣/時段/即時資料觸發不同版本）｜市場：中｜成熟：中**

- 說明：把「什麼條件 → 播什麼版本」用自然語言設定並上線。

- 常見於：DOOH / 進階 Signage（動態創意、即時觸發）。

- 代表產品：Broadsign DCO

- **店內分析調整**：可結合「人流資料」做即時觸發，但 ROI 不易量化

## 為什麼「廣告平台」值得非廣告領域參考（用白話講）

1. **它把「分散式 IoT 網路營運」做到極致**：Broadsign 管理超過 42.5 萬面螢幕，要穩定派送、監控、可追溯（Proof-of-Play）

2. **它非常重視「可被相信的成效」**：不只是報表好看，而是要能做增量/對照、甚至用 clean room 在高隱私前提下完成衡量

3. **它把「內容供應鏈」工程化**：素材量巨大，所以分類、審核、合規都會自動化（你做任何內容平台都會遇到）

4. **它是「實驗文化」的表親**：線上產品用 A/B 測試學習最佳做法，實體螢幕也在走向同樣的「對照 / 實驗 / 迭代」思路

5. **市場規模與投資正在上升**：T-Mobile 以 6 億美元收購 Vistar Media（2025年1月），反映 DOOH 技術被視為成長型市場

你上面列的 **Agent 類功能**，在商業模式上大致會落在 4 種「可賣的型態」。你可以把它想成：**你到底是在賣一個工具、賣一段工作流程、賣一個可被信任的證明、還是直接賣成果**。

---

## 先給你 4 種主流 Agent 商業模式（你這類 B2B 最常用）

### 1) 內嵌式 Copilot（SaaS 加值）

\*\*你賣什麼：\*\*在既有平台裡多一個「AI 幫手」按鈕/聊天框，提升效率。\
**怎麼收：**

- 每人每月（seat-based）＋（可選）用量計費（token/次數/算力）

- 或直接做「AI add-on」按模組收費

\*\*適合：\*\*Chat with Data、報表問答、規劃建議、維運建議這種「省時間」的 Agent。

---

### 2) 交易型／每次任務計費（Transaction-based）

\*\*你賣什麼：\*\*一段完整的工作流程結果（一次規劃、一次審核、一次報告、一次實驗）。\
**怎麼收：**

- 每次任務（per run / per report / per campaign / per experiment）

- 或按輸入規模（每支影片分鐘數、每個素材、每個門店/檔期）

\*\*適合：\*\*素材 QA、內容審核、投放計畫生成、報表自動生成（尤其給非日常使用者）。

---

### 3) 高信任「證明」型產品（Proof / Measurement-as-a-Product）

**你賣什麼：不是 AI，很像「鑑定報告」：你幫他把效果算得可信**、可交付給品牌/主管/客戶。\
**怎麼收：**

- 每次研究/每次 uplift study（固定費）

- 或按規模（曝光筆數、門店數、POS 交易量）

- 或企業年約（把研究能力包進合約：每月 X 次）

\*\*適合：\*\*Clean Room Uplift、實驗設計 Agent、成效歸因 Agent。

> 這種通常毛利高、成交慢，但一旦進去就很黏。

---

### 4) 成果分潤／風險共擔（Outcome-based / Rev-share）

\*\*你賣什麼：\*\*你敢跟客戶一起賭結果：例如 uplift 提升多少、媒體收入提升多少。\
**怎麼收：**

- uplift 的一部分（或增量營收 %）

- 或跟媒體投放金額抽成（% of spend / % of incremental sales）

\*\*適合：\*\*當你能控制影響鏈路（播放→曝光→內容→銷售），而且有足夠數據證明時。

> 風險：定義增量很難、資料權限/合規/結算爭議多，所以通常是「大型客戶 + 成熟階段」才敢做。

---

## 套回你列的 Agent：各自最合理的賣法

### A) Chat with Data（自然語言問數據、產圖表/頁面）

**最像：內嵌式 Copilot（模式 1）**

- 收費建議：

   - **每位使用者/月**（基本）＋ **用量**（token/查詢次數/重算次數）

   - 或「AI Insights 模組」按 **站點/品牌/租戶**收

- 為什麼：它是高頻日常工作、價值是省時間與降低門檻。

---

### B) 素材 QA / 審核 Agent（分類、合規、規格檢查）

**最像：交易型（模式 2）＋ 企業加值（模式 1）**

- 收費建議：

   - **每素材/每支影片/每分鐘影片**（很好算、客戶也好理解）

   - 大客戶可改成「月包量」或「年約含配額」

- 為什麼：素材進來就是一批一批，最自然就是按“件數”或“分鐘數”計價，而且客戶很在意可預期成本。

---

### C) 成效/實驗設計 Agent（Clean Room uplift、對照組）

**最像：證明型產品（模式 3），成熟後可走成果分潤（模式 4）**

- 收費建議（常見做法）：

   - **每次 uplift study**（含設計＋計算＋報告）

   - 或 **企業年約**：每月/每季固定幾次 study

- 為什麼：客戶不是買 AI，是買「可被信任的證明」。這類產品最值錢的是方法論、合規流程、可審計性。

---

### D) 投放/排程規劃 Agent（Planner：目標→產出計畫）

**最像：交易型（模式 2）或 Copilot（模式 1）**

- 收費建議：

   - 若面向「售前提案/代理商」：**每份 plan / proposal**

   - 若面向「內部營運團隊」：**每位/月**（搭配使用權限）

- 為什麼：它通常不是每天用，但每次用都要產出一份能交付的“計畫”。

---

### E) 維運 Agent（離線/漏播/異常→建議/工單）

**最像：SaaS 基礎功能綁定（模式 1）**

- 收費建議：

   - **含在 Enterprise 版**（提高 ARPA、降低 churn）

   - 或按 **螢幕數/站點數**收（很貼近成本）

- 為什麼：這是「平台可靠性」的一部分。通常不適合拆太細賣，拆了反而讓客戶覺得你在對關鍵能力加稅。

---

### F) 動態內容觸發設定 Agent（用天氣/時段/即時資料觸發）

**最像：模組加值（模式 1）＋ 用量（模式 2）**

- 收費建議：

   - **DCO 模組**（固定月費）＋ **觸發規則數/觸發次數/即時 API 次數**

- 為什麼：你在賣「即時決策能力」，成本跟觸發頻率、資料來源、邊緣端運算強相關。

## 你們情境下，我會建議的「打包成 SKU」方式（最好賣、最容易講清楚）

### 1) Core 平台（必買）

- CMS / 播放 / 派送 / 監控 / proof-of-play

- 收費：按 **螢幕數 / 站點數 / 租戶**（最直覺）

### 2) AI Insights Pack（高機率最快成交）

- Chat with Data + 自動洞察摘要 + 異常解釋

- 收費：按 **使用者席位** 或按 **站點**（二選一，看你客戶是誰）

### 3) Creative Intelligence Pack（容易擴張）

- 影片解析貼標 + 素材 QA/審核 Agent

- 收費：按 **素材量/影片分鐘數**（非常好談）

### 4) Proof Pack（最有護城河、但銷售週期長）

- Clean Room uplift + 實驗設計 Agent + 報告交付

- 收費：**每次 study** 或 **企業年約含配額**（成熟後再談分潤）

### 5) Realtime Pack（後期戰略）

- 動態觸發 + Edge 回饋閉環

- 收費：固定模組費 + 用量（觸發/規則/API）

## 一句話總結你問的「Agent 會是怎樣的模式？」

- **省時間/降低門檻的 Agent**（問數據、看報表、規劃建議、維運建議）→ **SaaS 加值（席位/模組＋用量）**

- **一批批處理的 Agent**（素材審核、內容 QA、計畫生成）→ **每次任務/每件計費（或包量）**

- **要讓客戶敢花大錢的 Agent**（uplift、歸因、可審計的證明）→ **證明型產品（每次 study 或年約配額），成熟後才走分潤**
