# AI 時代的 PM 轉型：從 Project Manager 到 Possibility Maker

> 來源：Gipi Yu「PM 思維升級實戰，打造變動時代的專案主導力」課程心得
> 核心主題：AI 時代 PM 角色從「執行者」轉變為「想法推動者」

## 0 - AI 與人生智慧

AI 進步的速度，已經遠遠超過你我進步的速度。至少在「智能」方面是如此，但「智慧」方面可能還有一段時間。

雖然 AI 叫做「人工智慧」，但我們更需要「人生智慧」。**尤其在 AI 的時代，人生智慧更成我們安身立命的最後聖地。**

## 1 - 神奇時刻變成日常時光

Agent 現在越來越會做事了，如果你曾經叫 Agent 去做事，一定記得那「第一次」的神奇時刻。

不過很快的，曾經的神奇時刻，一瞬間，今日已經變成日常時光。至少，在某些早已投誠 AI 陣營的人類來說，無時無刻指揮 10 幾個 Agent 做事，已經是每一天的日常場景。

你交辦完 Agent 做事，他就領了任務自己去完成，完成後才回報你。從 5 分鐘回報的小任務，到 50 分鐘回報完成的任務，到 5 小時才回報完成的困難任務。

就算花了 5 小時，但 agent 做出來的品質無可挑惕，反正這期間，你去跑了個步、喝了個下午茶，就是讓 agent 去忙。

看著 10 幾個 agent 在螢幕上忙碌，喝一口咖啡，感覺自己很威，宛若上帝。

## 2 - 執行不是困難，想法才是

**「執行」已經不是困難了，現在你需要的是「想法」。**

「要叫 AI 去做什麼呢？」尤其，當你把 AI 訂閱刷下去後，看這些 AI 閒著沒事做，總覺得浪費，就跟老闆一樣，看員工閒著，心裡就覺得虧錢。

而且務實來說，所謂的「想法」，是「想出能夠賺錢的方法」，才是大家真正關心的。畢竟養 AI ，也是很貴的，投入也是要有產出。

可是，要能賺錢，麻煩就來了，也就是需要「人生智慧」的地方。

因為，要賺錢，就是要去賺「人類」的錢，所以你得要跟人類溝通。

結果你發現，跟 AI 溝通越來越簡單，叫了就去做事，做得又快又好，還全年無休。

但跟人類溝通，在現在的時代，是有史以來的困難。

**無論如何，People Matters。（PM）**

## 3 - PM 從 Project Manager 到 Possibility Maker

結果 AI 越進步，「PM」的能力越來越重要。

雖然身為 PM，你可能曾經懷疑，PM 會被 AI 取代。

當你指揮 Agent 去做事的時候，你看著這些 agent 們，他們會先自己擬定計劃，讓你確認。然後依據計畫，還會自己拆出 To-do，接著自己照著 To-do，一條一條 task 去執行。

agent 還會自己準備測試計劃，自己測試，最後交出成果。

你感覺，這不是以前你自己身為 PM 在做的事，AI 做的...挺好的。怎麼辦？

**在 AI 的時代，PM 的工作重點，更應該從「執行」，要變成「想法」，尤其是「想法的推動」。**

**PM 的意義，正在從從前的「Project Manager」，慢慢變成「Possibility Maker」，讓想法成真。** 關鍵已經不是「執行」了，那部分 AI 會越做越好，但現在你需要的，是「推動」。

你需要去 Prompt 某人動起來。

## 4 - 從 Prompt Master 到 Proposal Master

你很會用 AI，但你要說服的人，可能並不是。

你晚上指揮了 10 隻 Agent 做事，覺得很威，白天回到人間，1 個人都說服不了，覺得沮喪。找不到 prompt 人類的方法。

**你是「Prompt Master」，你是「Prototype Master」，但你更需要變成「Proposal Master」。**

因為 AI，你看到了一些新機會，但老闆不買單。因為 AI，你做出了一些新產品，但客戶不買單。或是合作單位不買單。

最終都需要「提案」，提案才能破局。

對老闆提案、對客戶提案、對合作單位提案，才有辦法從這些「利害關係人」身上，獲得必要的資源，主要是錢，或是人力，才能去推動你想要完成的事情。

可是一提案，就會被挑戰，可能會被打槍，可能會需要負責，可能會失敗。

可是不去試著提案，Possibility 還是依然 impossible。

你是 PM，你要是 Possibility Maker，就要會是 Proposal Master。勇敢提案，努力破局，想法才會成為可能。

## 5 - 一切都是 Context

指揮 Agent 要理解 Context，而要改變人們的想法，也需要理解「Context」，屬於人類的 Context。

為什麼提案老闆不買單？為什麼提案客戶不買單？合作單位老是找渣？

**他們的 Context 是什麼？**

提案要容易成功，你必須理解他人的語言、了解他的脈絡。他最近的痛點是什麼？他的 KPI 是什麼？他想達成的目標是什麼？他真正的問題是什麼？

你不了解對方的 Context，你的提案對對方來說，就是一堆噪音。

AI 不了解你的 Context，AI 給你的回答也一樣馬馬虎虎。

**一切都是 Context。**

識別 Context、理解 Context、建立 Context，在現在變成格外重要。無論對 AI，還是對人。

## 6 - 看懂局勢：順勢、借勢、造勢

也就是要能夠看透「局勢」是什麼。

### 順勢
如果對方的 Context 跟你一致，提案容易成功，可以「順勢」而為。

### 借勢
如果老闆的 Context 雖然跟你方向雷同，但他有更急著要解決的問題。識別 Context，然後從老闆的優先順序出發來提案，可以「借勢」推動自己的提案。

### 造勢
如果所有的利害關係人的 Context 都互相衝突，就需要從中理出可能重疊的關鍵點，然後從這個焦點出發，發展出可以連解彼此，多贏的提案，也就是最困難的「造勢」。

**PM 更需要是 Positioning Maker，借勢、造勢的人。**

看懂局勢、了解局勢的脈絡，你才能循著脈絡，在對的位置注入你的想法，讓想法在對的位置慢慢發展，最後讓想法發生，改變局勢。

## 7 - PM 你的人生

PM 變成在 AI 時代中生存的重要技能。

不只要懂 AI，也更要懂人。讓 AI 去執行、讓想法在人類之間發生，你才會與眾不同。

# 拆解結果

## A. 主脈絡與個人映射
- **論證骨架**：從 Agent 執行力已超越多數 PM 的日常工作 → 執行不是困難，想法才是 → PM 從 Project Manager → Possibility Maker → 三個 Master 升級路線 → 核心能力是 Context（對 AI 和人都一樣）→ 順勢/借勢/造勢三種策略。
- **個人映射**：對系統建造者的雙重衝擊：(1) 技術人也需要 Proposal Master 能力，否則再好的系統也推不動；(2) Context 是跨領域的底層能力——寫好 prompt、做好提案、推動決策，底層都是 Context 管理。

## B. 卡片（Zettel）

序號 1
- 標題：PM 從 Project Manager 到 Possibility Maker：三個 Master 升級路線
- 類型：Model
- 概念（50–300 字）：Agent 已能自主擬定計劃、拆 To-do、準備測試、交出成果——PM 過去做的事 AI 做得挺好的。AI 時代 PM 的工作重點從「執行」變成「想法推動」：Prompt Master（會用 AI）→ Prototype Master（能做出東西）→ Proposal Master（能說服人買單）。最後一步是真正的瓶頸：你晚上指揮 10 隻 Agent 覺得很威，白天回到人間 1 個人都說服不了。提案才能破局——向老闆/客戶/合作單位獲取資源（錢、人力），不提案 Possibility 就永遠 impossible。更進一步，PM 要成為 Positioning Maker——看懂局勢，在對的位置注入想法。
- 重要性（1 句）：對任何想推動變革的人（不限 PM 職稱）的角色自我檢驗——你卡在哪個 Master？
- 邊界/反例（1–2 句）：在高度技術導向的組織中，Proposal 的說服力可能取決於技術實力而非提案技巧。三個 Master 不一定是線性升級，有些場景可以跳級。
- 知識鉤子：與 #Codex團隊40人1PM 形成對比——Codex 的 PM 是高速分流器，這裡的 PM 是想法推動者，兩者互補；與 #AI是槓桿你是支點 呼應——Proposal Master 就是讓 AI 槓桿產生商業價值的「支點」。

序號 2
- 標題：一切都是 Context：對 AI 和對人的底層能力是同一個
- 類型：Principle
- 概念（50–300 字）：不了解對方的 Context，你的提案就是噪音——不管對方是 AI 還是人。理解 Context 的四個維度：痛點、KPI、目標、真正的問題。三種基於 Context 的策略：(1) 順勢——對方 Context 跟你一致，直接推；(2) 借勢——從對方的優先順序出發重新包裝你的提案；(3) 造勢——所有利害關係人 Context 互相衝突時，找出重疊的焦點，發展多贏提案。這個能力對 AI 和人一樣通用：AI 不了解你的 Context，回答馬馬虎虎；老闆不了解你的 Context，提案不買單。
- 重要性（1 句）：這是跨領域最可遷移的底層能力——寫 prompt、做提案、推動決策、設計產品，底層都是 Context 管理。
- 邊界/反例（1–2 句）：Context 理解需要資訊取得管道；在資訊不對稱嚴重的場景（如跨國、跨文化），Context 判斷容易出錯。過度揣測 Context 也可能導致過度包裝、失去真實性。
- 知識鉤子：與 #不可壓縮才是護城河 形成互補——Context 理解能力是不可壓縮的；與 #垂直AI護城河 中的 Domain Know-how 呼應——Domain Know-how 本質就是特定領域的深度 Context。
