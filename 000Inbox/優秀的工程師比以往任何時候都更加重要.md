優秀的工程師，比以往任何時候都更加重要
󠀠
最近 OpenAI 的 Codex 與 Anthropic 的 Claude Code，都有工程師公開表示，自己發的 PR，程式碼 100% 由 AI 撰寫。
󠀠
於是有網友問「如果 Claude Code 已經能 100% 取代寫程式這部分，為什麼 Anthropic 目前還有超過百個工程相關的職缺?」
󠀠
Claude Code 作者 Boris 的回覆是「總得有人去給 Claude 下指令、與客戶溝通、協調其他團隊、決定下一步要開發什麼。工程領域正在改變，而優秀的工程師比以往任何時候都更加重要」。
󠀠
以我們自己用 AI 工具的經驗，不會否定「未來軟體工程實作的部分可能多數會由 AI 處理」的可能性，但是我們認為，要能發揮這些工具的最大功用，必須先掌握軟體工程的基礎。
󠀠
舉例來說 Andrew Ng 教授曾分享過一個例子，他在使用 MidJourney 時，發現自己無法精準描述想要的結果。而他的一位合作夥伴，因具備藝術史背景，熟悉不同藝術風格與流派，能精準描述需求，因此使用 MidJourney 的成果更好，儘管 Andrew Ng 教授是 AI 專家。
󠀠
這個例子凸顯了品味的重要性，對藝術史有品味、能精準描述需求的人，能夠更好掌握圖像生成 AI。
󠀠
而對軟體工程師來說，要能掌握程式生成的 AI，則要培養對程式設計的品味，需要了解程式設計的歷史脈絡、各種範式、設計模式及撰寫風格，因為唯有具備這些背景知識，才能有效與 AI 互動，精確表達你希望程式碼如何撰寫。
󠀠
