# New Camera

初步結論：我們技術團隊認為產品表還還不錯，直得繼續深入測試，以下問題



一、模型特性 或 後處理問題：

1\.發現：自行測試中，短暫臉朝 camera ( 小於 1S )，觀看時間數據會是 1 - 5S

問題： 目前狀況比較沒有過濾的規律，觀看率 … 等數據會很高出



2\.發現：戴安全帽會辨識不出這個人以及觀看

問題：可否說明模型實際辨識行為



3\.發現：純人臉的資訊來去測試年齡\
 但僅人臉影像較不易被偵測，

問題：這邊想確認人臉偵測模組的原理或運作機制，是否是先偵測人，再把人的資訊去做年齡性別辨識?



4\.發現：地端的 dashboard 中，有時會出現辨識資料突然消失或辨識結果被修改的情形

問題：目前模型怎麼判斷是否為同一個人 ( EX: 兩幀間臉移動距離小於門檻？）

這部分程式怎麼後處理模型輸出，是否能提供有那些後處理機制？



5\.發現：年齡有一致的偏年輕

問題：這樣的一致性是否建議我們在數據中心做偏移處理就好？



6\.發現：目前表格有兩個，一個是地端機器的，另外一個是雲端的 report\
 雲端 Age 區間顯示（0–18、18–24…）與地端顯示（0–10、11–21…）有所差異\
 問題：想請問一下這塊產生的方式有何不同，或是有經過什麼處理?



二、後續配合應用問題

1\.我們會有數據中心分析播放的廣告內容和觀看行為，未來是否可以配合數據串接？

（我們數據中心有 SaaS 版本 和 大客戶的獨立版本）



2\.貴司的模型演進規劃？如果我們量大的話是否可配合進行微調？

\
3\.. Report 頁面中包含多個指標，是否有地方可以查看這些指標的明確定義與其計算方式？



4\. 另外想確認一下，雲端顯示資料的延遲大約需要多少時間？



5\.機器按鈕僅提供 Restart 與 Power，若需安全且正規地關閉系統，請問是否有建議的操作方式？



6\.有沒有 Camera 的搭配規格？EX: 解析度…等



三、進階測試評估：

1\.是否可以詳細描述模型輸出 和 程式後處理流程？這樣我們比較能仔細比對



2\.是否有機會能直接對底層模型進行測試?\
 這樣對於某些指標可以測試的比較準確



3\..是否能提供地端機器所使用的 Linux 系統之登入帳號與密碼，以便後續測試？
