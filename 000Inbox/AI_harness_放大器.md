 大多數人以為 AI 的進步是模型的問題。更大的上下文窗口、更強的推理能力、更少的幻覺、更高的基準測試分數。這些沒有錯，只是真正的槓桿點不在那裡。
因為悄悄地把「會聊天的 LLM」和「能交付成果的 agent」區分開來的，不是模型的權重（weights），是 harness（驅動框架）。
我接下來要說的很多東西可能很快就都會是錯的。
這個領域變化太快，沒有人能百分之百正確。但方向已經夠清楚了，值得你拿未來 12 個月去押注。
⸻
幾個月前，「AI」意味著一個聊天框。現在，「AI」意味著一個迴圈。
一個能讀取檔案、執行指令、搜尋、規劃、驗證，並且持續運作直到事情完成的迴圈。那個迴圈就是 harness。
OpenAI 在描述 Codex 時講得很直白：核心的 agent 迴圈負責協調使用者、模型和工具之間的互動，而他們把這個 agent 稱為「the harness」。
Claude Code 也把話挑明了：它是一個「agentic harness」，提供工具、上下文管理和執行環境，把語言模型變成一個能寫程式的 agent。
所以如果你一直在糾結哪個模型「最強」，你大概是在爭論引擎好壞，卻忽略了整台車。
這就是為什麼很多人會輸。
⸻
常見的思考：「模型就是產品。」
這是大多數人走的路。你看到排行榜，選一個冠軍，然後期待冠軍幫你賺大錢。
但 agent 不只是模型，它們是被系統包裹起來的模型。Harness 就是那個系統，它是一套鷹架，迫使模型像個靠譜的隊友，而不是一隻聰明的鸚鵡。換個方式說，模型是原始的認知能力，而 harness 是落地的行動力。
這就是為什麼兩個產品明明用「同一個模型」，體驗起來卻像完全不同的物種。不同的節奏、不同的穩定度、不同的質感。同一顆大腦，不同的 harness。而這應該改變你思考職涯的方式。
我們正在進入的時代，工程能力不會消失，只是會轉移。花更少的時間一行一行寫程式，花更多的時間設計迴圈、限制條件、工具、記憶機制、驗證流程和介面。
OpenAI 自己那篇「Codex harness」的文章就深入說明了這一點：在核心 agent 迴圈之外，harness 還包含一套執行環境與對話執行緒的持久化機制，最終透過應用伺服器架構對外提供服務。這是一門全新的技藝。Harness 技藝。
⸻
人們聽到「harness」，腦中浮現的是一段提示詞，一段塞滿全大寫警告的巨型系統提示詞。
那只是其中一小部分。
Harness 更像是一副外骨骼，而不是一個句子，它是一整套技術堆疊。
當 OpenAI 說 harness 是協調使用者、模型和工具的 agent 迴圈時，他們指的是中樞神經系統。當 Claude Code 說 harness 是工具加上下文管理加執行環境時，他們指的是四肢、感官和肌肉。
合在一起看，一套實用的 harness 通常包含以下部分。
首先是迴圈，也就是協調器，負責不斷重複的循環：觀察、決定、行動、驗證、更新，然後再來一次。
其次是工具，也就是雙手與感官，涵蓋檔案讀寫、shell 執行、瀏覽器、API、程式碼搜尋、git 操作、資料庫查詢，任何能把文字變成實際動作的東西。
第三是上下文管理，也就是工作記憶，決定哪些內容要納入、哪些要摘要、哪些要壓縮、哪些要釘選為永遠生效的規則。
Anthropic 針對長時間執行 agent 的研究就聚焦在這一點：agent 在跨越多個上下文窗口時會遇到困難，而一套有效的 harness 能借鑑人類的工程實務，讓 agent 在數小時甚至數天內持續穩定推進。
第四是持久化，也就是跨越時間的身份，包括執行緒、狀態、任務記憶、產出物和檢查點，讓 agent 不會每一輪對話都像金魚一樣重新開始。OpenAI 明確指出「Codex core」既是函式庫也是執行環境，負責管理 Codex 執行緒的持久化。
第五是驗證，也就是真相與阻力，涵蓋跑測試、跑 linter、型別檢查、建置流程、合理性檢驗、評分標準、自我批評和外部審查。大多數「agent 的神奇表現」真正的來源就在這裡：不是天才般的靈光，而是不間斷的回饋。
第六是限制條件，也就是邊界與安全，界定它可以在哪裡寫入、可以執行什麼、什麼必須先取得許可、什麼絕對不能做。
如果你有注意到的話，會發現一件事。以上這些，沒有一項是「模型的改進」。
它們全都是系統/軟體工程上的改進。
Harness 的改進。
⸻
為什麼 harness 比你想的更重要？
因為失敗才是預設狀態。不只是在人生中，在 agent 的行為中也是。LLM 會漂移、會迷路、會過度擬合最後一條指令、會誤解目標、會追著閃亮的東西跑。就算非常聰明的模型也一樣。
所以 harness 做的，就是每一個好的環境為人類做的事：降低決策疲勞，提高回饋密度，讓正確的行動比錯誤的行動更容易，讓你始終朝著北極星前進。
這就是為什麼頂尖的團隊不只是「用 AI」，他們在 AI 周圍打造 harness。而我們正在見證前沿從「提示工程」轉向「編排工程」。Sean Goedecke 說得很直接：近期的許多進步不只是模型變強了，而是「agentic harness」的改進，也就是把 LLM 包在一個帶有工具的迴圈裡的那層程式碼。
這就是那個轉變。從問問題，到打造一套系統，讓它持續地問、持續地做、持續地檢查、持續地修正。
⸻
接下來是一個令人不太舒服的事實：短期來看，AI 讓平庸的執行變得廉價；長期來看，AI 讓平庸的決策變得致命。因為當任何人都能生成任何東西的時候，優勢就轉移到了那些知道「什麼東西值得存在」的人手上。那就是品味。但品味本身仍然是抽象的，它只是「想法」。Harness 是品味被具體實現的方式。
一套 harness 從頭到尾就是一連串的決策：我們給它什麼工具？我們強制什麼順序？我們拿什麼來衡量成功？它怎麼從失敗中恢復？它什麼時候該停下來？它記住什麼？它拒絕什麼？所以你的 harness 就是一面鏡子，它映照出你的價值觀。而在一個到處都是複製貼上 agent 的世界裡，價值觀就是差異化。
這就是為什麼「你本身就是利基」不再只是一句勵志標語，它是一條工程原則。兩個人可以用同一個模型，品味更好的那個人會打造出更好的 harness，更好的 harness 產出更好的成果。錢流向成果，錢是信任的度量衡，而信任靠的是可靠性。可靠性，主要是 harness 的問題。
⸻
接下來才是有意思的地方。Harness 不只是一個軟體成品，它是「行動力」的隱喻。想想你自己的人生：你也有一個「模型」，也就是你的大腦、你的直覺、你的智力；你也有一套 harness，也就是你的環境、日常作息、限制條件、工具、回饋迴圈，以及你的身份認同結構。
大多數人第一件事就是想升級「模型」，想要更多動力、更多意志力、更多自律。
但最快的改變，來自 harness 的升級：移除讓你分心的東西，增加回饋機制，壓縮從行動到看見結果之間的時間差，建立迫使你行動的預設值，設立防止自我破壞的限制。
行動力是習慣，不是天賦，而習慣通常只是 harness 的設計。所以當你在學習「harness」這個技術概念的時候，你同時也在學一項人生技能：別再苦苦哀求你的大腦自動自發，開始打造讓行為自然發生的系統。
這也是為什麼 agent 的競賽不只關乎智力，它關乎自我修正的能力。而自我修正，基本上就是把謙遜付諸實踐。
⸻
一套好的 harness 只做一件事：把模型的輸出變成現實，再把現實回饋給模型。就這樣。文字、行動、回饋、更好的文字、更好的行動。這個迴圈，就是讓 agent 感覺「活著」的原因。這也是「聊天」這個框架正在消亡的原因。聊天是一次性的智力，被 harness 驅動的 agent 是迭代式的智力。而迭代贏過天才，每一次都是。
⸻
Harness 能解決的問題包括卡住不動、遺失上下文、默默失敗、沒有查核就產生幻覺、偏離目標、沒有工具就空轉、沒有獲得許可就過度行動，以及信心滿滿地做錯事。Anthropic 在長時間執行 agent harness 方面的研究，基本上就是一種坦承：難的不是模型不會思考，而是隨著時間推移，進展非常脆弱。
Harness 無法完全解決的則是另一類問題：目標本身就有問題、意圖不明確、沒有定義什麼叫「完成」、工具品質低落、缺少關鍵資料、需求本身互相矛盾，以及人類自己也還沒決定。
這帶我們來到最重要的一點。Harness 不會取代你，它會放大你。如果你有品味、有方向、有標準，harness 讓你變得強大到危險。如果你沒有，它只會讓你製造更多噪音。
⸻
我們正在即時見證一個角色的誕生。
不是「提示工程師」，那是昨天的頭銜了。真正的角色更接近 agent 設計師、工具編排者、工作流程工程師、上下文工程師，或者叫 harness 架構師。
你不需要資工學位就能入門，但你確實需要一種新的思維方式。你得停止問「這個模型能做什麼」，開始問「什麼樣的系統能讓這個模型變得可靠」。
這是一個更成熟的問題，也是那種能把你從 99% 的消費者變成 1% 的創造者的問題。
⸻
我最喜歡的科學概念之一是：當一個系統被推到離平衡態夠遠的時候，秩序可以從混沌中自發湧現。
漩渦在河中形成，不是因為水「決定」要有秩序，而是因為限制條件加上流動，創造出了有組織的模式。Agent 就是這樣。
原始的模型是流動的水，強大、混亂、沒有方向。Harness 是河床，是限制、回饋和引導。
把足夠的流量推過正確的限制條件，一個連貫的結構就會自然湧現。一個有能力的 agent、一個創造者、一份事業、一段人生。
這就是為什麼跌到谷底有時候反而能帶來突破。當混沌劇烈到一個程度，你原本的 harness 就會崩塌，你被迫建造一套新的。那不是失敗，那是演化。
⸻
這裡有一個陷阱：不管你有沒有刻意設計，你都會有一套 harness。
學校是一套 harness，工作是 harness，社交圈是 harness，App 是 harness，演算法是 harness。它們塑造你的注意力、你的決策、你的身份認同。
如果你不自己打造 harness，就會有人幫你安排一套。
然後你在某一天醒來，才發現自己一直活在別人設計的工作流程裡。
那就是糟糕 agent 迴圈的存在主義版本：只是反應而不是主動出擊，只是消費而不是創造，只是漂流而不是瞄準。所以在 AI 的脈絡下學習「harness」，同時也是在訓練自我主導的能力。你開始注意到迴圈無處不在，你開始設計屬於自己的。
⸻
大多數人把這件事搞砸了，因為他們把它想得太玄。他們想像一套完美的系統，能自動運轉他們的整個人生和事業。那是幻想。Harness 是一次一個迴圈慢慢蓋起來的。
你目前需要做的事：用一句話定義「完成」，你定義不出來的話 agent 也做不到，你自己也做不到。
給它一個能創造現實的工具，不論是跑測試、查資料庫、交付草稿還是發布文章，一個行動介面就好。
加一道回饋關卡，一個必須通過才能往前走的檢查，例如測試全綠、評分達標、二次編輯完成。
釘選一組不可妥協的原則，包括風格規則、安全規則、品牌語調、限制條件和優先順序。
最後，記錄每一輪迴圈，保留過程中的產出物，這樣下一次迭代才是進步，而不是重蹈覆轍。
這就是一套 harness。小的、真實的、會複利的。一旦你建好一個能運作的迴圈，就開始疊加迴圈。技術堆疊，就是 harness 的堆疊。
⸻
AI 的下一個階段不會被背誦模型更新日誌的人贏得，會被那些能把智力編排成具體成果的人贏得。
Harness 是隱形的優勢，它是模型外面那層系統，也是你外面那層系統。如果你覺得自己落後了，很好，那是彈弓正在拉滿。因為一旦你看見了 harness，就再也回不去了。而從你開始設計迴圈，不再苦苦乞求動力的那一刻起，你的人生開始重新感覺是你自己的了。
那才是真正的升級。
