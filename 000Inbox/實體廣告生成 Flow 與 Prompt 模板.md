# **實體廣告生成 Flow 與 Prompt 模板**

## 前置資訊

- 工具: 

   - Nano Banana + Google Veo3 (都有 API )

   - MoviePy: 影像編輯、文字補充與貼上 ( 尚未使用，理論可行，惟文字放置位置要安排 )

   - 進階: 獨立配音: MMAudio AI 之類的工具 ( 有 API，但尚未研究 )

- 目前不足:

   - 文字生成很不穩定，最好後製

   - 聲音生成效果普通

   - Veo3 目前解析度最高 1080P

- 尚未串成 Flow，目前都用 Chat UI 介面測試效果

- 影片是分階段生成，各斷點的延續性還無法解決的很好，可能透過上一部影像的最後截圖加工處理



## 主要步驟

1. 廣告資訊萃取: 將廣告原始資訊變成固定格式的 Meta 資訊

2. 素材備料生成: 發散過程，利用上述資訊加上圖片，進行發想，增加切入點

3. 分階段產生腳本:

   1. 開場

   2. 主體 ( 因應 Veo3 最多8秒，可多段 )

   3. 收尾

4. 產生主文案: 將後製到 Camera 

5. 各段影像生成:

   1. 產生圖像:  輸入:素材;  輸出:基礎場景圖

   2. 產生影像 Prompt: 將腳本、相關素材，與上述影像當輸入;  輸出 Veo3 的 Prompt ( 通用 prompt + 插入變數效果不好 ) 

   3. 產生影像: 輸入: 用 Prompt + 基礎場景圖;  輸出: 影像

   4. 擷取最後一幀影像

   5. 重複(依據開場或

6. 產生收尾圖: 輸入: 相關素材;  輸出: 廣告收尾或呼籲行動圖片

7. 以上用工具組合

## 1\.素材備料生成

```markdown
任務：
你將協助擴散與發想，關於實體電子看板廣告文案和腳本的材料素材，請利用廣告擴散發想發電機，
將個觀點的衍伸素材整理出來


<廣告擴散發想發電機>
1) 逆向痛點切入（Inversion for Ads）

目的：先想「怎樣讓人不看、不記、不到場」，倒推真正有效的主張＋鉤子。
  •	怎麼問
	•	如果要讓【單一受眾】完全忽視這支看板／短片，我會怎麼做？
    •	反向：要「救回注意力」，我能用的唯一一句主張是什麼？
  •	示例（親子受眾）
	•	忽視原因：像一般科教展＝「無聊又學術」。
	•	核心主張：「玩到會發光的科學」
	•	Headline：玩到發光｜補充：互動×3 關卡開啟

2) 動機五問（Five Whys → 承諾語）
目的：一路問到受眾的真動機與恐懼，萃成一句承諾語（Promise Line）。
  •	怎麼問
	•	為什麼要來看？→為什麼那很重要？（連問5次）
	•	還擔心什麼？（時間？小孩會無聊？擁擠？）
	•	輸出樣板
  •	示例
	•	承諾：15分鐘玩出一個光電觀念
	•	化解：動線短、三關卡、館內有遮蔽
	•	證據：3座大型互動裝置可見即信
3) 隱喻鉤子（Analogy → 視覺化）
目的：把抽象利益「翻成」直覺畫面與短語。
  •	怎麼問
	•	這次主題像什麼日常事物？（園藝、魔術、遊戲破關、料理…）
	•	哪個隱喻一看就懂、好拍成第一鏡？
	•	輸出樣板
  •	示例
	•	隱喻：魔術 → 光=魔法；光譜=咒語；面板發亮=成功施法
	•	Headline：會發光的魔法課

⸻

4) 角色視角切換（Perspective → 鏡頭語言）
目的：不同利害關係人＝不同第一句與第一鏡。
  •	怎麼問
	•	用誰的眼睛看？（孩子、家長、老師、館方、看板自己）
	•	他/她第一秒最在乎什麼？
	•	輸出樣板
  •	示例（家長）
	•	Hook：玩得到、學得住
	•	第一鏡：孩子手按裝置→面板亮起→笑聲

⸻

5) 組合式主張（Combination Grid）
目的：用「場景 × 益處 × 社會梗/流行語」做批量組合，快速爆想10+方向。
  •	怎麼問／怎麼做
	•	列3欄各5項：
	•	場景（櫃檯上方屏／館內中庭牆／電扶梯旁…）
	•	益處（動手玩、快速懂、親子互動、避暑有遮蔽…）
	•	梗（開箱、闖關、升級、打怪、魔法…）
	•	隨機連線，選好懂＋好拍者。
</廣告擴散發想發電機>

<廣告資訊>
</廣告資訊>
```

```markdown

《廣告文案金句大師》
請根據我上述的文字內容，依照<文案金句模板>中提到的技巧，將輸入的文案，改寫成極
有力道且犀利的實體電子看本用的金句。
每一個文案要寫一個取代
請抓出輸入資訊的內容核心概念來寫金句，不要拿枝微末節來寫。


## 工作流程
1. 請你先點出這篇廣告的核心概念
2. 請你依照<電子看板金句模板>，每個文案寫出十個金句
3. 為每個文案挑選一個最好的，只需呈現最好的。
4. 以輸出格式呈現，要與輸入的順序相同。


格式 Json：
{["文案1內容", "文案N內容"]}



<電子看板金句模板>

【對比式金句】

在 1 秒內製造強烈反差，讓觀眾「被震一下」。
範例：
	•	「玩，不只是玩。」
	•	「不是來學光，而是讓光動起來。」
	•	「白天太陽發光，夜晚你發亮。」
	•	「不是科技展，是魔法現場。」
	•	「我們不賣燈，賣的是發亮的驚喜。」

【具象化金句】
讓觀眾「看見畫面」，不是「讀懂文字」。
範例：
	•	「光，從孩子手中跑了出來。」
	•	「一推開門，整個房間都發光。」
	•	「小手一動，世界亮了。」
	•	「陽光被收進盒子裡，變成遊戲。」
	•	「笑聲，閃閃發亮。」

【搜詞法】
以一個「關鍵字」為中心，創造節奏感與記憶點。
範例：
	•	「玩光，發亮光，懂光。」
	•	「光一動，夢就亮。」
	•	「動手玩，動心看。」
	•	「一閃一亮，一學就懂。」
	•	「光的遊戲，亮的知識。」

 小結 — 電子看板金句的3個原則
	1.	單一聚焦：只說一件事，一眼理解。
	2.	視覺可想像：讓人腦中有畫面，不需文字解釋。
	3.	節奏與聲音：像說話、像節拍、像心跳。
</電子看板金句模板>
```

## 5\. 廣告影片生成

開場影片 Prompt 產生器 - Google Veo3

```
你將擔任專業影片導演與廣告創意總監，專精於為 Google Veo3 撰寫高效的生成式 Prompt。

你的任務是將我提供的**「影像腳本」，轉化為一段單一、完整、可直接使用於 Google Veo3 的英文 Prompt**，專為實體電子看板廣告的開場而設計，確保能在最短時間內抓住觀眾目光。

在收到腳本後，你將遵循以下經過優化的專業流程來建構 Prompt：

1.強力開場 (Impactful Opening):

確保 Prompt 的開頭能指令 Veo3 在影片前 1-2 秒內生成一個極具衝擊力的視覺掛鉤 (Visual Hook)，並立即確立畫面的主要焦點與主題。

2. 電影感場景 (Cinematic Scene-setting):

你會用精準的詞彙描述場景的時間、地點、光線與氛圍，營造出強烈的電影感。同時，會指定攝影機的初始角度（如：低角度特寫、環繞視角）。

3. 精準運鏡 (Precise Cinematography):

將明確定義核心的動態，並使用專業的鏡頭語言指令，如 slow push-in (緩慢推進)、cinematic pan (電影感平移) 或 match cut transition (匹配剪輯轉場)。

4.核心主角／產品 (Core Subject/Product):

若有主角，你會細緻描寫其關鍵的外觀特徵、微表情與動作。

若為產品，則會著重於其材質質感、反光細節、以及與環境的互動方式。

5. 沉浸式音效 (Immersive Audio):

你會為場景指定匹配的背景音樂風格（如 ambient electronic music）與環境音（如 sound of a bustling city at night），以強化整體氛圍。

6. 銜接式結尾 (Transitional Ending):

Prompt 的結尾會導向一個清晰的畫面，通常是產品的特寫或主角的定格，為後續的資訊或 Call-to-Action 影片留下完美的銜接點。

7. 視覺風格準則 (Visual Style Guide):

你將嚴格遵守您的風格要求：指令將導向簡潔的構圖、明確的焦點，並追求高飽和度與高對比的色彩表現。

同時，你會加入提示詞以避免畫面出現不必要的雜亂元素，並確保影片本身不包含任何由 AI 生成的文字或字幕。

最終，你將嚴格按照您的要求，僅輸出該段英文 Prompt，不包含任何額外的說明或註解。

</本影片腳本>
```

**內容影片** Prompt 產生器 - Google Veo3

```markdown

你將擔任專業影片導演與廣告創意總監，專精於為 Google Veo3 撰寫高效的生成式 Prompt。

你任務是，根據**「前導影片的最後一幀畫面」**與**「新的影像腳本」**，生成一段用於廣告**主體 (Main Body)** 的 Veo3 Prompt。

此任務的核心是**「無縫銜接 (Seamless Continuity)」**。你將確保這段影片能與前導影片完美串接，創造流暢連貫的觀看體驗。

收到腳本與參考畫面後，你將遵循以下專業流程來建構 Prompt：

1.  **關鍵幀銜接 (Keyframe Continuity):**
    * 你會以**前導影片最後一幀畫面**作為視覺基準。Prompt 的開頭將精確描述該畫面的核心元素（如主角、產品、環境），確保 Veo3 能從完全一致的狀態開始生成，達成無縫的視覺過渡。

2.  **場景發展 (Scene Development):**
    * Prompt 將指令 Veo3 在延續的場景中引入新的變化或深度，而不是重新建立場景。你會描述環境、光線或氛圍的細微演進。

3.  **動態敘事 (Dynamic Narrative):**
    * 你將設計鏡頭運動與主要動作，以推動故事發展。例如，從前一幕的靜態特寫，轉為 `slowly pulling back to reveal` (緩慢拉遠以揭示...) 的動態鏡頭。
```
