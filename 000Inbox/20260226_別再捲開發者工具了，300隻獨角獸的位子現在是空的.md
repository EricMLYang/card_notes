# 別再捲開發者工具了，300 隻獨角獸的位子現在是空的？！

> 來源：YC CEO Garry Tan 基於 Anthropic 最新數據的分析
> 核心議題：AI Agent 垂直領域市場的巨大機會

## 核心事實：AI Agent 市場的極度失衡

YC CEO —— Garry Tan 透過 Anthropic 的最新數據，點出了一個極具顛覆性的事實：**AI Agent 市場有一半被「寫 Code 產品」佔據了。剩下的另一半是滿地黃金的荒野。**

Anthropic 發布了 AI Agent 在真實世界運作的數據：**Software Engineering 佔據了高達 49.7% 的 Agent API 呼叫量。** 而醫療 (1%)、法律 (0.9%)、教育 (1.8%) 等 16 個垂直領域，全部不到 5%。這是市場根本還沒開始的訊號。

## 能力溢出：Trust Gap 就是產品機會

現在最大的機會在於 **「AI 能做的」與「人類敢讓它做的」之間的巨大落差。** 測試顯示，Claude 已經有能力獨立解決需要人類 5 小時的工作。但在實際生產環境中，最極端的重度使用者，最多也只讓它連續跑 42 分鐘。這是 Trust (信任) 和所謂調配好的流程還沒跟上。**這個落差，就是巨大的做產品的機會。**

## 老手反而更常打斷 AI

數據揭露了一個反直覺的現象：新手會自動批准 (Auto-approve) 20% 的操作；而累積了 750 次對話的老手，自動批准率超過 40%。但有趣的是，**老手打斷 AI 的頻率 (9%) 竟然比新手 (5%) 還高。**

因為老手已經掌握了 Agentic Workflow。他們從「每做一步都要我同意 (Pre-approval)」，進化成了**放手去做，走偏了我再拉你回來**。更重要的是，好的 Agent 會自己停下來問問題（Claude 主動發問的次數是人類打斷的兩倍）。**主動發問不是 Bug，是 Feature。真正的 Autonomy 是由模型、使用者與產品共同建構的 (Co-constructed)。**

## 垂直領域的護城河

過去 20 年，SaaS 創造了 300 多隻獨角獸。接下來的十年，**每一個 SaaS 獨角獸，都會對應誕生一個 Vertical AI 獨角獸。而且規模會大 10 倍，因為它們取代的不只是軟體，還有操作軟體的人。**

任何人都能寫個 API Wrapper，那沒有護城河。**真正的護城河，是深入醫療帳單、法律搜證、營造合規這些又髒又重的 Domain Know-how。** 能解決舊有系統的摩擦力，並完成企業端的變革管理 (Change Management)，才有機會獲勝。

## 個人心得

這份報告給了我們最清晰的 Roadmap。我常覺得 AI 賽道已經很擁擠了，但其實擁擠的也許只有同溫層。當你手握世界上最強大的 Agentic 武器，去挑選一個市場份額不到 1% 的傳統領域，帶入你的專業與洞察，把那些還沒體驗過 AI 的古老工作流，徹底重構。**別再捲了，去佔領那等待你解放的綠洲。**

## 16 個極度期待被 Agent 重構的產業板塊

順著 Garry Tan 的 SaaS 獨角獸邏輯，以下盤點 16 個極度期待被 Agent 重構的產業板塊：

### 高監管高摩擦力的護城河產業

#### 1/ 醫療照護 (Healthcare) - 佔比 1%
從病歷摘要整理、保險理賠自動化
到醫療法規合規檢查

#### 2/ 法律法務 (Legal) - 佔比 0.9%
合約自動審閱、判決書重點提取
跨卷宗的證據比對（Discovery）

#### 3/ 會計與稅務 (Accounting & Tax)
自動對帳、稅務申報自動化、跨國稅務法規的動態適配

#### 4/ 金融服務 (Finance / Wealth Management)
個人化投資組合分析
KYC/AML 自動化審查
非結構化財報數據提取

### 高度依賴流程與調度的實體產業

#### 5/ 營造與建築：Garry Tan 特別點名的領域
包含工程許可證自動申請
建築圖面與法規的比對
工地進度與物料追蹤

#### 6/ 物流與供應鏈 (Logistics)
跨國貨運報關文件自動生成、動態路線與倉儲空間優化

#### 7/ 製造與工業生產 (Manufacturing)
設備預測性維護的排程、上下游供應商的自動化詢價與議價

#### 8/ 農業與食品科技 (Agriculture & FoodTech)
氣候與產量預測分析、農產品供應鏈溯源管理的自動化

### 重度依賴人力互動的服務產業

#### 9/ 客戶服務 (Customer Service)
不再只是死板的 Chatbot，而是能直接進系統改訂單、退款、調度資源的 L3 級別客服 Agent。

#### 10/ 教育與培訓 (Education) - 佔比 1.8%
一對一的蘇格拉底式 AI 家教
自動批改並給出引導式回饋的助教 Agent

#### 11/ 人力資源與招募 (HR & Recruiting)
自動篩選幾千份履歷並進行初步語音面試
自動安排 onboarding 流程

#### 12/ 政府與公共服務 (GovTech)
將冗長的市民辦事流程（如補助申請、證件換發）
轉化為對話式的 Agent 服務

### 內容與轉換率為王

#### 13/ 銷售與業務開發 (Sales & BDR)
自動分析潛在客戶背景
生成高度客製化的 Cold Email
甚至自動安排會議

#### 14/ 行銷與公關 (Marketing & PR)
根據不同受眾自動生成 A/B 測試文案
社群媒體的自主監控與危機公關預警

#### 15/ 零售與電子商務 (Retail & E-commerce)
從個人的 AI 導購員（能理解「我要去北海道滑雪該買什麼」）
到後台的自動化庫存預測

#### 16/ 媒體與娛樂 (Media & Entertainment)
影音內容的自動標籤與二次剪輯分發
劇本或報導的初期資料蒐集分析

## 策略建議

**最好的策略是挑選一個你懂 Domain-know-how 的領域，鎖定一個人類覺得很煩、流程很固定、但極度耗時的單一節點，把它做到極致。**

# 拆解結果

## A. 主脈絡與個人映射
- **論證骨架**：Garry Tan 引用 Anthropic 數據（SE 佔 49.7%，其他全 <5%）→ 市場極度失衡 → Trust Gap 是產品機會 → Agentic Workflow 的使用模式數據 → 垂直 AI 獨角獸的護城河邏輯 → 16 個待重構產業板塊 → 策略建議（選 Domain，鎖單一節點）。
- **個人映射**：對系統建造者的核心洞見：AI 工具賽道確實擁擠，但那只是同溫層。真正的機會在把 Agent 能力帶到「又髒又重」的垂直領域，而護城河是 Domain Know-how + Change Management，不是技術。

## B. 卡片（Zettel）

序號 1
- 標題：Trust Gap 就是產品機會：AI 能做 5 小時，人類只敢讓它跑 42 分鐘
- 類型：Leverage
- 概念（50–300 字）：Anthropic 數據揭露「AI 能做的」與「人類敢讓它做的」之間的巨大落差：Claude 能獨立解決需要人類 5 小時的工作，但最極端的重度使用者最多讓它連續跑 42 分鐘。老手自動批准率 40%（新手 20%），但老手打斷 AI 頻率 9%（新手 5%）——因為老手已從「每步都要我同意」進化到「放手做，走偏再拉回」。更關鍵的數據：AI 主動發問次數是人類打斷的 2 倍——主動發問是 Feature 不是 Bug。真正的 Autonomy 是由模型、使用者與產品共同建構的。**這個 Trust Gap 就是做產品的巨大機會**——誰能設計出讓人類安心放手的信任機制，誰就能釋放 AI 的真正產能。
- 重要性（1 句）：這是判斷「AI 產品該優先解決什麼問題」的核心框架——不是讓 AI 更強，而是讓人類更敢放手。
- 邊界/反例（1–2 句）：Trust Gap 會隨模型可靠性提升而縮小；某些高風險領域（醫療、法律）的 Trust Gap 可能長期存在，反而構成垂直產品的結構性機會。
- 知識鉤子：與 #EDD品質閉環 形成「如何建立信任」的具體方法；與 #TDD作為控制介面 呼應——TDD 也是一種建立信任的機制。

序號 2
- 標題：垂直 AI 護城河：Domain Know-how + Change Management，不是 API Wrapper
- 類型：Principle
- 概念（50–300 字）：任何人都能寫 API Wrapper，那沒有護城河。真正的護城河是深入醫療帳單、法律搜證、營造合規這些又髒又重的 Domain Know-how，再加上能解決舊系統摩擦力並完成變革管理（Change Management）的能力。Garry Tan 預測：過去 20 年 SaaS 創造 300+ 獨角獸，接下來每個 SaaS 獨角獸都會對應誕生一個 Vertical AI 獨角獸，而且規模大 10 倍——因為它們取代的不只是軟體，還有操作軟體的人。策略：選自己懂的 Domain，鎖定單一痛點節點（人類覺得很煩、流程固定、極度耗時），做到極致。
- 重要性（1 句）：提供了 AI 產品選題的決策框架——不是追技術前沿，而是深入特定領域的痛點。
- 邊界/反例（1–2 句）：Domain Know-how 護城河的前提是領域足夠複雜和受監管；在簡單、標準化的領域（如基本文書處理），護城河很淺。大企業可能用資源優勢快速複製垂直方案。
- 知識鉤子：與 #不可壓縮才是護城河 形成互補——Naval 講個人層的不可壓縮，這裡講企業層的不可壓縮（Domain Know-how）；與 #AI取代SaaS錢搬家了 形成產業鏈上下游對話。
