## 三、怎麼從「GenAI 圖表建議」演進成 Agent？

你現在：

> 金質 Table → 統計描述 → Markdown → GenAI 給圖表建議 → 前端畫圖

可以往下加 2–3 層：

1. **加「任務目標」維度**：

   - 不是只問「要畫什麼圖？」，而是問：

      - 「我要：找問題 / 做週報 / 做實驗結論 / 找新商品 / 做異常監控？」

   - Agent 依「任務類型」選一條流程。

2. **把每個任務拆成一串可重複的 Tool Call**：

   - Query tool、比較 tool、異常偵測 tool、報告組裝 tool、通知 tool …

   - 這樣就有「可持續優化的 Agent Pipeline」。

3. **為每種 Agent 指定「可度量的成果 KPI」**：

   - 週報 Agent：生成報表數量、人工校正時間、錯誤率

   - 成效診斷 Agent：被採用的建議數、採用後 uplift

   - 異常 Agent：提前發現的 incident 數量、損失減少估算

只要做到這三點，你的「GenAI + 圖表建議」就不只是好看的 BI，而是可以**清晰收費、甚至談成果共創**的 Agent 系列。



### 1️⃣ 自動週報／月報 Agent（Reporting Agent）

**做的事**

- 每週 / 每月自動：

   1. 從金質 table 抓指定客戶、場域、campaign 的數據

   2. 計算一組固定 KPI（曝光、觀看、轉換、CTR、CPM、CPV…）

   3. 找出「TOP / BOTTOM 表現」的素材、場域、時段

   4. 生成 Markdown 分析 + 建議圖表（你現在已經會）

   5. 自動輸出成：PDF / PPT 大綱 / 寄給 AE / 上傳到客戶 Portal

**為什麼適合 Agent？**

- 輸入固定（客戶 & 時間區間），輸出格式穩定

- 步驟明確、可工具化 (SQL → 描述 → 圖表 spec → 報表檔案)

- **好驗證**：KPI 可以隨時 SQL 比對

**可收費模式**

- **按量**：

   - 每「客戶 × 月」收報表產出費

   - 例如：每個 end-customer 月報 + 週報共 N 份，超過一定份數加量收費

- **打包進高階方案**：

   - 「自動 BI Copilot 報表」做為 Pro / Enterprise tier 的賣點

### 4️⃣ 異常偵測 & Alert Agent（Monitoring Agent）

**做的事**

- 24x7 監控金質 table 上的核心指標：

   - 例如：播放成功率、AI Camera 上報率、曝光 / 觀看 / 轉換等

- 當指標出現明顯異常（突降 / 突升 / pattern 改變），Agent：

   1. 自動跑一輪「初步 Root Cause 探索」（最近有新素材？有新排程？某些裝置 offline？）

   2. 生成一個簡短診斷 + 建議

   3. 推播到 Slack / Teams / Line：

      - 「XX 連鎖店北區 20 家螢幕今晚觀看率驟降 40%，可能原因：AI Camera 連線異常。」

**為什麼適合 Agent？**

- 拆成清楚 pipeline：監控 → 異常偵測 → 自動查詢 → 生出診斷

- **容易驗證**：

   - 你可以算「提早發現的事件數」、「提早多少時間」、「減少的損失」

**可收費模式**

- **按量**：

   - 按監控的「螢幕數 / 店數 / 指標數」收費（像監控服務線上常見模式）

- **價值說服點**：

   - 減少因系統異常造成的曝光損失 → 可以附案例

### 2️⃣ 廣告成效診斷 & Action List Agent

**做的事**

- 定期掃描金質 table：

   - 對比**前一週 / 前一檔期 / 控制組場域**

   - 找出：

      - 表現明顯下滑的螢幕 / 場域 / 素材 / 受眾包

      - 表現明顯超標的機會點

- 產出一份 **「成效診斷 & 建議動作清單」**，例如：

   - 「下調 A 店午餐時段的咖啡廣告 30% 預算」

   - 「把這支表現好的影片複製到相似受眾包」

- 後期可進階到：

   - 寫入「建議調整表」→ 等人按「同意」，才真正改 CMS 或排程

**為什麼適合 Agent？**

- 任務目標明確：**找問題 + 列行動建議**

- 可以設計一套固定決策規則（例如 drop > 20% 才報、p-value 門檻）

- 可以記錄「建議有被採用嗎？」以及「採用後成效」，形成 feedback loop

**可收費模式**

- **按量**：

   - 按「每月診斷的 campaign / 店數 / 螢幕數」收費

- **按成果**（進階）：

   - 對某些客戶做「成效 uplift sharing」，例如 uplift 帶來的額外營收抽成 X%

### 5️⃣ 受眾包建議 & 定價 Agent（Audience Pack Agent）

**做的事**

- 讀金質 table 中：受眾輪廓、觀看行為、轉換結果

- 自動找出「值得商品化的受眾包」，例如：

   - 「平日午餐時段、商辦區、25–40 歲上班族、對咖啡與輕食有高互動」

- 為每個候選受眾包產出：

   - 基本規模（可觸及人數/曝光量）

   - 過去平均表現 (CTR / CVR / CPM 可接受範圍)

   - 建議售價（例如 CPM 區間）

- 最後產出一份「新受眾商品提案」，可給 Sales 拿去賣。

**為什麼適合 Agent？**

- 工作流程清楚：挖掘 → 評估 → 篩選 → 文件化成商品

- 可被驗證：實際上線後的成效 vs 估計值

- 非常容易跟「Selling the Work」綁在一起：

   - 這個 Agent 的輸出 = 你公司未來多幾個可以賣的產品

**可收費模式**

- 一開始可以內部使用，幫你自己產出商品 line

- 對大型廣告主 / 代理商，可變成「Audience Planning Copilot」

   - 按「每個品牌／每年使用的受眾提案數量」收費
