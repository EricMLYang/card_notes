# 對外系統開發注意事項

- 外場域的基本差別在於: 如何讓系統達到客戶滿意的程度

   - 客戶有自己的時程、預算和優先序的考量，你認為重要的功能，客戶不一定要先做

   - 系統的完成度：UI/UX的設計、程式 error/event handing

   - 維運 incident management，在設計、開發、驗測、試營運等各階段都要納入考量。

   - side event ：**使用者活動、系統監控與日誌、異常處理、數據同步（** 最後可能都會是客戶對系統滿意度的關鍵）

   - 資訊安全(非功能性需求)，資安要展開會說不完，除了黑白箱一定要做外，還有就是操作型資安考量(ex: 密碼規則、資料輸入驗證、回上頁漏洞…等等)

   - 效能包含系統運作的穩定與回應速度，另外更重要的是使用者操作的效率(ex: CRUD最便捷路線、避免page reload…等等)，這和 UX設計會有高度相關(操作流程的設計，畫面分割和比例的規劃、功能性觸發物件的配置，眼球移動的軌跡，重點資訊的易讀性…等等)，也會和前端framework的應用有關 (Vue: dataset, Text compression, Reduce unused JavaScript, Static assets with cache policy…等等 )
