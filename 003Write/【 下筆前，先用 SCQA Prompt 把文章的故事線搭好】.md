▋ 愛不釋手的老書，在 AI 時代還是適用

《麥肯錫教我的思考武器》是一本 2012 年出版的老書。

記得我年輕的時候看到這本書滿震撼的，

非常喜歡書裡面的觀念，放到現在 AI 時代都還適用。

例如

「議題度 & 解答值」，

白話文就是：別浪費時間在「不重要」或「無法解決」的問題上。

這在 AI 時代的意義尤其深刻：AI 可以回答任何問題，但你要先問對問題。



「假設思考」，

研究前先猜一個答案，然後去證明對錯。

體現在 AI 提問上：爛問法是：「AI，分析一下市場」。

好問法是：「AI，我猜年輕人不買是因為價格太貴，幫我驗證」。



「MECE 拆解」

把大問題切成小塊，確保沒有遺漏或重複。

AI 容易天馬行空，你需要給它「框架」來整理思考。



「組成故事線」，

有了答案還不夠，要包裝成別人聽得懂、想聽的故事。

畢竟 AI 能產生內容，但只有你知道老闆愛聽什麼。



這些經典的思考工具，至今仍是我分析與溝通的基石。

▋ 將思考武器，轉化為高效的寫作流程

其中議題的 MECE 拆解，和組成故事線，

一直到現在我做簡報都還是被這觀念影響。

近期在練習寫作，

有很多 prompt 在協助「選題」上很有幫助。

但因為工作上關係，

我常常要針對一些問題做「分析」。

我需要一個方法，

可以先拆完一個「分析型標題」再開始寫稿。

因此，我嘗試一個拆解分析型標題的 prompt。

我期望這個 prompt 能幫我建立文章架構，大幅提升寫作效率。

---



▋ 核心任務：打造從「議題」到「故事線」的架構師

這個 prompt 的名稱是：

「# 從『議題』到『故事線』的深度文章架構師」。

T: Task (工作指引)

它的任務是扮演一名資深的「策略架構師」與「溝通顧問」。

核心工作，

是將一個「分析型文章標題」（這代表一個複雜的核心議題），

轉化為一個具有高度邏輯性、說服力、且易於閱讀的「故事線大綱 (Storyline Outline)」。

我們必須假定，

這篇文章的最終讀者是忙碌的決策者。

他們沒有時間閱讀雜亂無章的分析，

只關心「情境、衝突、與解答」。



C: Context (任務脈絡)

我（作者／分析師）已經確定了想探討的核心議題（即標題），

但我需要一個清晰的「論述藍圖」來指導我撰寫。

這份大綱不是文章本身，而是指揮文章寫作的「施工圖」。

---



▋ 步驟一：議題拆解 (Deconstruction)

R: Reference (參照點)

為了產出最高品質的故事線，必須嚴格遵循兩大步驟。

步驟 1：議題拆解 (Deconstruction) - 運用 MECE 與邏輯框架。

首先，要解析議題 (Analyze the Issue)，

辨識出【文章標題】中隱含的核心「問題」或「主張」。

接著，要選用框架 (Select Framework)，

選擇一個最適合分析此議題的「既有邏輯框架」。

例如：

如果標題是關於市場進入，你可能會選用 3C 或 SWOT。

如果標題是關於原因分析，你可能會用「Why-What-How」或「問題-原因-對策」。

最後，要應用 MECE (Apply MECE)，

運用你選擇的框架，

將這個核心議題拆解為 3-5 個關鍵組成部分，

並確保這些部分符合 MECE 原則（相互獨立、完全窮盡）。

這個拆解步驟，是將混亂思緒結構化的關鍵。

---



▋ 步驟二：故事組成 (Assembly)

步驟 2：故事組成 (Assembly) - 搭建 SCQA 故事線。

你必須使用經典的「SCQA 敘事框架」，

來組織你剛才拆解的 MECE 組件。

這能將冰冷的分析轉化為引人入勝的故事。

S (Situation - 情境)：

描繪一個讀者普遍認同的「背景」或「現狀」。

(這通常是文章的開頭)



C (Complication - 衝突)：

點出目前發生的「變化」、「問題」或「挑戰」。

(這通常就是你的核心議題)



Q (Question - 疑問)：

基於上述的衝突，自然地導出這篇文章必須回答的「核心問題」。



A (Answer / Analysis - 回答與分析)：

這是文章的主體。

你必須將【步驟 1】中 MECE 拆解的 3-5 個關鍵組件，

作為回答這個核心問題的「主要論點」，並依序展開。



透過 SCQA 框架，冰冷的分析就能被轉化為有說服力的故事。

---



▋ 最終產出：一份清晰的「故事線大綱」

產出格式 (Output Format) 必須非常清晰。

請使用清晰的 Markdown 階層格式（H1, H2, H3），

來呈現這份「故事線大綱」。

重點是，不要只給標題 (Topic)，

例如，不要只寫「市場分析」。

請給出關鍵訊息 (Key Message)，

例如：「H2: 市場趨F勢：AI 正在重塑使用者期待，但現有工具已無法滿足」。

最後，

只要在【 請在這裡貼上你的分析型文章標題 】處，

貼上你想分析的標題即可。

\[範例標題：「AI 浪潮下，B2B 企業如何應對『數據孤島』挑戰並實現精準行銷？」\]

這樣產出的「施工圖」，才能確保分析文章真正命中要點。

我（作者／分析師）已經確定了我想探討的核心議題（如下方標題所示），

它必須確保我的論點清晰、結構完整、且從頭到尾都能抓住讀者的注意力。

## R: Reference (參照點)

為了產出最高品質的故事線，你必須嚴格遵循以下兩大步驟與格式要求：

步驟 1：議題拆解 (Deconstruction) - 運用 MECE 與邏輯框架

- MECE 拆解 (Apply MECE)： 運用你選擇的框架，將這個核心議題拆解為 3-5 個關鍵組成部分。你必須確保這些部分符合 MECE 原則（相互獨立、完全窮盡）。在你的思考過程中（非正式產出），要能解釋你為何這樣拆解。

你必須使用經典的「SCQA 敘事框架」來組織你剛才拆解的 MECE 組件。這能將冰冷的分析轉化為引人入勝的故事。

- C (Complication - 衝突)： 點出目前發生的「變化」、「問題」或「挑戰」。(這通常就是你的核心議題)
- Q (Question - 疑問)： 基於上述的衝突，自然地導出這篇文章必須回答的「核心問題」。
- A (Answer / Analysis - 回答與分析)： 這是文章的主體。你必須將【步驟 1】中 MECE 拆解的 3-5 個關鍵組件，作為回答這個核心問題的「主要論點」，並依序展開。
