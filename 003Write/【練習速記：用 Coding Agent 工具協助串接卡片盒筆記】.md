【練習速記：用 Coding Agent 工具協助串接卡片盒筆記】



最近上一堂課，有一章節是利用 Cursor 來串接卡片盒筆記，



我自己初步嘗試，感覺很有潛力，



將自己邊做邊學的經驗分享，



讓其他想精進自己知識彙整流程的人可以少走一點彎路



▋卡片盒筆記要重點 - 一張卡片只能一個觀念



一張卡片不自覺塞太多觀念是我之前最大的敗筆，



在卡片盒筆記中，



卡片是未來串連的最小單位，



如果塞太多觀念變小短文，



串接效果就出不來，



反而變成很畸形的筆記，



目前陸續再把以前打卡片變成只有單一核心觀念的小卡片





▋卡片盒筆記專注在串連後產出文章

上課老師提到，



卡片盒筆記從紀錄到產出文章最快速最短路徑的一種筆記，



你可以很快的串連不同卡片後，



產生一個新的主題，



調整一下就變成一篇新文章了，



因此我目前就是把卡片盒筆記當作串連想法跟產出文章用，



跟我在 Heptabase 的白板主題，



和多 Tag 的分類筆記做區隔，



之前的沒有單一知識中心的糾結被消除



▋Inbox > Index > Cards 結構

卡片盒筆記我會以地端環境進行，

編輯用 Obsidian,  和 VSCode 做切換，

並輔以 Git, GitHub 版控，

最後整個資料夾會放在 Google Drive 桌面版資料夾方便偶爾 AI Chat 引入，

這樣的環境下，

重心資料夾只有三個：

\-inbox: 待處理內容

\-Cards: 只剩單一核心觀念的每張卡片

\-index: 用 Obsidian 連結功能的指引，將 卡片的分大類，連結到 index 的一個檔案上 EX: index 1 - AI 應用，將所有相關卡片連到這個檔案上



就這樣，持續累積卡片，等待連結的威力湧現



▋串接測試

目前大概才整理了 10％的卡片，約 22 張，



我先用 GitHub Copilot  Agent 測試連結效果，



我請 AI 幫我 串連至少 5 張卡片，並給一個文章主題，
